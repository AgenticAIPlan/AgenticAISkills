# 飞桨星河社区项目评审专家Skill - GitHub提交指南

## 🚀 快速提交（推荐）

### 方法1：通过GitHub网页提交（最简单）

#### 步骤1：Fork仓库
1. 访问：https://github.com/AgenticAIPlan/AgenticAISkills
2. 点击右上角的 **Fork** 按钮
3. Fork到你自己的GitHub账号下

#### 步骤2：创建分支
1. 进入你Fork后的仓库
2. 点击分支下拉菜单
3. 输入分支名：`feat/aistudio-project-reviewer`
4. 点击 **Create branch: feat/aistudio-project-reviewer**

#### 步骤3：上传文件

在 `feat/aistudio-project-reviewer` 分支下，创建以下目录结构和文件：

**目录结构：**
```
skills/
└── aistudio-project-reviewer/
    ├── SKILL.md
    ├── assets/
    │   ├── aistudio-project-reviewer.skill.json
    │   └── run_review.py
    └── references/
        ├── DESIGN.md
        ├── README.md
        └── templates/
            ├── 评分标准模板.md
            ├── 回复模板库.md
            └── 快速检查清单.md
```

**文件内容获取：**
所有文件的内容都在本地目录：
`/Users/v_wangminglei/ComateProjects/comate-zulu-demo/skills/aistudio-project-reviewer/`

你需要：
1. 打开每个文件
2. 复制内容
3. 在GitHub上创建对应文件并粘贴

#### 步骤4：提交更改
1. 填写 **Commit message**：
   ```
   feat: 添加飞桨星河社区项目评审专家Skill

   - 实现多维度项目评审（创新性、技术深度、文档质量、可复现性、社区价值）
   - 添加AI生成内容检测与减分机制
   - 提供快速筛选功能
   - 包含完整的模板资源库和评分标准
   - 实现可执行评审脚本
   ```
2. 点击 **Commit new file**

#### 步骤5：创建Pull Request
1. 在你的Fork仓库中，点击 **Pull requests**
2. 点击 **New pull request**
3. 确认信息：
   - **Base repository**: AgenticAIPlan/AgenticAISkills
   - **Base branch**: dev
   - **Head repository**: [你的用户名]/AgenticAISkills
   - **Compare branch**: feat/aistudio-project-reviewer
4. 点击 **Create pull request**
5. 填写PR信息：
   - **Title**: `feat: 添加飞桨星河社区项目评审专家Skill`
   - **Description**:
     ```markdown
     ## 变更内容

     - 新增飞桨星河社区项目评审专家Skill
     - 符合仓库规范：skills/<skill-slug>/ 结构
     - 包含完整的SKILL.md、assets/和references/
     - 实现AI生成内容检测与减分机制

     ## 文件清单

     - skills/aistudio-project-reviewer/SKILL.md
     - skills/aistudio-project-reviewer/assets/aistudio-project-reviewer.skill.json
     - skills/aistudio-project-reviewer/assets/run_review.py
     - skills/aistudio-project-reviewer/references/DESIGN.md
     - skills/aistudio-project-reviewer/references/README.md
     - skills/aistudio-project-reviewer/references/templates/
     ```
6. 点击 **Create pull request**

### 方法2：使用Git命令行（推荐给有经验者）

#### 步骤1：准备本地文件
确认你的本地文件在正确的目录：
```
/Users/v_wangminglei/ComateProjects/comate-zulu-demo/
```

#### 步骤2：Fork仓库
1. 访问：https://github.com/AgenticAIPlan/AgenticAISkills
2. 点击 **Fork** 按钮

#### 步骤3：克隆你的Fork
```bash
# 替换 YOUR_USERNAME 为你的GitHub用户名
git clone https://github.com/YOUR_USERNAME/AgenticAISkills.git
cd AgenticAISkills

# 添加原始仓库为upstream
git remote add upstream https://github.com/AgenticAIPlan/AgenticAISkills.git
```

#### 步骤4：复制文件到Fork仓库
```bash
# 从你的项目目录复制文件
cp -r /Users/v_wangminglei/ComateProjects/comate-zulu-demo/skills/* /path/to/AgenticAISkills/skills/
```

#### 步骤5：提交并推送
```bash
# 创建分支
git checkout -b feat/aistudio-project-reviewer

# 添加文件
git add .

# 提交
git commit -m "feat: 添加飞桨星河社区项目评审专家Skill

- 实现多维度项目评审（创新性、技术深度、文档质量、可复现性、社区价值）
```

1. 访问：https://github.com/AgenticAIPlan/AgenticAISkills/compare/dev...YOUR_USERNAME:feat/aistudio-project-reviewer
2. 填写PR信息（同方法1步骤5）

## 🆘 遇到问题？

### 推送失败
确保你已经Fork了仓库，并且remote指向你的Fork。

### PR创建失败
确认：
- 目标分支是 `dev` 不是 `main`
- 你的Fork仓库是最新的（从upstream拉取最新代码）

### GitHub Actions失败
检查：
- 目录结构是否正确
- 文件命名是否符合规范
- SKILL.md是否存在
