# 数据资源评估 - 参考文档

## 数据表字段设计（可选）

如果需要在知识库中创建数据表进行结构化管理，可参考以下字段设计：

| 字段名 | 字段类型 | 说明 | 示例 |
|--------|----------|------|------|
| 提交时间 | 日期 | 数据资源提交评估的时间 | 2026-04-08 |
| 数据来源 | 文本 | 企业或伙伴名称 | XX科技公司 |
| 评估类型 | 单选 | 资源置换/数商数据线索 | 资源置换 |
| 数据样例描述 | 文本 | 数据类型和内容描述 | 医疗行业图文对 |
| 数据量级 | 文本 | 数据规模 | 50万条，约500GB |
| 数据格式 | 文本 | 文件格式 | PDF, 图片 |
| 数据内容 | 文本 | 分布、模态等详细信息 | 包含文本和图像，主要为中文 |
| 数据来源/制作方式 | 文本 | 数据来源说明 | 自有数据，通过XX方式采集 |
| 质量控制方法 | 文本 | 质检方法说明 | 人工标注，准确率95% |
| 负责人 | 单选 | 张红鸾/杨威 | 张红鸾 |
| 当前状态 | 单选 | 待评估/评估中/已完成/已关闭 | 待评估 |
| 友商采购情况 | 文本 | 仅数商线索需要 | XX友商采购了100万条，金额XX万 |
| 资源置换意向 | 文本 | 仅资源置换需要 | 希望置换XX资源 |
| 备注 | 文本 | 其他补充信息 | 联系人：张三 |
| 跟进记录 | 文本 | 每周跟进记录 | 2026-04-08: 已提交评估 |

## 状态流转

```
待评估 → 评估中 → 已完成
                 ↘ 已关闭
```

## 常见问题

### Q1: 如何判断是资源置换还是数商线索？

- **资源置换**：合作伙伴希望通过数据交换、生态合作方式获取资源，不涉及现金采购
- **数商数据线索**：数据提供商希望通过现金销售的方式出售数据

### Q2: 如果用户信息不完整怎么办？

主动询问缺失的字段，特别是：
- 资源置换：资源置换意向说明
- 数商线索：友商采购情况

### Q3: 定时提醒发送给谁？

定时提醒发送给当前会话的用户，提醒内容包含知识库链接。

## 搜索关键词

在知识库文档中搜索时，可使用以下 grep 模式：

```bash
# 搜索特定数据来源
grep "数据来源.*XX公司" [文档]

# 搜索特定评估类型
grep "评估类型.*资源置换" [文档]

# 搜索待评估状态
grep "当前状态.*待评估" [文档]

# 搜索特定日期
grep "2026-04-08" [文档]
```
