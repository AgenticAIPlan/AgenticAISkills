# 数据资源评估

## 概述

此 skill 用于管理外部伙伴数据资源的评估流程。将数据资源按类型（资源置换/数商线索）提交给对应负责人，同步记录到知识库文档，并设置定时跟进提醒。

## 核心流程

```
┌─────────────────┐
│  接收评估请求    │
└────────┬────────┘
         ▼
┌─────────────────┐
│  判断评估类型    │
│  资源置换/数商   │
└────────┬────────┘
         ▼
┌─────────────────┐
│  收集数据信息    │
│  使用标准模板    │
└────────┬────────┘
         ▼
┌─────────────────┐
│  发送评估消息    │
│  给对应负责人    │
└────────┬────────┘
         ▼
┌─────────────────┐
│  同步更新知识库  │
│  追加评估记录    │
└────────┬────────┘
         ▼
┌─────────────────┐
│  确认定时提醒    │
│  每周五跟进      │
└─────────────────┘
```

## 第一步：判断评估类型

根据用户意图判断数据资源的评估类型：

### 资源置换（生态合作）→ 发送给张红鸾
**触发词**：资源置换、生态合作、生态资源置换、合作评估、数据交换

**适用场景**：合作伙伴希望通过数据交换、生态合作方式获取资源，不涉及现金采购

### 数商数据线索（采购）→ 发送给杨威
**触发词**：数据采购、数商线索、采购评估、线索提交、现金采购

**适用场景**：数据提供商希望通过现金销售的方式出售数据

## 第二步：收集数据信息

根据评估类型，使用对应模板收集信息。如果用户未提供完整信息，需主动询问缺失字段。

### 资源置换模板（对接张红鸾）

```markdown
**数据来源**：（企业/伙伴）
**数据样例及描述**：（XX行业图文对，文本，古籍等）
**数据量级**：（xx万级，xxT）
**数据格式**：（PDF, 图片，视频，音频等）
**数据内容**：（分布、模态）
**数据来源/制作方式**：（自有数据，或者伙伴通过什么方式获取的数据，若有模型生成内容需说明使用的模型版本）
**质量控制方法**：（来源方的质检、标注后数据准确率，如何保证正确率or质检模型版本）
**资源置换意向说明**：（生态合作诉求说明，用于判断合作可行性；如置换条件为现金采购，按数商线索处理）
```

### 数商数据线索模板（对接杨威）

```markdown
**数据来源**：（企业/伙伴）
**数据样例及描述**：（XX行业图文对，文本，古籍等）
**数据量级**：（xx万级，xxT）
**数据格式**：（PDF, 图片，视频，音频等）
**数据内容**：（分布、模态）
**数据来源/制作方式**：（自有数据，或者伙伴通过什么方式获取的数据，若有模型生成内容需说明使用的模型版本）
**质量控制方法**：（来源方的质检、标注后数据准确率，如何保证正确率or质检模型版本）
**友商采购情况**：（友商采购量级、金额、是否独家等，越详细越好）
```

使用 `infoflow_send` 工具发送如流消息给对应负责人。

```
  ---

  提交人：xxx
```

### 数商数据线索 → 杨威

```
to: "yangwei18"

  【数据资源评估 - 数商线索】
  
  **数据来源**：xxx
  **数据样例及描述**：xxx
  ...（使用数商线索模板格式化的完整信息）
```

### 发送要求

- 使用 Markdown 格式发送，便于阅读
- 包含完整的数据信息，不要遗漏字段
- 在消息末尾标注提交时间和提交人
- 发送成功后，立即执行下一步同步知识库

## 第五步：设置定时跟进提醒

使用 `cron` 工具创建定时任务，每周五上午 10:00 提醒跟进数据表进度。

### 检查现有任务

首先使用 `cron` 的 `list` 操作检查是否已存在"数据资源评估周跟进"任务。

### 创建定时任务

如果不存在，则创建：

```
action: add
job:
  name: "数据资源评估周跟进"
  schedule:
    kind: "cron"
    expr: "0 10 * * 5"
    tz: "Asia/Shanghai"
  payload:
    kind: "systemEvent"
    text: "【数据资源评估周跟进提醒】请检查数据资源评估进度，跟进本周提交的评估。知识库链接：https://ku.baidu-int.com/knowledge/HFVrC7hq1Q/YdCYR5HH9T/yDvRJac9u1/-sQu8hODwG2i_h"
```

### 注意事项

- 避免重复创建相同的定时任务
- 提醒内容包含知识库链接，便于快速访问
- 如需调整跟进频率或时间，使用 `cron` 的 `update` 功能修改

## 使用示例

### 示例 1：资源置换评估

**用户输入**：
> 这是一个外部合作伙伴的数据，需要做资源置换评估。数据来源是XX科技公司，数据是医疗行业图文对，大约50万条，主要是PDF和图片格式...

**执行流程**：
1. 识别为"资源置换"类型
2. 使用资源置换模板收集完整信息（询问缺失字段）
3. 发送给张红鸾（zhanghongluan）
4. 同步更新到知识库文档
5. 确认定时跟进任务已设置

### 示例 2：数商数据线索

**用户输入**：
> 有个数据采购线索，XX公司有大量古籍数据，需要提交给采购团队评估

**执行流程**：
1. 识别为"数商数据线索"类型
2. 使用数商线索模板收集信息（重点询问友商采购情况）
3. 发送给杨威（yangwei18）
4. 同步更新到知识库文档
5. 确认定时跟进任务已设置
