# 数据资源评估

## 概述

此 skill 用于管理外部伙伴数据资源的评估流程。将数据资源按类型（资源置换/数商线索）提交给对应负责人，同步记录到知识库文档，并设置定时跟进提醒。

## 核心流程

```
┌─────────────────┐
│  接收评估请求    │
└────────┬────────┘
         ▼
┌─────────────────┐
│  判断评估类型    │
│  资源置换/数商   │
└────────┬────────┘
         ▼
┌─────────────────┐
│  收集数据信息    │
│  使用标准模板    │
└────────┬────────┘
         ▼
┌─────────────────┐
│  发送评估消息    │
│  给对应负责人    │
└────────┬────────┘
         ▼
┌─────────────────┐
│  同步更新知识库  │
│  追加评估记录    │
└────────┬────────┘
         ▼
┌─────────────────┐
│  确认定时提醒    │
│  每周五跟进      │
└─────────────────┘
```

## 第一步：判断评估类型

根据用户意图判断数据资源的评估类型：

### 资源置换（生态合作）
**触发词**：资源置换、生态合作、生态资源置换、合作评估、数据交换

**适用场景**：合作伙伴希望通过数据交换、生态合作方式获取资源，不涉及现金采购

**配置项**：负责人信息通过 `responsible_persons.resource_exchange` 配置

### 数商数据线索（采购）
**触发词**：数据采购、数商线索、采购评估、线索提交、现金采购

**适用场景**：数据提供商希望通过现金销售的方式出售数据

**配置项**：负责人信息通过 `responsible_persons.data_vendor_leads` 配置

---

## 第二步：收集数据信息

根据评估类型，使用对应模板收集信息。如果用户未提供完整信息，需主动询问缺失字段。

### 资源置换模板

```markdown
**数据来源**：（企业/伙伴）
**数据样例及描述**：（XX行业图文对，文本，古籍等）
**数据量级**：（xx万级，xxT）
**数据格式**：（PDF, 图片，视频，音频等）
**数据内容**：（分布、模态）
**数据来源/制作方式**：（自有数据，或者伙伴通过什么方式获取的数据，若有模型生成内容需说明使用的模型版本）
**质量控制方法**：（来源方的质检、标注后数据准确率，如何保证正确率or质检模型版本）
**资源置换意向说明**：（生态合作诉求说明，用于判断合作可行性；如置换条件为现金采购，按数商线索处理）
```

### 数商数据线索模板

```markdown
**数据来源**：（企业/伙伴）
**数据样例及描述**：（XX行业图文对，文本，古籍等）
**数据量级**：（xx万级，xxT）
**数据格式**：（PDF, 图片，视频，音频等）
**数据内容**：（分布、模态）
**数据来源/制作方式**：（自有数据，或者伙伴通过什么方式获取的数据，若有模型生成内容需说明使用的模型版本）
**质量控制方法**：（来源方的质检、标注后数据准确率，如何保证正确率or质检模型版本）
**友商采购情况**：（友商采购量级、金额、是否独家等，越详细越好）
```

---

## 第三步：发送评估消息

**⚠️ 重要：发送前必须向用户确认**

在发送消息前，必须向用户展示完整的消息内容和目标接收人，并获得用户明确确认。

### 确认流程

1. 展示格式化的消息内容
2. 显示目标接收人（从配置读取：`responsible_persons.resource_exchange.account` 或 `responsible_persons.data_vendor_leads.account`）
3. 询问用户是否确认发送
4. 用户确认后再执行发送操作

### 发送示例

```
使用 infoflow_send 工具：

- to: 从配置读取负责人账号
- message: 使用 Markdown 格式的评估信息
```

### 发送要求

- 使用 Markdown 格式发送，便于阅读
- 包含完整的数据信息，不要遗漏字段
- 在消息末尾标注提交时间和提交人
- **必须获得用户确认后再发送**
- 发送成功后，立即执行下一步同步知识库

---

## 第四步：同步更新知识库

**⚠️ 重要：更新前必须向用户确认**

每次发送评估消息后，需要同步将数据信息更新到知识库文档。但在执行更新前，必须获得用户确认。

### 知识库配置

从 Skill 元数据配置读取：
- **文档链接**：`knowledge_base.doc_url`
- **知识库 ID**：`knowledge_base.kb_id`
- **文档 ID**：`knowledge_base.doc_id`

### 确认流程

1. 向用户展示即将追加的记录内容
2. 显示目标知识库文档链接（从配置读取）
3. 询问用户是否确认更新
4. 用户确认后再执行更新操作

### 操作步骤

1. 使用 `ku-doc-manage` skill 获取文档当前内容
2. 在文档末尾追加新的评估记录
3. 使用 `ku-doc-manage` skill 的编辑文档功能更新文档内容

### 追加记录格式

```markdown
## [YYYY-MM-DD] 数据资源评估记录

### 基本信息
- **数据来源**：[企业/伙伴名称]
- **评估类型**：[资源置换/数商数据线索]
- **提交人**：[用户名]
- **负责人**：[从配置读取]

### 数据详情
- **数据样例及描述**：[描述]
- **数据量级**：[量级]
- **数据格式**：[格式]
- **数据内容**：[分布、模态]
- **数据来源/制作方式**：[说明]
- **质量控制方法**：[说明]

### 其他信息
- **友商采购情况**（数商线索）：[详情]
- **资源置换意向说明**（资源置换）：[说明]

### 状态
- **当前状态**：待评估
- **备注**：[其他补充]

---
```

## 第五步：设置定时跟进提醒

**⚠️ 重要：创建前必须向用户确认**

使用 `cron` 工具创建定时任务前，必须向用户确认提醒配置。

### 定时提醒配置

从 Skill 元数据配置读取：
- **是否启用**：`reminder.enabled`
- **执行时间**：`reminder.schedule`（cron 表达式）
- **时区**：`reminder.timezone`
- **提醒内容**：`reminder.message`

### 确认流程

1. 向用户展示提醒配置（时间、频率、内容）
2. 询问用户是否需要创建或修改提醒
3. 用户确认后再执行创建操作

### 检查现有任务

首先使用 `cron` 的 `list` 操作检查是否已存在相关定时任务。

### 创建定时任务

如果用户确认创建且不存在重复任务，则根据配置创建：

```
action: add
job:
  name: "数据资源评估周跟进"
  schedule:
    kind: "cron"
    expr: 从配置读取（默认："0 10 * * 5" 每周五上午 10:00）
    tz: 从配置读取（默认："Asia/Shanghai"）
  payload:
    kind: "systemEvent"
    text: 从配置读取提醒内容
```

### 注意事项

- 避免重复创建相同的定时任务
- 提醒内容可通过配置自定义
- 如需调整跟进频率或时间，使用 `cron` 的 `update` 功能修改
- **必须获得用户确认后再创建**

## 使用示例

### 示例 1：资源置换评估

**用户输入**：
> 这是一个外部合作伙伴的数据，需要做资源置换评估。数据来源是XX科技公司，数据是医疗行业图文对，大约50万条，主要是PDF和图片格式...

**执行流程**：
1. 识别为"资源置换"类型
2. 使用资源置换模板收集完整信息（询问缺失字段）
3. 向用户展示消息内容和目标接收人，获得确认后发送
4. 向用户展示知识库更新内容，获得确认后更新
5. 询问用户是否需要设置定时提醒，确认后创建

### 示例 2：数商数据线索

**用户输入**：
> 有个数据采购线索，XX公司有大量古籍数据，需要提交给采购团队评估

**执行流程**：
1. 识别为"数商数据线索"类型
2. 使用数商线索模板收集信息（重点询问友商采购情况）
3. 向用户展示消息内容和目标接收人，获得确认后发送
4. 向用户展示知识库更新内容，获得确认后更新
5. 询问用户是否需要设置定时提醒，确认后创建

## 部署指南

### 1. 配置 Skill

在使用此 Skill 前，需要根据实际情况修改元数据配置：

1. 设置负责人信息：
   - `responsible_persons.resource_exchange`：资源置换负责人
   - `responsible_persons.data_vendor_leads`：数商线索负责人

2. 设置知识库信息：
   - `knowledge_base.doc_url`：知识库文档链接
   - `knowledge_base.kb_id`：知识库 ID
   - `knowledge_base.doc_id`：文档 ID

3. （可选）设置定时提醒：
   - `reminder.enabled`：是否启用提醒
   - `reminder.schedule`：提醒时间（cron 表达式）
   - `reminder.message`：提醒内容

### 2. 安全注意事项

- 不要在公开仓库中提交具体的负责人账号、内部知识库链接等敏感信息
- 使用配置项或环境变量管理敏感信息
- 定期检查和更新配置

### 3. 用户确认机制

此 Skill 在执行关键操作前会请求用户确认：

- 发送消息前：展示消息内容和目标接收人
- 更新知识库前：展示更新内容
- 创建定时提醒前：展示提醒配置

这确保了操作的可控性和透明性。
