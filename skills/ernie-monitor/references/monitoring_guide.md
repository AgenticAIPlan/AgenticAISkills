## 监测概述

### 监测目标

1. **全面性**：覆盖文心大模型相关的所有公开讨论
2. **时效性**：确保数据在24小时内更新
3. **准确性**：通过多源验证确保数据准确
4. **价值性**：提取有价值的洞察和建议

### 监测原则

| 原则 | 说明 | 执行标准 |
|------|------|----------|
| 合规性 | 遵守平台规则和法律法规 | 不使用非法手段获取数据 |
| 隐私保护 | 保护用户隐私 | 不收集个人敏感信息 |
| 客观中立 | 保持分析客观 | 避免选择性偏差 |
| 安全保密 | 保护监测数据 | 数据加密存储 |

## 平台详细说明

### 1. 微信公众号

**特点**：深度内容、专业分析、传播范围广

**数据获取方式**：
- 微信公众平台API（需认证）
- 搜狗微信搜索
- 第三方数据平台（如新榜、清博）

**监测重点**：
- 科技媒体账号（36氪、虎嗅、量子位等）
- AI领域大V
- 百度官方账号

**数据字段**：
```
- 标题
- 作者/公众号名称
- 发布时间
- 阅读量
- 点赞数
- 在看数
- 分享数（估算）
- 文章内容
- 文章链接
```

**注意事项**：
- 微信阅读量有延迟，需等待更新
- 部分账号开启转载保护，需注意版权
- 付费内容无法获取完整内容

## 数据处理规范

### 去重规则

#### 1. 完全重复
- URL完全相同
- 标题和作者完全相同

#### 2. 内容相似
- 标题相似度 > 90%
- 正文相似度 > 85%

#### 3. 转载处理
- 保留原始来源
- 标注转载关系
- 合并互动数据

### 分类标准

#### 内容分类

| 类别 | 判断标准 | 优先级 |
|------|---------|--------|
| 产品发布/更新 | 包含发布、更新、版本等关键词 | 高 |
| 用户体验 | 包含体验、评测、测试等关键词 | 高 |
| 技术分析 | 包含技术、分析、性能等关键词 | 中 |
| 行业观点 | 包含观点、行业、分析等关键词 | 中 |
| 争议/负面 | 包含争议、问题、bug等关键词 | 高 |
| 教程/分享 | 包含教程、指南、技巧等关键词 | 低 |
| 友商动态 | 包含友商、竞争、对比等关键词 | 中 |

### 情感分析

#### 情感分类

| 情感 | 定义 | 特征 |
|------|------|------|
| 正向 | 表达满意、称赞、推荐 | 包含正面词汇 |
| 负向 | 表达不满、批评、质疑 | 包含负面词汇 |
| 中性 | 客观描述、无情感倾向 | 无明显情感词汇 |
| 混合 | 同时包含正负面 | 正负面词汇都有 |

#### 情感评分

```
正向得分 = 正面词汇数量 × 1
负向得分 = 负面词汇数量 × 1
情感分 = 正向得分 - 负向得分

情感分类规则：
- 情感分 > 0：正向
- 情感分 < 0：负向
- 情感分 = 0 且有情感词汇：混合
- 情感分 = 0 且无情感词汇：中性
```

## 质量保证

### 数据质量检查

#### 1. 完整性检查
- 所有必需字段是否完整
- 链接是否有效
- 时间格式是否正确

#### 2. 准确性检查
- 相关性验证（人工抽检）
- 分类准确性（抽样验证）
- 情感分析准确性（人工复核）

#### 3. 及时性检查
- 数据延迟是否在允许范围内
- 历史数据是否及时归档

### 质量标准

| 指标 | 目标值 | 检查方法 |
|------|--------|----------|
| 数据完整性 | > 99% | 自动检查 |
| 分类准确率 | > 90% | 抽样验证 |
| 情感分析准确率 | > 85% | 抽样验证 |
| 数据延迟 | < 24小时 | 系统监控 |

## 附录

### A. 常见问题

**Q: 如何处理付费内容？**
A: 付费内容无法直接获取，可以通过公开标题和摘要进行分析，或等待公开转载。

**Q: 如何识别水军和机器人？**
A: 通过发布频率、内容相似度、账号注册时间、互动模式等特征进行识别。

**Q: 数据存储有什么要求？**
A: 原始数据保留至少6个月，分析报告保留至少1年，敏感信息需要加密存储。
