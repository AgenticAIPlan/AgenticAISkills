# AI 硬件生态分类法

## 用途

用这份分类法统一 AI 硬件生态监测中的环节、主题、影响维度和信号词，避免不同报告对同一新闻使用不一致的标签体系。

## 环节分类

### 上游

- 晶圆与制程
- 先进封装与测试
- HBM / DRAM / NAND
- 基板、互连、光模块、电源、散热

### 中游

- GPU、NPU、ASIC、加速卡
- 服务器整机、机柜、液冷、网络设备
- ODM / OEM、板卡与模组厂商

### 下游

- 云厂商
- 数据中心与算力基础设施运营方
- 企业客户与行业解决方案商
- 开发者和开源生态

### 模型厂商

- 通用大模型厂商
- 垂类模型厂商
- 开源模型社区和推理平台

## 主题分类

使用以下主题标签：

1. **product_release**：新品发布、规格更新、路线图
2. **upstream_supply**：产能、良率、封装、HBM、关键器件供给
3. **compute_infrastructure**：服务器、集群、液冷、网络、数据中心建设
4. **model_vendor_strategy**：模型厂商采购、部署、迁移、合作和替代
5. **cloud_deployment**：云厂商上架、算力服务、推理基础设施扩张
6. **price_capacity**：价格、报价、capex、交付周期、库存
7. **partnership_ecosystem**：合作伙伴、案例、认证、生态扩展
8. **software_stack**：驱动、SDK、软件栈、兼容与部署工具
9. **policy_compliance**：制裁、出口管制、合规、监管、标准
10. **technical_issue**：过热、兼容问题、崩溃、部署失败、召回
11. **competitor_watch**：竞品发布、替代趋势、结构性变化

## 影响维度

围绕以下维度判断新闻价值和影响：

- 是否影响供给与交付
- 是否影响模型训练或推理成本
- 是否影响云厂商和企业客户采购
- 是否影响部署效率和稳定性
- 是否改变上下游合作关系或替代趋势
- 是否带来政策、合规或资本开支变化

## 关键信号词

### 风险信号

- 缺货 / 断供 / 跳票 / 砍单
- 制裁 / 禁售 / 出口管制 / 合规限制
- 过热 / 崩溃 / 兼容问题 / 部署失败
- 价格波动 / 报价上调 / 资本开支收缩
- 良率问题 / 封装瓶颈 / HBM 紧张

### 正向信号

- 扩产 / 交付恢复 / 供给改善
- 新合作伙伴 / 新客户 / 新部署案例
- 推理成本下降 / 部署效率提升 / 性能提升
- 路线图推进 / 生态扩张 / 认证完成
