# AI 硬件生态关键机构与 KOL 识别框架

## 用途

用这份框架统一识别 AI 硬件生态中的关键媒体、分析师、工程师、渠道方、云厂商角色和企业客户，避免只按粉丝数做粗糙排序。

## 角色分类

重点识别以下角色：

1. **行业媒体 / 科技媒体**：具备新闻放大和议题设置能力
2. **分析师 / 研究机构**：影响市场预期和产业判断
3. **工程师 / 技术专家**：具备技术背书和部署判断能力
4. **渠道商 / 集成商 / ODM**：影响采购、交付和供给判断
5. **云厂商 / 企业客户**：影响真实采用和大规模部署
6. **模型厂商相关人物**：影响硬件选型、部署策略和生态路线

## 评分维度

| 维度 | 权重 | 说明 |
|------|------|------|
| 覆盖面 | 20% | 粉丝、订阅、媒体曝光、机构影响力 |
| 互动度 | 15% | 评论、转发、讨论深度 |
| 专业可信度 | 25% | 是否提供数据、配置、案例、原始依据 |
| 业务影响力 | 25% | 是否影响采购、部署、合作或产业判断 |
| 持续跟踪能力 | 10% | 是否长期关注该领域 |
| 增长潜力 | 5% | 是否处于快速上升期 |

## 分层标准

- **S 级**：跨平台高影响，可直接影响采购、舆论方向或产业判断
- **A 级**：垂类强影响，能显著左右目标圈层意见
- **B 级**：潜力作者或细分社区关键观察对象
- **C 级**：活跃用户、持续观察对象或友好机构

## 输出要求

每个对象至少给出：

- 平台
- 角色类型
- 影响力分
- 态度（支持 / 中立 / 质疑 / 负面）
- 代表内容
- 建议动作（关注 / 建联 / 邀测 / 解释 / 持续观察）

## 使用要求

交付名单前至少确认：

- 角色标签是否准确
- 影响力分是否与真实业务影响匹配
- 态度判断是否基于持续样本而非单条内容
- 建议动作是否与其所在环节相匹配
