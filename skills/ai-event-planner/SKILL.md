# Event Planner - AI技术活动策划助手

## Overview

为大模型运营部门提供AI技术交流活动的一站式策划支持，包括主题生成、嘉宾画像推荐、议程设计和文案输出。

## Workflow

### Step 1: 收集输入信息

收集以下关键信息：
- 活动规模（人数）
- 活动时间（日期和时长）
- AI技术领域（宏观大类：如大模型、小模型、智能体、多模态等）
- 具体技术名称（如PaddleOCR-VL1.5、文心一言等）

如果信息不完整，向用户询问补充。

### Step 2: 生成活动主题

基于输入的AI技术领域和具体技术名称，生成5个主题选项：

**生成维度**：
1. 当前热点趋势（结合最新AI发展）
2. 行业应用方向（医疗、金融、教育、制造、互联网等）
3. 技术深度（入门/进阶/专家）
4. 实践案例
5. 未来展望

**主题格式**：
- 主标题（简洁有力，吸引注意力）
- 副标题（说明技术领域和应用方向）
- 简要描述（2-3句话，说明内容重点）

示例：
- "大模型时代的技术变革：从理论到实践的多维探索"
- "PaddleOCR-VL1.5在医疗影像领域的创新应用"

### Step 3: 生成嘉宾画像

为每个活动方案推荐3-5位嘉宾画像，包含：

**画像要素**：
- 专业背景（学术界/工业界/创业界）
- 职位级别（高层/中层/专家）
- 领域专长（具体技术方向）
- 适合环节（主题演讲/圆桌讨论/Demo展示）

**画像模板**：
```
嘉宾画像1
- 职位级别：[CTO/技术总监/教授等]
- 专业背景：[学术界/工业界]
- 领域专长：[技术方向]
- 适合环节：[主题演讲/圆桌讨论/Demo展示]
- 演讲方向建议：[主题类型]
```

**多样性考虑**：
- 平衡学术界与工业界
- 覆盖不同职位级别
- 确保领域相关性和互补性

参考文件：[guest_profile_types.md](references/guest_profile_types.md)

### Step 4: 生成议程方案

生成两套完整议程：半天版本（4小时）和全天版本（8小时）

**议程必须包含**：
1. 签到环节（活动开始前30分钟）
2. 开场致辞（主办方，10分钟）
3. 主题演讲（主办方至少1场，其他嘉宾2-4场，每场30-40分钟）
4. 茶歇/休息环节（每1-2小时15分钟）
5. 圆桌讨论（30-40分钟，需配合主题生成讨论话题）
6. Demo展示（20-40分钟）
7. 自由交流环节（10-30分钟）
8. 总结收尾（10分钟）

**百度AI伙伴邀请建议**：
- 根据主题推荐应邀请的领域伙伴（硬件、医疗、金融、教育、制造、互联网等）
- 说明推荐理由和技术相关性

**全天活动额外要求**：
- 午餐休息（90分钟）
- 圆桌讨论2场（可覆盖不同话题）
- 主题演讲5场

参考模板：[agenda_template.md](assets/agenda_template.md)
参考指导：[agenda_design_guide.md](references/agenda_design_guide.md)

**议程格式**：
使用表格形式，包含：时间、环节、内容、时长

### Step 5: 生成宣传文案

生成以下两种文案：

#### 5.1 邀请函文案
生成正式和简洁两个版本，包含：
- 活动主题
- 时间地点
- 活动亮点
- 核心议题
- 嘉宾邀请
- 回复方式

参考模板：[invitation_template.md](assets/invitation_template.md)

#### 5.2 社群预热文案
生成适合不同平台的文案：
- 微信社群正式版
- 微信社群简洁版
- 微信朋友圈文案
- 钉钉/飞书社群文案

包含要素：
- 活动预告
- 核心看点
- 时间地点
- 报名方式
- 氛围营造

参考模板：[promotion_template.md](assets/promotion_template.md)

### Step 6: 生成完整Markdown文档

整合以上内容，生成结构化的Markdown文档：

**文档结构**：
```markdown
# [活动名称] 活动策划方案

## 一、活动概述
- 活动背景
- 活动目标
- 基本信息（时间、地点、规模）

## 二、主题方案
列出5个主题选项，每个包含：
- 主标题
- 副标题
- 描述
- 适用场景

## 三、嘉宾画像推荐
列出3-5位嘉宾画像

## 四、议程方案
### 4.1 半天活动议程（4小时）
[议程表格]

### 4.2 全天活动议程（8小时）
[议程表格]

### 4.3 百度AI伙伴邀请建议
[具体建议和理由]

## 五、宣传文案
### 5.1 邀请函文案
[正式版本]
[简洁版本]

### 5.2 社群预热文案
[各平台版本]

## 六、执行建议
- 时间节点提醒
- 备选方案
- 注意事项
```

## Quality Standards

### 主题质量
- 与输入的技术领域强相关
- 结合当前热点趋势
- 覆盖不同应用场景
- 标题简洁有力，描述清晰

### 嘉宾画像质量
- 画像具体可执行
- 多样性合理
- 与主题匹配度高
- 明确适合的环节

### 议程质量
- 时间分配合理
- 环节完整无遗漏
- 流程逻辑清晰
- 包含必要休息和互动

### 文案质量
- 语言专业得体
- 信息准确完整
- 符合平台调性
- 具有吸引力和感染力
