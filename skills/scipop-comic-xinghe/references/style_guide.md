# 连环画风格指南

## 风格种子（style_seed）设计原则

### 什么是 style_seed？

`style_seed` 是一个简短的风格描述字符串，用于确保所有 Panel 的视觉风格一致。它在 Phase 1 生成后，会在后续所有图像生成调用中原样追加。

### 设计原则

1. **简洁性**：控制在 15-30 个中文字符以内
2. **具体性**：描述具体的视觉特征而非抽象概念
3. **一致性**：风格元素之间不冲突

### 风格维度

| 维度 | 示例描述 |
|-----|---------|
| **画风** | 扁平插画、手绘素描、水彩风格、赛博朋克 |
| **色调** | 柔和粉彩、深邃太空蓝、霓虹光效、复古暖色 |
| **线条** | 清晰轮廓、粗犷笔触、细腻描边 |
| **质感** | 2D矢量艺术、纸质感、数字艺术 |
| **氛围** | 科学严谨、可爱治愈、神秘奇幻 |

### 结构模板

```
[主体] + [动作/状态] + [环境/背景] + [情绪/氛围]
```

### 示例

| 好的描述 | 问题描述 |
|---------|---------|
| 科学家在实验室中激动地盯着屏幕上的黑洞图像，背景是复杂的仪器设备 | 科学家看屏幕（太简单） |

## 图像提示（image_prompt）撰写

### 结构化模板

采用五段式结构，确保提示词完整、清晰、易于理解：

```
{主体特征}，{动作姿态}，{环境背景}，{细节元素}，{风格质量}
```

**推荐长度**：80-150 中文字符

---

### 五要素详解

#### 1. 主体特征（必填）

**目标**：清晰描述画面中心对象的外观、数量、位置。

| 描述维度 | 示例 |
|---------|------|
| **人物** | 一位戴眼镜的女科学家，穿着白色实验服，棕色短发 |
| **动物/角色** | 一个可爱的Y字形抗体角色，戴着蓝色小警帽，圆眼睛 |
| **物体** | 一个巨大的橙色黑洞光环，中心是深邃的黑色 |
| **抽象概念** | 五个彩色的小圆球，分别标注"金木水火土" |

**撰写技巧**：
- 明确数量（一个、两个、一群）
- 描述外观特征（颜色、形状、大小、服饰）
- 指定位置（居中、左侧、前景）
- 避免模糊描述（"一个人" → "一位穿白大褂的中年男科学家"）

#### 2. 动作姿态（必填）

**目标**：描述主体正在做什么，赋予画面动态感和叙事性。

| 场景类型 | 好的描述 | 问题描述 |
|---------|---------|---------|
| 人物动作 | 正激动地用手指着屏幕上的数据，嘴巴微张 | 看屏幕（无情绪） |
| 角色互动 | 正在追逐一个滑稽的球形病毒角色，伸出双臂 | 追病毒（无姿态） |
| 物体状态 | 正在高速旋转，周围产生明亮的吸积盘 | 旋转（无细节） |
| 静态展示 | 静静悬浮在深蓝色的太空中，散发着柔和的光芒 | 在太空中（无状态） |

**撰写技巧**：
- 使用具体动词（追逐、指向、旋转、漂浮）
- 添加情绪或状态（激动地、轻松地、快速地）
- 描述姿态细节（伸出双臂、微微前倾、仰头看天）

#### 3. 环境背景（必填）

**目标**：交代场景设置，为主体提供合理的空间背景。

| 场景类型 | 背景描述示例 |
|---------|-------------|
| **实验室** | 现代实验室背景，有显微镜、电脑、试管架，墙上贴着公式图表 |
| **太空** | 深邃的太空背景，点缀着闪烁的星星和远处的星云 |
| **人体内部** | 红色的血管内壁背景，周围有流动的血细胞 |
| **自然场景** | 郁郁葱葱的热带雨林，阳光透过树叶洒下斑驳光影 |
| **抽象场景** | 渐变的蓝紫色科技感背景，有流动的光线条纹 |

**撰写技巧**：
- 描述背景元素（有哪些物体、什么颜色、如何分布）
- 注意前景与背景的层次关系
- 背景不要过于复杂，避免喧宾夺主

#### 4. 细节元素（可选）

**目标**：添加标志性细节，增强画面的信息量和科普准确性。

| 细节类型 | 示例 |
|---------|------|
| **屏幕内容** | 屏幕上显示橙红色的黑洞光环图像 |
| **文字标注** | 黑板上有手写的"E=mc²"公式 |
| **标志性物品** | 手中拿着一个DNA双螺旋模型 |
| **环境细节** | 桌上放着一杯冒着热气的咖啡 |

**撰写技巧**：
- 细节应服务于科普主题
- 避免过多细节导致画面杂乱
- 细节描述要具体（"屏幕显示数据图表" → "屏幕上显示柱状数据图，标注'温度变化'"）

#### 5. 风格质量（必填）

**目标**：确保画面风格统一、质量稳定。

**基础组合**：
```
{style_seed}，高质量，连环画
```

**质量增强词**：
- 画面质量：高质量、清晰细节、精美绘制
- 构图：居中构图、三分法构图、对称构图
- 光影：柔和光照、明暗对比、逆光效果
- 色彩：鲜艳色彩、柔和色调、科学配色

### 常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|-----|------|---------|
| **主体不清晰** | 未明确描述主体的位置和大小 | 添加"居中"、"特写"、"全身"等位置词 |
| **风格不统一** | style_seed 未追加或描述冲突 | 确保每个 prompt 末尾追加 style_seed |
| **画面杂乱** | 细节元素过多 | 精简背景描述，保留2-3个关键元素 |
| **角色变形** | 人体结构描述缺失 | 添加"人体结构正确"、"比例协调"约束 |
| **色彩偏差** | 未指定色调 | 明确描述"主色调为XX"、"XX配色" |
| **缺少科普感** | 过于艺术化 | 添加科学元素（标签、公式、数据图表） |
| **审核拒绝** | 敏感词汇 | 替换为中性描述，避免品牌、暴力词汇 |

### 中文 Prompt 撰写技巧

Ernie-image-turbo 对中文 Prompt 支持良好，以下技巧可提升效果：

1. **直接使用中文术语**：
   - ✅ "黑洞吸积盘"、"DNA双螺旋"
   - ❌ "black hole accretion disk"

2. **善用量词增强具体感**：
   - ✅ "一位科学家"、"三只小蚂蚁"
   - ❌ "科学家"、"蚂蚁"

3. **使用中文习惯的描述顺序**：
   - ✅ "穿着白大褂的中年男科学家"（修饰语在前）
   - ❌ "中年男性科学家，穿白大褂"（拆分描述）

4. **描述中国元素更自然**：
   - ✅ "中国古代天文学家，穿着汉服，手持浑天仪"
   - 适合本土化科普内容

### 禁忌词避免

避免使用可能导致审核拒绝的词汇：

| 类别 | 禁忌词 | 替代方案 |
|-----|-------|---------|
| 暴力血腥 | 打架、杀、血 | 追逐、对抗、红色能量 |
| 政治敏感 | 政治人物、国旗 | 概括性描述、通用形象 |
| 商标品牌 | iPhone、特斯拉 | 智能手机、电动汽车 |
| 敏感内容 | 裸露、药物 | 穿着得体、药品（中性描述） |

## Panel 数量决策

### 4格适用场景

- 内容结构清晰，可划分为4个阶段
- 信息量适中，每格承载一个核心概念
- 适合故事性强、有明确起承转合的内容

### 5格适用场景

- 内容有5个层次或要点
- 需要一个过渡或高潮格
- 适合复杂流程或多层次解释

### 6格适用场景

- 内容丰富，知识点多
- 需要展示完整过程（如实验步骤）
- 适合系统性强的科普内容

## 全局合成注意事项

> ⚠️ **重要变更**: 全局大图不再使用模型生成，而是**直接拼接用户确认的单 Panel 图像**。这样可以确保全局大图与用户确认的单 Panel 完全一致。

### 布局选择

| Panel数 | 推荐布局 | 视觉效果 |
|--------|---------|---------|
| 4 | 2×2 | 正方形，均衡 |
| 5 | 2×3 | 横向阅读，留白收尾 |
| 6 | 2×3 | 标准竖向长图 |

### 阅读顺序

中文连环画采用**从左到右、从上到下**的阅读顺序：
```
┌─────┬─────┐
│  1  │  2  │
├─────┼─────┤
│  3  │  4  │
├─────┼─────┤
│  5  │  6  │
└─────┴─────┘
```

### 拼接实现方式

使用 Pillow 库直接拼接用户确认的单 Panel 图像：

```python
from PIL import Image

def create_comic_grid(panel_images, rows, cols, border_width=4):
    """将多个 Panel 图像拼接成连环画"""
    panel_width, panel_height = panel_images[0].size

    # 计算画布尺寸
    total_width = cols * panel_width + (cols + 1) * border_width
    total_height = rows * panel_height + (rows + 1) * border_width

    # 创建画布
    canvas = Image.new('RGB', (total_width, total_height), (255, 255, 255))

    # 按阅读顺序粘贴
    for idx, panel in enumerate(panel_images):
        if idx >= rows * cols:
            break
        row, col = idx // cols, idx % cols
        x = border_width + col * (panel_width + border_width)
        y = border_width + row * (panel_height + border_width)
        canvas.paste(panel, (x, y))

    return canvas
```

### 拼接优势

| 优势 | 说明 |
|-----|------|
| **风格一致** | 每个格子的风格与用户确认的单 Panel 完全相同 |
| **内容一致** | 每个格子的内容与用户确认的单 Panel 完全相同 |
| **旁白一致** | 每个格子的旁白位置、样式与单 Panel 完全相同 |
| **无模型漂移** | 避免模型重新生成导致的风格和内容偏差 |
