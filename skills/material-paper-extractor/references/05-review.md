# 审查提示词

## 输出路径
**必须保存至：** `{output_base}/05_review/{id}_review.md`

## 目的

对提取结果进行批判性审查，以原始论文文字为基准。采用最大限度的怀疑态度——扮演魔鬼代言人，在错误传播之前将其捕获。

审查有两个同等重要的维度：**准确性**（已提取的数据是否正确）和**完整性**（原文中的数据是否被遗漏）。

## 输入

你将收到：
1. 原始合并文本（来自 Step 2）
2. 提取提示词（Step 3 的 system + user 提示词）
3. 验证后的提取结果 JSON（来自 Step 4）

## 审查流程

对提取结果中的每个 item，逐一对照原文进行以下六个维度的审查。每个维度都包含准确性校验和完整性校验。

---

### 1. 成分准确性

**准确性校验：**
- 将每个成分值与原文对照，确认数值精确匹配
- 检查元素总和是否约为 100%（±1 容差）
- 验证名义成分（Nominal）与实测成分（Measured）的区分是否正确
- 检查 Composition_Type 是否为 "wt%" 或 "at%"
- 验证原文指定时是否填写了 Measurement_Method
- 检查 Note 字段是否捕获了特殊标注（范围、上限、杂质说明等）
- 标记任何捏造或幻觉产生的成分数据

**完整性校验：**
- 原文中是否提到了微量元素、杂质含量或合金元素范围，但提取中遗漏？
- 商业牌号（如 Inconel 718、316L SS）虽然可以不填 Elements_Normalized，但 Note 字段是否说明了原因？
- 如果原文有多个成分变体（如名义成分和实测成分同时存在），是否都被提取？

---

### 2. 工艺忠实度与参数完整性

**准确性校验：**
- 将 Key_Params 中的每个参数值与原文逐一对照
- 验证温度单位正确（°C → K 转换是否正确）
- **热处理范围值不得取平均**（例如 "1000-1100°C" 必须保留为范围字符串，不能存为 1050）
- 多段处理必须使用阶段后缀（如 `_Stage1_K`），不得取平均
- 验证 Process_Text.original 是否完整详细地保留了原文所有工艺描述细节（未简化、未删减）
- 验证 Process_Text.simplified 是否保留了关键信息同时去除了冗余
- 验证原文提到具体设备/型号时是否填写了 Equipment 字段
- 检查 Cooling_Medium（如水淬、空冷、炉冷）是否提取
- 检查 Protective_Atmosphere（如氩气、真空、氮气）是否提取

**完整性校验——参数遗漏检查（关键）：**
这一步是审查中最容易遗漏的维度。你的任务是：**扫描原文中所有出现过的量化工艺参数，逐一检查是否被提取到 Key_Params 中。**

具体做法：
1. 在原文中标记出**每一个带数值和单位的工艺参数**（包括但不限于激光功率、扫描速度、层厚、扫描间距、基板温度、粉末粒径、能量密度、热处理温度/时间/冷却方式、保温时间、变形参数等）
2. 对照提取结果中的 Key_Params，逐一确认每个参数是否已被提取
3. 检查提取提示词中定义的三层参数体系是否被充分执行：
   - **Tier 1（核心参数）**：`Laser_Power_W`、`Scanning_Speed_mm_s`、`Layer_Thickness_um`、`Hatch_Spacing_um`、`Build_Plate_Temperature_K`、`Solution_Temperature_K`、`Solution_Time_h`、`Aging_Temperature_K`、`Aging_Time_h`、`Annealing_Temperature_K`、`Annealing_Time_h`、`Cooling_Medium`、`Protective_Atmosphere` —— 原文中有提及的必须提取
   - **Tier 2（粉末/变形参数）**：`Powder_Size_avg_um`、`Volumetric_Energy_Density_J_mm3`、`Oxygen_Content_ppm`、`Rolling_Reduction_pct`、`Forging_Temperature_K`、`Extrusion_Ratio` —— 原文中有提及的必须提取
   - **Tier 3（动态兜底参数）**：原文中出现的任何其他量化工艺参数（如扫描策略、预热温度、送粉速率、保护气流速等），是否都用描述性名称+单位后缀提取到了 Key_Params 中？
4. 在审查报告中，以表格形式列出"原文参数 → 是否已提取 → 差异说明"

---

### 3. 微观结构验证与数据完整性

**准确性校验：**
- 将 Microstructure_Text 中的描述与原文对照，确认未篡改、未简化过度
- **Main_Phase 禁令**：在 Ni/Co/Fe 基高温合金中，Laves 相、碳化物（MC、M2C、M6C）绝对禁止被标识为 Main_Phase，应放入 Precipitates[] 数组
- **Relative_Density_pct**：如果被推断为 100% 或在原文中未提供时被猜测，标记为问题。原文未提供时必须为 null
- Precipitates[] 数组中的 Phase_Type 是否与原文描述一致
- Gradient_Material / Gradient_Group_ID 是否设置正确

**完整性校验——定量数据遗漏检查（关键）：**
扫描原文中所有关于微观结构的**定量描述**，逐一检查是否被提取：

1. **Tier 1（宏观/晶粒级）**：
   - `Porosity_pct`：原文是否给出了孔隙率数据？如果有，是否提取？
   - `Relative_Density_pct`：原文是否给出了相对密度数据？如果有，是否提取？
   - `Main_Phase`：原文是否明确指出了主相？
   - `Grain_Size_avg_um`：原文是否给出了晶粒尺寸数据（包括 B 面、S 面、不同方向）？如果有，是否提取？

2. **Tier 2（析出相/相组成级）**：
   - `Precipitate_Size_avg_nm`：原文是否给出了析出相尺寸？如果有，是否提取？
   - 析出相体积分数（如 `Laves_Volume_Fraction_pct`、`Carbide_Volume_Fraction_pct`）：原文是否给出了？如果有，是否提取？
   - Precipitates[] 数组：原文描述的所有析出相/第二相是否都被列入？

3. **Tier 3（高级定量特征）**：
   - Advanced_Quantitative_Features 中是否捕获了原文提到的其他定量微结构数据？
   - 例如：位错密度、亚晶尺寸、织构强度、晶粒长径比、colony 尺寸、lath 宽度、相比例等
   - 原文中是否有从图表中可以读出的定量微结构信息未被提取？

4. 在审查报告中，以列表形式说明"原文提到的微结构定量数据 → 是否已提取 → 遗漏原因"。

---

### 4. 性能数据有效性与动态兜底

**准确性校验：**
- 将 Properties_Info 中的每个属性值与原文逐一对照
- 测试温度使用开尔文（室温 = 298.15 K）
- **温度隔离**：Test_Temperature_K 仅用于测试环境温度，绝不用于制造/热处理温度
- 检查 YS ≤ UTS（物理约束）
- 检查延伸率 ≤ 100%
- 验证压缩试验是否带有 `_Compressive` 后缀
- **三分类延伸率**：验证 Elongation_Total、Elongation_Uniform、Elongation_At_Fracture 的区分是否正确
- **拉伸试验条件**：Strain_Rate_s1 和 Tensile_Speed_mm_min 是否已提取
- **硬度试验条件**：Hardness_Load 和 Hardness_Dwell_Time_s 是否已提取
- **Data_Source**：文字表格为 "text"，图表为 "image"
- 标记原文中实际不存在的性能数据（幻觉数据）

**完整性校验——动态兜底检查（最关键）：**
性能数据是最容易遗漏的维度，因为论文中的性能类型远不止标准列表中的那些。你的任务是：**扫描原文中所有出现的性能数据（包括表格、正文叙述、图表），逐一检查是否被提取到 Properties_Info 中。**

具体做法：
1. 在原文中标记出**每一个带数值和单位的性能数据点**，包括但不限于：
   - Tier 1（核心力学性能）：屈服强度、极限拉伸强度、弹性模量、延伸率（三种）、硬度
   - Tier 2（扩展性能）：残余应力、冲击韧性、疲劳强度、断裂韧性、热导率、比热、热膨胀系数、电导率、电阻率、腐蚀速率、磨损率、摩擦系数
   - **Tier 3（动态兜底——最重要的检查项）**：任何未被上述列表覆盖的性能数据，例如：
     - 疲劳裂纹扩展阈值（ΔK₀ / Fatigue_Crack_Growth_Threshold）
     - Paris 指数 / Paris 常数（m / C）
     - 疲劳寿命（Nf）
     - 蠕变性能（蠕变速率、持久强度）
     - 超弹性/伪弹性应变
     - 阻尼性能
     - 磁性能（矫顽力、磁导率）
     - 生物相容性指标
     - 任何其他用自定义 Property_Name + Unit 描述的性能
2. 对照提取结果中的 Properties_Info，逐一确认每个数据点是否已被提取
3. 特别注意以下容易遗漏的场景：
   - 散落在正文叙述中的性能值（不在表格中）
   - 以范围形式给出的性能（如 "σY of 1029-1161 MPa"）
   - 以对比形式出现的性能（如 "比传统方法高出 30%"）
   - 从图表中可以读取的数据点（虽然不应盲猜曲线，但明确标注的数据点应提取，Data_Source 标为 "image"）
   - 论文摘要或结论中提到的关键性能值，但详细数据在正文中
4. 在审查报告中，以表格形式列出"原文性能数据 → 是否已提取 → Property_Name → 差异说明"

---

### 5. 测试条件与元数据完整性

这个维度检查每个性能数据点附带的测试条件是否完整：

- **Test_Temperature_K**：是否填写？室温测试是否标为 298.15K？高温/低温测试是否准确？
- **Strain_Rate_s1**：拉伸试验是否提取了应变率？
- **Tensile_Speed_mm_min**：拉伸试验是否提取了拉伸速度/位移速率？
- **Hardness_Load** 和 **Hardness_Dwell_Time_s**：硬度试验是否提取了载荷和保载时间？
- **Test_Specimen**：原文是否描述了试样尺寸或适用标准（如 ASTM E8、GB/T 228）？如果有，是否提取？
- **Test_Condition**：是否描述了测试方向（如平行/垂直于建造方向）、加载模式、环境条件等？
- **Data_Source**：每个属性是否正确标注为 "text"（来自文字/表格）或 "image"（来自图表）？

---

### 6. 全局完整性检查

- 识别原文中提及但完全未被提取的数据
- 验证 Role 标注（Target vs Reference）是否正确——论文自身实验数据应为 Target，引用其他文献的数据应为 Reference
- 检查隐藏在表格脚注、补充材料说明、图片标注中的数据
- 检查 item 分解是否正确：原文中不同的成分/工艺/测试方向组合是否被正确拆分为独立 item？
- 检查 Sample_ID 命名是否清晰且与原文样品编号对应

---

## 核心原则

1. **准确性 + 完整性并重**：不仅要检查已提取的数据对不对，还要检查原文的数据是否都被提取了。
2. **动态兜底是关键**：工艺参数和性能数据都有 Tier 3 的动态兜底机制，审查时必须主动扫描原文中所有带数值的参数和性能，不能只检查预定义列表。
3. **采用最大限度的怀疑态度**：如果有任何可疑之处，一律标记。宁可过度报告，也不遗漏真正的问题。

## 约束

- 报告必须使用中文撰写
- 所有判断必须追溯到原文，引用原文语句作为证据
- 专业、客观、建设性的语调
- 直接输出报告——不要客套话或前言
