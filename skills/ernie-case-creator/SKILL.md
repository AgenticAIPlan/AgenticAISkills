# ERNIE Case Creator - 文心案例创作技能

## 技能简介

本技能用于自动化完成文心大模型应用案例的内容创作工作。基于用户提供的案例材料（支持Word、PPT、PDF等多种格式），按照标准化SOP流程，生成符合规范的案例文字内容，并最终输出一页标准化的案例PPT。

## 核心功能

### 1. 案例材料解析
- 支持多种格式：Word(.docx)、PPT(.pptx)、PDF(.pdf)
- 支持多种内容类型：文字、图片、表格
- 自动提取关键信息：项目背景、行业痛点、技术方案、业务效果

### 2. 案例文字生成
- 遵循"问题-场景-技术-方案-价值"五段式逻辑
- 字数范围：250-350字
- 符合标准化案例说明框架

### 3. 案例PPT生成
- 基于提供的PPT模板生成一页案例PPT
- 自动填充案例文字内容
- 支持图片展示选择

## 使用时机

当出现以下场景时使用本技能：

1. 用户需要整理文心大模型应用案例
2. 用户提供了案例材料（Word、PPT、PDF等），需要生成标准化案例说明
3. 用户需要基于案例材料生成案例PPT
4. 用户提到"案例整理"、"案例写作"、"文心案例"、"ERNIE案例"等关键词

## 工作流程

### 第一阶段：案例材料整理

**目标**：从用户提供的材料中提取关键信息

**步骤**：

1. **材料格式识别**
   - 检测材料格式（Word/PPT/PDF）
   - 根据格式选择对应的解析方法

2. **关键信息提取**
   - 项目名称和行业背景
   - 使用的文心模型型号（重点：ERNIE-4.5-21B、ERNIE-4.5-28B-VL、ERNIE-5.0）
   - 行业痛点与业务背景
   - 技术方案与核心功能
   - 量化效果与业务价值

3. **信息结构化**
   - 将提取的信息填入标准化模板
   - 标注信息完整度
   - 识别缺失信息并提示用户补充

**输出**：结构化的案例信息表

### 第二阶段：案例文字生成

**目标**：基于提取的信息生成标准化的案例说明文字

**框架**：遵循"问题-场景-技术-方案-价值"五段式

**各部分撰写要点**：

#### 1. 痛点/背景（1-2句话）
- 明确指出具体行业或用户的痛点问题
- 说明痛点带来的实际影响（效率低、成本高、风险大等）
- 避免笼统表述，简洁有力

#### 2. 应用环境（1句话）
- 明确产品适用的行业领域
- 说明具体的使用场景
- 界定目标用户群体

#### 3. 技术支撑（1-2句话）
- 明确使用的文心模型型号
- 强调该模型能力对解决行业痛点的适配性
- 突出技术差异化优势

#### 4. 解决方案（2-3句话）
- 阐述产品的核心功能模块
- 说明解决问题的具体路径和工作流程
- 强调与痛点对应的针对性解决方案

#### 5. 价值成果（2-3句话）
- 量化业务提升数据（效率提升X%、成本降低Y%）
- 说明实际落地应用规模（部署X套、服务Y用户、覆盖Z场景）
- 展示长期价值

**字数要求**：250-350字

**输出**：标准化的案例说明文字

### 第三阶段：案例PPT生成

**目标**：基于PPT模板生成一页案例PPT

**PPT模板结构**（固定模板）：

模板采用固定布局，包含以下固定要素：

1. **顶部标题区域**
   - 案例名称（主标题）
   - 右上角公司logo位置

2. **左侧主体区域**
   - 蓝色标识区域
   - 案例文字说明内容（250-350字）

3. **右侧数据展示区**
   - "文心大模型应用"绿色标签
   - 效果数据展示（数字+百分比形式）
   - 量化成果指标（通常2-3个关键指标）

4. **设计风格**
   - 主色调：蓝色（#1E88E5）
   - 辅助色：绿色（#4CAF50）
   - 整体简洁专业

**生成步骤**：

1. **内容提取**
   - 从案例文字中提取量化效果指标
   - 识别关键数据（百分比、数字、时间等）
   - 确定展示优先级

2. **布局填充**
   - 标题区域：填入案例名称
   - 左侧区域：填入完整案例说明文字
   - 右侧区域：
     * 顶部添加"文心大模型应用"绿色标签
     * 下方展示2-3个核心效果指标
     * 每个指标包含：大号数字 + 说明文字

3. **格式优化**
   - 确保文字大小合适，易于阅读
   - 指标数字突出显示
   - 整体布局平衡美观

**效果指标示例**：
- 效率提升：95%
- 处理时间：10分钟
- 项目规模：300个

**输出**：一页标准化的案例PPT（.pptx格式）

## 重要注意事项

### 材料处理

1. **材料来源**：材料可能是Word、PPT、PDF等格式，内容可能包含文字、图片、表格
2. **材料完整性**：优先检查材料完整性，如缺失关键信息需提示用户补充
3. **材料准确性**：确保提取的信息准确无误，特别是文心模型型号

### 内容生成

1. **模型型号**：重点关注ERNIE-4.5-21B、ERNIE-4.5-28B-VL、ERNIE-5.0
2. **字数控制**：严格控制在250-350字范围内
3. **逻辑结构**：必须遵循五段式框架
4. **数据量化**：尽可能使用量化数据体现效果

### PPT生成

1. **模板使用**：必须基于用户提供的PPT模板生成
2. **内容适配**：确保内容与模板布局匹配
3. **视觉优化**：注意文字排版和图片选择

## 技能限制

1. **材料格式限制**：目前仅支持Word、PPT、PDF格式
2. **语言限制**：目前仅支持中文案例
3. **模板依赖**：生成PPT需要用户提供模板
4. **图片处理**：如材料包含图片，需要额外处理步骤
