# 文心案例说明框架详细说明

## 一、框架概述

### 五段式逻辑结构

文心案例说明采用"问题-场景-技术-方案-价值"五段式逻辑结构：

```
问题 → 场景 → 技术 → 方案 → 价值
```

这种结构能够：
1. 快速抓住读者注意力
2. 清晰展示项目逻辑
3. 突出文心模型价值
4. 体现业务实际效果

## 二、各组成部分详细说明

### 1. 痛点/背景（1-2句话）

**核心目标**：让读者理解产品存在的必要性

**撰写要点**：
- 明确指出具体行业或用户的痛点问题
- 说明痛点带来的实际影响（效率低、成本高、风险大等）
- 或明确项目发起的业务背景，明确需求来源
- 避免笼统表述，要具体、可感知

**示例**：
- 差：制造业存在效率问题
- 好：汽车制造企业的工艺文件管理依赖人工查阅，单次查询耗时平均20分钟，且易出错

**技巧**：
- 用数据说话（如：耗时XX分钟、准确率XX%）
- 描述真实困境（如：依赖老师傅经验、手工操作繁琐）

### 2. 应用环境（1句话）

**核心目标**：清晰界定产品的适用范围

**撰写要点**：
- 明确产品适用的行业领域（化工、电力、医疗等）
- 说明具体的使用场景
- 界定目标用户群体

**示例**：
- 在汽车制造领域，用于新车投产的风险评估场景，服务于生产决策团队

**技巧**：
- 行业+场景+用户三位一体
- 避免过于宽泛的描述

### 3. 技术支撑（1-2句话）

**核心目标**：体现产品所用模型的型号和差异化优势

**撰写要点**：
- 明确产品/方案使用到的文心模型
- 强调该模型能力对于解决前文所述行业痛点问题的适配性
- 突出技术优势（如：参数规模、推理能力、多模态能力等）

**模型型号重点**：
- ERNIE-4.5-21B：适合需要定制化微调的场景
- ERNIE-4.5-28B-VL：适合需要多模态能力的场景
- ERNIE-5.0：适合需要最强通用能力的场景

**示例**：
- 基于ERNIE-4.5-21B进行SFT微调，充分利用其强大的语言理解和生成能力

### 4. 解决方案（2-3句话）

**核心目标**：详细说明产品如何解决前述痛点问题

**撰写要点**：
- 阐述产品的核心功能模块（避免罗列功能清单）
- 说明解决问题的具体路径和工作流程
- 强调与痛点对应的针对性解决方案
- 突出创新点和差异化优势

**示例**：
- 构建"自主规划-工具执行-结果反思"的智能体闭环
- 实现从问题识别到解决方案生成的端到端流程

**技巧**：
- 按逻辑顺序描述解决流程
- 突出AI技术的应用点
- 避免技术细节堆砌

### 5. 价值成果（2-3句话）

**核心目标**：用数据证明产品的实际价值

**撰写要点**：
- 量化业务提升数据（效率提升X%、成本降低Y%）
- 说明实际落地应用规模（部署X套、服务Y用户、覆盖Z场景）
- 展示长期价值（年均预防X问题、提升Y指标）

**示例**：
- 单项目分析耗时从2-3小时降至10分钟内，效率提升95%
- 已完成300个项目分析，报告自动生成率100%、准确率超90%

**技巧**：
- 用对比数据展示提升效果
- 说明落地规模和持续性
- 突出可复制性和扩展性

## 三、字数控制与风格要求

### 总字数：250-350字

**各段字数建议**：
- 痛点/背景：30-50字
- 应用环境：15-25字
- 技术支撑：30-50字
- 解决方案：60-100字
- 价值成果：60-100字

### 语言风格

1. **专业严谨**
   - 使用行业专业术语
   - 避免口语化表达

2. **简洁有力**
   - 避免冗长句子
   - 每句话表达一个核心意思

3. **数据导向**
   - 尽可能使用量化数据
   - 数据要具体、可信

4. **逻辑清晰**
   - 各段之间要有逻辑衔接
   - 整体要有递进关系

## 四、常见问题与规避

### 1. 痛点描述不明确

**问题**：痛点描述过于笼统，如"效率低、成本高"

**解决方法**：
- 具体化：什么环节效率低？成本高在哪里？
- 数据化：效率低了多少？成本高了多少钱？

### 2. 技术方案不聚焦

**问题**：罗列多个功能，没有突出核心

**解决方法**：
- 聚焦解决痛点的核心功能
- 用流程描述代替功能罗列

### 3. 效果描述不具体

**问题**：用"显著提升"、"大幅改善"等模糊描述

**解决方法**：
- 提供具体数据对比
- 说明计算方法和依据

### 4. 逻辑衔接不自然

**问题**：各段之间过渡生硬

**解决方法**：
- 使用过渡词语：因此、基于此、由此
- 建立因果联系：因为...所以...

## 五、优秀案例示例

### 案例：国网四川电力项目

**字数**：320字

**内容**：
> 国网四川电力面对35千伏及以上项目投资问效分析效率低下的痛点，传统方法单项目分析耗时2-3小时，严重制约了投资决策的时效性。
>
> 在电力投资决策场景中，服务于电网规划部门的项目评估团队。
>
> 项目基于文心大模型与Agentic智能体架构，创新构建"自主规划-工具执行-结果反思"的闭环分析系统。
>
> 该系统通过智能体自动识别项目关键要素，调度专业分析工具进行模拟计算，并对结果进行自我评估与优化，实现了端到端的智能化分析。
>
> 实施后，单项目分析耗时从2-3小时骤降至10分钟内，效率提升95%以上。已成功完成300个项目分析，报告自动生成率100%、准确率超90%，为电力行业投资决策智能化树立了新标杆。

**分析**：
- 痛点具体：2-3小时降至10分钟内
- 场景明确：电力投资决策场景
- 技术突出：文心大模型+Agentic智能体
- 方案清晰：自主规划-工具执行-结果反思
- 价值量化：效率提升95%，300个项目，准确率90%

## 六、检查清单

在完成案例说明后，请检查：

- [ ] 总字数是否在250-350字之间
- [ ] 是否包含五段式结构
- [ ] 痛点描述是否具体、可感知
- [ ] 技术方案是否突出文心模型优势
- [ ] 效果描述是否有量化数据
- [ ] 各段之间是否有逻辑衔接
- [ ] 语言是否专业、简洁
- [ ] 是否有错别字或语法错误

## 七、适用场景

### 适合本框架的场景

1. **产品推广材料**：用于展示产品价值的案例说明
2. **客户汇报材料**：向客户展示成功案例
3. **内部总结材料**：总结项目成果和价值
4. **媒体报道材料**：向媒体介绍项目亮点

### 不适合的场景

1. **技术详细说明**：需要详细技术架构
2. **操作手册**：需要具体操作步骤
3. **学术论文**：需要理论分析和参考文献
4. **项目立项报告**：需要详细项目规划

## 八、模板变量

在使用案例说明Prompt时，可以替换以下变量：

- `{项目名称}`：案例项目的名称
- `{行业领域}`：项目所属行业
- `{文心模型}`：使用的文心模型型号
- `{核心功能}`：项目核心功能描述
- `{效果数据}`：量化效果数据
- `{落地规模}`：实际落地应用情况
