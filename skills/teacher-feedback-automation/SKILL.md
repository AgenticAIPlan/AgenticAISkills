# 定时自动解决教师版用户反馈（邮件回复 + 权限开通）

本技能专门用于自动处理AI Studio教师版用户反馈，实现从邮件接收到权限开通的完整自动化流程。教师从发邮件申请权限到最终开通权限，全程无需人工介入。

## 🎯 目标
让 Agent 每天自动完成：
1. **回复教师邮件**：
   - 权限申请类 → 回复问卷链接
   - 业务咨询类 → 检索知识库回答
   - 算力申请类 → 按学校类型处理
2. **开通教师权限**：
   - 根据问卷为新教师开通权限
   - 自动登录后台系统操作

## 📍 常见使用场景
- 每天收到权限申请邮件，内容高度重复
- 教师填完问卷后等待开通，需要有人每天去后台操作
- 教师咨询使用问题，答案都在文档里但要人工查找回复

## 🔄 整体流程

### 每日自动执行流程
1. **读取昨天的未读邮件** → 分类（权限申请/开课咨询/算力申请）→ 分类回复
2. **下载问卷文件** → 识别新教师 → 自动登录后台开通权限 → 更新开通记录
3. **发送执行汇总到如流会话**
4. **人工复检，确认无遗漏**

### 设置定时任务
- **邮件回复任务**：每天上午10:15、10:25执行
- **权限开通任务**：每天凌晨4点执行
- **结果汇总任务**：每天上午10:12执行

1. 使用Python读取CSV文件：
   ```python
   import csv
   with open('/tmp/playwright-artifacts-{ID}/{文件名}', 'r', encoding='gbk') as f:
       reader = csv.reader(f)
       rows = list(reader)
   ```

2. **从表格最后一行逐行往前检查**

3. **找到新增的所有的我没有开通过权限的老师**

4. **遇到我开通过权限的老师的时候就停止**

## ⚠️ 踩过的坑

### 1. 表单填写必须用act逐字段操作
- **错误做法**：用JavaScript批量填看着能填进去，但不触发校验，保存报错。
- **正确做法**：使用browser工具的act操作，逐个字段填写，确保触发页面验证机制。

### 2. 如流龙虾会"忘记"自己的skill
- **现象**：如流龙虾或dodo是最适合接入邮件、后台等公司业务的，但有点笨。上次做对的流程，下次可能又做错。
- **解决方法**：计划尝试增加一个自研的记忆插件，固定正确动作。目前先靠在Prompt里把注意事项写全来缓解。

### 3. 每2天人工复检
- **原因**：前期，Agent有时表现不稳定或遇到特殊情况，定期检查能及时发现。
- **方法**：检查记录文件的开通数量与后台是否一致，问卷新增数据是否都已处理，邮件有无遗漏回复。
