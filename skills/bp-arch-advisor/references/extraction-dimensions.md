# BP 信息提取维度定义

从创业项目计划书中提取以下 8 个维度的关键信息。

## 提取维度表

| 维度 | 提取字段 |
|------|---------|
| 项目概述 | 项目名称、所属行业、目标市场、核心价值主张 |
| 产品形态 | 产品类型（SaaS / App / 硬件 / 平台）、核心功能列表、用户交互方式 |
| 技术架构 | 技术栈、系统架构描述、核心模块划分、数据流转路径 |
| AI 能力现状 | 当前是否使用 AI、使用了哪些模型或服务、AI 在产品中的角色定位 |
| 数据资产 | 核心数据类型、数据规模预估、数据获取方式 |
| 团队技术背景 | 技术团队规模、核心技术能力方向 |
| 商业模式 | 盈利模式、单位经济模型（如有） |
| 竞争格局 | 主要竞品名称、技术差异化点 |

## 提取规则

1. 逐维度在 BP 文档中查找对应信息，精确引用原文关键表述。
2. 如某维度在 BP 中完全未提及，对应字段填写 `[BP未提及 - 建议补充]`。
3. 如某维度仅有部分信息，提取已有内容并在缺失字段后标注 `[部分缺失]`。
4. 技术架构维度为后续分析的核心输入，应尽可能完整提取。
5. 对于 AI 能力现状为空的项目，后续阶段应重点设计从零开始的文心接入方案。
