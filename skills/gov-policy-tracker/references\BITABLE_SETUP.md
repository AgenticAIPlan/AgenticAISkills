# 飞书多维表格配置指南

## 概述

政策追踪系统现在支持将抓取的政策数据直接写入飞书多维表格（Bitable），便于您进行更灵活的数据管理和分析。

## 配置步骤

### 1. 创建飞书应用

1. 访问 [飞书开放平台](https://open.feishu.cn/)
2. 登录后点击"创建企业自建应用"
3. 填写应用名称（如"政策追踪系统"）和应用描述
4. 记录 **App ID** 和 **App Secret**

### 2. 开启多维表格权限

在应用后台：
1. 进入"权限管理"
2. 添加以下权限：
   - `bitable:app` - 读取多维表格应用信息
   - `bitable:record` - 读取和修改记录
   - `bitable:field` - 读取字段信息

### 3. 创建多维表格

1. 在飞书中新建一个多维表格
2. 创建数据表，字段设置如下：

| 字段名称 | 字段类型 | 说明 |
|---------|---------|------|
| 抓取日期 | 日期 | 系统自动填充 |
| 政策发布日期 | 日期 | 政策原始发布日期 |
| 来源部门 | 文本 | 如：工信部、科技部等 |
| 地区 | 文本/单选 | 全国或地方名称 |
| 政策标题 | 文本 | 完整政策标题 |
| 政策摘要 | 文本 | 自动生成的摘要 |
| 原文链接 | 超链接 | 点击跳转原文 |
| 文件原名 | 文本 | 从URL解析的文件名 |
| 政策类型 | 单选 | 通知/意见/办法/规定/其他 |
| 关键词标签 | 多选 | 人工智能/数据要素 |
| 优先级 | 单选 | 高/中/低 |
| 处理状态 | 单选 | 待处理/处理中/已完成 |
| 相关政策 | 文本 | 关联的其他政策 |
| 备注 | 文本 | 额外信息 |

### 4. 获取表格Token

1. 打开多维表格的网页版
2. 从URL中复制 `base_token`（App Token）
   - URL格式：`https://example.feishu.cn/base/xxx`
   - `xxx` 部分就是 base_token

### 5. 获取数据表ID

1. 点击左下角数据表名称
2. 复制数据表的 `table_id`
   - 或者通过API查询：使用 base_token 调用 API 获取所有表

### 6. 配置config.yaml

编辑 `references/config.yaml`：

```yaml
feishu_bitable:
  enabled: true
  app_id: "cli_xxxxx"              # 你的App ID
  app_secret: "your_secret"        # 你的App Secret
  base_token: "xxxxx"              # 多维表格App Token
  table_id: "tblxxxxx"             # 数据表ID
```

## 字段映射说明

系统自动将抓取的数据映射到以下字段：

| 系统字段 | Bitable字段 | 示例值 |
|---------|------------|--------|
| 抓取时间 | 抓取日期 | 2026-04-22 |
| pub_date | 政策发布日期 | 2026-04-20 |
| source_name | 来源部门 | 工信部 |
| 解析的地区 | 地区 | 全国/北京/上海 |
| title | 政策标题 | 关于...的通知 |
| summary | 政策摘要 | 该政策主要... |
| url | 原文链接 | [查看原文](...) |
| 解析的文件名 | 文件原名 | notice_2026.pdf |
| 解析的政策类型 | 政策类型 | 通知/意见/办法 |
| categories | 关键词标签 | [人工智能, 数据要素] |
| 优先级判断 | 优先级 | 高/中/低 |
| 固定值 | 处理状态 | 待处理 |

## 运行系统

配置完成后运行：

```bash
cd scripts
python main.py
```

系统会自动将抓取到的政策数据写入配置的多维表格中。

## 查看数据

在飞书中打开对应的多维表格，即可看到：
- 按日期排序的政策列表
- 按地区、部门、类型筛选的数据
- 可进行二次编辑和标注

## 常见问题

### 1. 无法获取token
- 检查 App ID 和 App Secret 是否正确
- 确认应用已发布并可见范围包含当前用户

### 2. 无法写入数据
- 检查 base_token 和 table_id 是否正确
- 确认字段名称与上述列表完全一致
- 检查应用是否有足够的权限

### 3. 数据格式错误
- 日期字段需要是日期类型
- 多选字段需要配置对应的选项
- 超链接字段需要是超链接类型

## 高级用法

### 自定义字段

如需添加自定义字段，修改 `scripts/feishu_bitable.py` 中的 `_build_fields` 方法。

### 自动去重

系统会自动检查已存在的URL，避免重复写入。

### 批量更新

可以通过Bitable API对已抓取的数据进行批量更新，如更改处理状态。
