# Gov Policy Tracker - 政府政策自动追踪系统

用于监控政府网站政策动态，自动抓取、解析、筛选人工智能和数据要素相关政策，并推送到飞书/钉钉/邮件。

## 使用场景

- 搭建政策追踪自动化系统
- 监控政府网站政策更新
- 获取AI/数据政策最新动态
- 将政策推送到飞书表格或群聊

## 使用方式

### 1. 环境准备

```bash
pip install requests beautifulsoup4 lxml pyyaml jinja2 python-dateutil
```

### 2. 配置文件

编辑 `references/config.yaml` 配置数据源、关键词和推送目标：
- `sources`: 监测的政府网站列表（URL、解析器配置）
- `keywords`: 匹配关键词（AI组、数据组）
- `targets`: 推送目标（飞书Webhook、钉钉、邮件）

### 3. 运行脚本

```bash
python scripts/main.py
```

### 4. 定时任务

使用系统定时器（如Windows任务计划程序、cron）每日执行。

## 系统架构

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  调度与编排   │ -> │  信息采集    │ -> │  解析与筛选  │ -> │  加工与分发  │
│  main.py    │    │  collector  │    │  parser     │    │  processor  │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

## 核心模块

### collector.py - 信息采集层

- **User-Agent轮换**: 每次请求随机使用真实浏览器UA
- **超时控制**: 连接5s，读取10s
- **重试策略**: 指数退避（2s, 4s），最多2次
- **并发控制**: 最大5个并发

### parser.py - 解析与筛选层

- **配置化解析器**: 每个数据源独立CSS选择器
- **日期标准化**: 统一转换为 `YYYY-MM-DD`
- **关键词匹配**: 标题匹配AI/数据关键词

### processor.py - 加工与分发层

- **摘要生成**: 关键词上下文截取或大模型API
- **Jinja2模板**: 格式化报告输出
- **多渠道推送**: 飞书/钉钉/邮件

## 异常处理

| 场景 | 处理方式 |
|------|----------|
| 解析失效 | 连续3天0条目 → 告警 + 人工介入 |
| 推送失败 | 自动切换备用渠道 |
| 网络不通 | 跳过并记录，下周期重试 |
| API超限 | 自动降级到本地摘要法 |
