# Gov Policy Tracker - 政府政策追踪

## 适用场景

当用户需要：
- 监控政府网站最新政策动态
- 获取人工智能/数据要素相关政策
- 将政策信息整理到飞书表格
- 定期汇总政策更新

## 执行流程

### Step 1: 搜索政策

使用 `web_search` 搜索目标政府网站的政策信息。

**搜索关键词组合：**
- `site:miit.gov.cn 人工智能 通知 2026`
- `site:most.gov.cn 数据要素 政策 2026`
- `site:ndrc.gov.cn 大模型 实施方案 2026`
- `site:shanghai.gov.cn 数字经济 政策 2026`
- `site:gd.gov.cn 人工智能 政策 2026`

**参考配置见：** `references/gov_sources.md`

### Step 2: 筛选政策

根据以下条件筛选：
- **关键词匹配**：标题包含 "人工智能"、"AI"、"大模型"、"数据要素"、"数据安全"、"数字经济"
- **时间范围**：优先最近30天内发布的政策
- **政策类型**：通知、意见、办法、方案、规划、指南

### Step 3: 提取信息

对每条政策提取：
- **政策标题**：完整标题
- **发布部门**：发文机关
- **发布日期**：官方发布日期（YYYY-MM-DD格式）
- **原文链接**：官方网页URL
- **政策摘要**：使用LLM生成3-5点摘要（背景、措施、适用范围、预期效果）
- **地区标签**：北京/上海/广东/全国等
- **关键词标签**：人工智能/数据要素/大模型等

### Step 4: 去重检查

检查飞书表格中是否已存在相同标题或链接的政策，避免重复。

### Step 5: 推送到飞书

使用飞书Bitable API写入数据：

**字段映射：**
| 飞书字段 | 值 |
|---------|-----|
| 总览 | 【地区】标题+来源+类型+标签 |
| 地区 | 北京/上海/广东/全国等（彩色单选）|
| 政策标题 | 完整标题 |
| 来源部门 | 发文机关全称 |
| 政策类型 | 通知/方案/规划等 |
| 政策摘要 | 分点概括 |
| 原文链接 | 官方URL |
| 关键词标签 | 人工智能/数据要素等（彩色多选）|
| 优先级 | 高/中/低（彩色单选）|
| 处理状态 | 待处理（彩色单选）|
| 政策发布日期 | 时间戳 |
| 抓取日期 | 当前时间戳 |

**API调用示例：**
```
POST https://open.feishu.cn/open-apis/bitable/v1/apps/{base_token}/tables/{table_id}/records
Authorization: Bearer {tenant_access_token}
Content-Type: application/json

{
  "fields": {
    "政策标题": "xxx",
    "地区": "北京",
    ...
  }
}
```

**凭证配置见：** `references/config.yaml`（需填写真实凭证）

## 关键配置

### 数据源列表
见 `references/gov_sources.md`，包含：
- 国家部委：工信部、科技部、发改委、数据局
- 地方政府：北京、上海、广东、深圳、浙江等

## 注意事项

1. **时间准确性**：以官方文件"发布日期"为准，非抓取日期
2. **去重机制**：同一政策（相同标题或URL）只写入一次
3. **字段完整**：所有字段必须填写，不留空值
4. **彩色标签**：地区/关键词标签/优先级使用飞书预设的彩色选项
5. **安全性**：飞书凭证请填写在本地配置文件，切勿提交到公开仓库

## 执行频率建议

- **每日更新**：每天早上10点执行，抓取前一天的政策
- **手动触发**：用户需要时即时搜索

## 失败处理

- 搜索无结果：返回"今日无新政策"
- API写入失败：记录错误日志，提示用户检查飞书凭证
- 网络超时：重试3次后跳过
