# 定时任务设置说明

## 功能
每天早上10点自动采集前一天的政策新闻数据，并更新到飞书表格。

## 文件说明

- `auto_update.py` - 自动更新主程序
- `run_auto_update.bat` - Windows批处理脚本（用于定时任务调用）
- `logs/` - 日志目录，记录每次执行情况

## 设置步骤

### 第一步：测试脚本

双击运行 `run_auto_update.bat`，确保能正常执行。

### 第二步：创建Windows定时任务

#### 方法一：使用命令行快速创建（推荐）

以**管理员身份**打开 PowerShell，执行以下命令：

```powershell
# 创建任务 - 每天早上10点执行
$action = New-ScheduledTaskAction -Execute "C:\Users\renmeijing\ComateProjects\comate-zulu-demo\policy-tracker\run_auto_update.bat"
$trigger = New-ScheduledTaskTrigger -Daily -At "10:00AM"
$settings = New-ScheduledTaskSettingsSet -AllowStartIfOnBatteries -DontStopIfGoingOnBatteries
Register-ScheduledTask -TaskName "政策追踪自动更新" -Action $action -Trigger $trigger -Settings $settings -Description "每天早上10点自动采集前一天的政策新闻"
```

#### 方法二：使用图形界面手动设置

1. 打开 **任务计划程序**（搜索"任务计划程序"或 `taskschd.msc`）

2. 点击右侧 **"创建基本任务"**

3. 填写信息：
   - 名称：`政策追踪自动更新`
   - 描述：`每天早上10点自动采集前一天的政策新闻`

4. 触发器选择：**每天**

5. 设置时间：**10:00:00**

6. 操作选择：**启动程序**

7. 程序/脚本填写：
   ```
   C:\Users\renmeijing\ComateProjects\comate-zulu-demo\policy-tracker\run_auto_update.bat
   ```

8. 完成时勾选 **"打开属性对话框"**

9. 在属性对话框中：
   - 勾选 **"不管用户是否登录都要运行"**
   - 勾选 **"使用最高权限运行"**
   - 条件选项卡：取消勾选 **"只有在计算机使用交流电源时才启动"**

10. 点击 **确定** 保存

### 第三步：验证任务

1. 在任务计划程序中找到 **"政策追踪自动更新"**
2. 右键点击 **"运行"** 测试
3. 检查飞书表格是否有更新

### 第四步：查看日志

执行日志保存在 `logs/auto_update.log`，可以查看每次执行的时间和结果。

## 任务管理命令

```powershell
# 查看任务列表
Get-ScheduledTask | Where-Object {$_.TaskName -like "*政策*"}

# 手动运行任务
Start-ScheduledTask -TaskName "政策追踪自动更新"

# 停止任务
Stop-ScheduledTask -TaskName "政策追踪自动更新"

# 删除任务
Unregister-ScheduledTask -TaskName "政策追踪自动更新" -Confirm:$false
```

## 常见问题

### Q: 任务执行了但飞书表格没有更新？
A: 检查 `logs/auto_update.log` 查看执行日志，确认是否有报错。

### Q: 如何修改执行时间？
A: 在任务计划程序中找到任务，右键 **"属性"** → **"触发器"** 选项卡 → 编辑。

### Q: 能否改为每小时执行一次？
A: 修改触发器设置，选择 **"重复任务间隔"** 为1小时。

### Q: 电脑关机时错过的任务怎么办？
A: 在任务属性 → **"设置"** 选项卡中，勾选 **"如果错过了计划开始时间，立即启动任务"**。
