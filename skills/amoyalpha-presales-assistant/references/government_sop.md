# 政府客户摸底 SOP

适用于客户单位名称含"局/委/办/厅/部/处/管委会/政务"等关键词的场景。

---

## 一、核心摸底维度

### A. 采购与决策机制

| 摸底项 | 说明 | 优先级 |
|--------|------|--------|
| 预算来源 | 年度财政预算 / 专项资金 / 中央转移支付 | 高 |
| 采购方式 | 公开招标 / 竞争性谈判 / 单一来源 / 框架协议 | 高 |
| 预算节点 | 当年预算是否已申报 / 预算批复时间 | 高 |
| 决策层级 | 处室主任→分管副局长→局长→政府常务会 | 高 |
| 信创要求 | 是否强制要求信创产品目录 / CPU/OS/数据库 | 高 |

### B. 项目背景

| 摸底项 | 说明 | 优先级 |
|--------|------|--------|
| 政策驱动 | 对应哪项国家/地方政策文件（如数字政府、数字中国） | 高 |
| 主管部门 | 是否有上级主管部门指导或要求 | 中 |
| 历史项目 | 是否有前期建设基础，数据迁移压力如何 | 中 |
| 竞品接触 | 是否已接触华为、阿里、腾讯等主要竞争方 | 高 |

### C. 关键干系人

| 角色 | 关注点 | 沟通策略 |
|------|--------|---------|
| IT处/信息中心主任 | 技术可行性、运维负担、信创合规 | 重点公关，提供技术方案评审支撑 |
| 分管副局长 | 政策合规性、上级督导压力、风险 | 强调政策背书与案例 |
| 财务/预算处 | 预算合规、支付周期 | 提前对齐采购方式和合同结构 |
| 法制处/合规 | 合同条款、知识产权、数据安全 | 提前准备标准合同审查版本 |

## 二、关键问题清单（首次拜访）

1. 本次项目是否已列入年度预算？预算规模大致在哪个区间？
2. 贵单位是否有信创采购要求？对国产化适配有哪些具体规定？
3. 项目决策需经过哪些层级审批？预计决策周期多长？
4. 是否有上级主管部门对本项目有指导意见或验收要求？
5. 目前是否已接触其他供应商？评标标准是否已初步确定？

---

## 三、政府客户常见痛点

- **数据孤岛**：各委办局数据标准不统一，无法横向共享
- **信创合规压力**：采购目录限制多，国产化替代周期长
- **运维能力弱**：缺乏专业IT运维团队，希望交钥匙工程
- **考核压力**：上级有数字政府、政务服务评分等考核指标
- **预算刚性**：超预算采购程序复杂，二期采购不确定性高

---

## 四、风险提示

- 信创不达标会被一票否决，务必提前确认产品目录
- 政府采购周期长（3-6个月），需提前布局预算申报节点
- 避免在正式采购前承诺价格，可能触发"打招呼"合规问题
- 不得在方案中引用未经核实的政府客户案例名称
