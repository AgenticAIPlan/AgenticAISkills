# 技术型企业客户摸底 SOP

适用于客户含"AI/SaaS/初创/API/开发者/科技/互联网"等技术关键词的场景。

---

## 一、核心摸底维度

### A. 采购与决策机制

| 摸底项 | 说明 | 优先级 |
|--------|------|--------|
| 技术决策模式 | CTO/技术VP直接决策 / 工程团队主导 / 产品驱动 | 高 |
| 预算类型 | 研发预算 / 云服务预算 / API调用量计费 / 项目制 | 高 |
| 评估方式 | POC优先 / Benchmark优先 / 开发者体验优先 | 高 |
| 付费偏好 | 按量付费 / 年度订阅 / 私有化部署 | 中 |

### B. 技术现状

| 摸底项 | 说明 | 优先级 |
|--------|------|--------|
| 技术栈 | 主要使用哪些编程语言/框架/云平台 | 高 |
| 现有AI能力 | 是否已有自建模型 / 使用哪些API服务 | 高 |
| 数据情况 | 是否有标注数据 / 数据安全级别 / 是否可上云 | 高 |
| 团队规模 | 算法团队人数 / 工程化能力 / 是否有MLOps | 中 |
| 集成需求 | 需要与哪些内部系统/平台集成 | 中 |

### C. 关键干系人

| 角色 | 关注点 | 沟通策略 |
|------|--------|---------|
| CTO / 技术VP | 技术架构先进性、API质量、扩展性、避坑 | 技术深度对话，提供架构白皮书 |
| 算法工程师 | 模型效果、接口易用性、文档质量 | 提供试用账号和技术文档 |
| 产品经理 | 功能覆盖度、上线周期、用户体验 | 聚焦场景落地，提供Demo |
| CEO / 创始人 | 商业价值、战略合作、融资背书 | 强调生态价值和增长潜力 |

## 二、关键问题清单（首次拜访）

1. 当前技术团队的规模和构成如何？是否有专门的AI/算法团队？
2. 目前在AI能力上的主要痛点是什么？是效果问题、工程化问题还是成本问题？
3. 是否做过技术选型评估？对比了哪些方案？评估标准是什么？
4. 数据是否可以上云？对数据安全和隐私有哪些要求？
5. 预计的集成周期和上线时间节点是什么？

---

## 三、技术型企业常见痛点

- **模型效果天花板**：自建模型效果不稳定，需要更强的基础模型
- **工程化成本高**：从模型到产品的工程化链路复杂，周期长
- **数据飞轮不足**：用户规模小，标注数据积累慢，难以持续优化
- **API调用成本**：随规模扩张，API费用快速增长
- **私有化 vs 云端**：数据安全诉求与云端部署成本之间的矛盾

---

## 四、风险提示

- 技术团队决策快（1-2个月），但对技术细节极为敏感，虚假承诺代价高
- POC阶段是关键窗口，需要配置专属技术支持资源
- 避免用非技术话术应对技术问题，会失去信任
- 技术型企业竞争意识强，不得在方案中披露竞品技术细节
