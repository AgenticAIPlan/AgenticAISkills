# 行业头部企业客户摸底 SOP

适用于客户单位名称含"集团/上市/行业龙头/500强/控股"等关键词的场景。

---

## 一、核心摸底维度

### A. 采购与决策机制

| 摸底项 | 说明 | 优先级 |
|--------|------|--------|
| 采购流程 | 是否有集团采购委员会 / 战略采购 / 分子公司自采 | 高 |
| 预算审批 | 年度IT预算周期 / 临时立项流程 / 资本支出 vs 运营支出 | 高 |
| 决策链路 | CIO/CTO→CFO→CEO→董事会（视金额而定） | 高 |
| 供应商准入 | 是否有供应商白名单 / 认证体系 | 中 |
| 集采框架 | 是否已有战略供应商框架合同可利用 | 中 |

### B. 项目背景

| 摸底项 | 说明 | 优先级 |
|--------|------|--------|
| 业务驱动 | 降本增效 / 数字化转型 / 新业务拓展 / 监管合规 | 高 |
| 竞品现状 | 现有供应商是谁 / 为何考虑替换 / 满意度如何 | 高 |
| 技术债务 | 现有系统架构、数据积累、迁移成本 | 中 |
| 战略时机 | 是否有组织架构调整、新CIO上任等窗口期 | 高 |

### C. 关键干系人

| 角色 | 关注点 | 沟通策略 |
|------|--------|---------|
| CIO / IT负责人 | 技术先进性、集成复杂度、运维成本、团队能力 | 技术深度对话，展示架构优势 |
| CFO / 财务负责人 | ROI、TCO、付款周期、资本化处理 | 提供量化测算模型 |
| 业务部门负责人 | 使用体验、业务目标达成、上线周期 | 聚焦业务价值，避免技术细节 |
| 采购/供应链 | 合规、价格谈判、合同条款 | 标准条款先行，减少谈判摩擦 |

---

## 二、关键问题清单（首次拜访）

1. 本次项目的核心业务诉求是什么？是降本、提效还是支撑新业务？
2. 现有供应商/系统存在哪些主要痛点，促使贵司考虑新方案？
3. 项目的预算规模和审批流程是怎样的？预计何时完成立项？
4. 决策链中除IT部门外，还需要哪些部门介入？
5. 对供应商有哪些硬性要求（资质、服务能力、本地化支持）？

---

## 三、头部企业常见痛点

- **数据资产管理**：数据量大、质量参差，难以统一治理
- **供应商锁定风险**：希望避免单一供应商依赖，要求开放接口
- **集成复杂度**：现有系统众多（ERP/CRM/SCM），集成工作量大
- **交付风险**：项目预算大、周期长，对供应商的项目管理能力要求高
- **ROI压力**：CFO要求量化收益，难以用软性价值说服

---

## 四、风险提示

- 头部企业决策周期中等（2-4个月），但竞争激烈，需提前建立信任
- 避免只对接IT部门，需同步打通业务部门和C层
- 大型集团可能存在内部政治，需识别真实决策者
- 不得在方案中引用竞品负面数据，除非有公开来源
