# 异议处理作战手册

本手册覆盖三类客户（政府、头部企业、技术型企业）的高频异议，提供结构化话术框架。

---

## 使用说明

每条异议处理遵循**三层话术结构**：
1. **共情层**：认可对方顾虑，建立信任
2. **重框架层**：转换视角，重新定义问题
3. **证据层**：用具体数据/案例/承诺支撑观点

---

## 一、价格类异议

### P1：「你们比竞品贵太多了」

**真实顾虑**：预算压力 + 对价值差异认知不足

**话术**：
- 共情：「价格确实是采购决策的重要因素，我完全理解。」
- 重框架：「我们来拆解一下总拥有成本——初始采购价只是第一年，但运维成本、二次开发成本、迁移成本往往是3-5年的持续支出。」
- 证据：「【待补充：同类客户 TCO 对比数据】。如果您方便，我们可以一起做一个5年 TCO 测算。」

**红线说法**：❌「我们其实不贵」 ❌「竞品质量差，你会后悔的」

---

### P2：「现在预算不够，等明年」

**真实顾虑**：预算周期限制，或观望态度

**话术**：
- 共情：「预算节点的限制我们很理解，这是采购流程的现实。」
- 重框架：「我们可以一起设计一个分期方案：今年完成核心模块立项，明年扩展。这样不影响您今年的预算额度，又能让项目今年启动。」
- 证据：「我们有几个客户用过类似结构，【待补充：具体案例】。」

---

## 二、技术类异议

### T1：「你们的模型效果我们测试过，不如XX」

**真实顾虑**：技术选型风险，担心效果达不到生产要求

**话术**：
- 共情：「Benchmark 测试确实是技术选型的重要参考，您的严谨性让我印象深刻。」
- 重框架：「通用 Benchmark 和特定业务场景的效果有时会有差距。能否分享一下您的测试数据集和评估指标？我们希望在您的真实场景上做一次针对性验证。」
- 证据：「我们在【待补充：同类场景】上的专项优化效果是【待核实】，POC 阶段我们会配置专属算法支持。」

---

### T2：「你们的 API 不稳定，我们听说过故障」

**真实顾虑**：生产可用性风险，担心影响自身业务

**话术**：
- 共情：「稳定性是 API 服务最核心的要求，这个顾虑完全合理。」
- 重框架：「我们来看实际的 SLA 数据，而不是单一事件。」
- 证据：「【待补充：SLA 可用率数据】。我们愿意在合同中约定 SLA 条款和赔偿标准，以实际承诺代替口头保证。」

---

## 三、信任/资质类异议

### C1：「你们在我们行业没有成功案例」

**真实顾虑**：行业经验风险，担心踩坑

**话术**：
- 共情：「行业经验确实能降低项目风险，这是非常务实的考量。」
- 重框架：「我们在【待补充：相近行业】积累了深厚经验，核心技术逻辑在您行业同样适用。更重要的是，我们愿意为这个项目配置有该行业背景的交付团队。」
- 证据：「我可以安排【待补充：行业背景顾问/客户参考】与您直接交流。」

---

### C2：「你们公司规模小，担心长期服务保障」

**真实顾虑**：供应商持续经营风险

**话术**：
- 共情：「对于长期项目，供应商的持续服务能力确实需要慎重评估。」
- 重框架：「我们的持续经营能力体现在：【待补充：融资情况/营收规模/客户留存率】。同时，我们可以在合同中加入源代码托管、数据导出权利等条款，保护您的权益。」
- 证据：「【待补充：公司背景/股东背景】」

---

## 四、政府客户专项异议

### G1：「你们不在信创目录里」

**话术**：
- 共情：「信创合规是政府采购的前提条件，这个要求我们完全尊重。」
- 重框架：「我们目前【待补充：信创认证进展】。在认证完成前，有两个可行路径：一是通过我们已认证的信创合作伙伴以集成方式参与；二是在政策允许的情况下，以技术服务而非产品采购形式立项。」
- 证据：「我可以提供具体的合规路径说明文件供贵处法制审核。」

---

### G2：「领导说要等上级政策明确再推进」

**话术**：
- 共情：「等待上级指导是负责任的做法，我们完全理解。」
- 重框架：「我们可以利用这段时间做两件事：一是完成需求调研和方案预研，不占用预算；二是参加我们组织的同类政府客户交流活动，了解其他地区的推进经验，供领导决策参考。」

---

## 五、通用话术禁区

以下表达方式在任何场景下**严禁使用**：

| 禁用表达 | 原因 |
|---------|------|
| 「竞品有严重 BUG」 | 无法核实，损害专业形象 |
| 「我们绝对没有问题」 | 过度承诺，埋下纠纷隐患 |
| 「这个价格只有今天有效」 | 制造虚假紧迫感，破坏信任 |
| 「你们不懂技术才会这么想」 | 对客户不尊重 |
| 「XX客户就是这么用的」（未经授权） | 违反客户保密义务 |
