# 专家运营内容评分口径

## 目标

对“行业专家运营经验帖/教程/文章”做统一质量评估，得到 0-100 的参考分值，并自动映射分层。

默认规则：`>=60` 才值得继续进入策略拆解与整合环节。

## 七维评分

| 维度 | 权重 | 高分信号 | 低分信号 |
|---|---:|---|---|
| 专业可信度 | 1.8 | 作者背景清晰，结论符合行业常识 | 身份不明、观点玄学化 |
| 方法可复用性 | 1.8 | 方法能迁移到专家运营场景 | 强依赖个案或特殊资源 |
| 证据完整度 | 1.4 | 有数据、案例、过程、样本 | 只有结论，没有证据 |
| 业务匹配度 | 1.4 | 与招募、冷启动、内容、分层、激励、转化等强相关 | 泛运营鸡汤 |
| 落地可操作性 | 1.4 | 有步骤、SOP、话术、机制 | 只讲理念 |
| 时效性 | 1.1 | 规则/平台机制仍适用 | 明显过期 |
| 风险透明度 | 1.1 | 有边界、风险、反例说明 | 只讲收益不讲代价 |

## 分层规则

| 分值 | 分层 | 使用建议 |
|---|---|---|
| 90-100 | S-可直接复用 | 优先沉淀、优先试点 |
| 80-89 | A-高价值参考 | 建议进入策略库 |
| 70-79 | B-部分参考 | 可保留，但需要改造 |
| 60-69 | C-谨慎参考 | 只提取局部做法 |
| 0-59 | D-不建议参考 | 默认不进入策略库 |

## 评分依据写法

建议始终包含三段：

```markdown
优点：
- 为什么值得参考

风险：
- 哪些地方不能直接照搬

适用前提：
- 在什么条件下适用
```
