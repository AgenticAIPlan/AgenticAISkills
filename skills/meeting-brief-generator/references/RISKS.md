# 风险标签库

本文档整理会谈中需要注意的风险点和避坑建议。

---

## 一、风险类型定义

| 风险类型 | 说明 | 处理建议 |
|----------|------|----------|
| 数据合规 | 涉及数据跨境、敏感数据 | 需法务确认，明确数据使用边界 |
| 商业敏感 | 融资、上市、分润等 | 需内部确认后谨慎表述 |
| 未公开信息 | 未发布产品、未签约合作 | 避免主动提及，已公开的才说 |
| 政策表达 | 涉及政府、国家战略 | 用词需准确，避免夸大 |
| 竞品关系 | 对方同时与竞品合作 | 避免过度承诺，明确合作边界 |
| 合同法律 | 未正式签署的协议 | 用"拟"、"计划"等表述 |
| 人事变动 | 可能涉及的人事信息 | 避免敏感人事讨论 |
| 财务信息 | 收入、成本等财务数据 | 不对外披露内部数据 |

## 二、DeepX风险标记示例

### 商业敏感信息

- ⚠️ 融资信息：Morgan Stanley融资及韩国总统批复资金（需确认公开性）
- ⚠️ 上市计划：2026-2027年美国上市（需谨慎表述）
- ⚠️ 分润比例：百度30%软件层产品分润（内部信息，领导口径需谨慎）

### 技术信息

- ⚠️ 2nm工艺：与三星签署2纳米工艺合同（需确认是否可公开）
- ⚠️ 量产计划：2025年正式量产（确认当前状态）

### 国际政治信息

- ⚠️ 总统接见：受到韩国现任总统接见（需确认官方表述）

### 表述建议

```
❌ 避免：DeepX获得了数亿美元政府资金，总统亲自接见
✅ 建议：DeepX获得了重要的政府支持，在韩国AI战略中扮演关键角色

❌ 避免：我们签署了30%分润协议
✅ 建议：双方已就商业化合作达成一致（具体比例内部确认）
```

## 三、港中文风险标记示例

### 数据合规

- ⚠️ 数据跨境：20T农业数据的跨境使用（需法务评估）
- ⚠️ 国家重点实验室：需准确使用官方称谓
- ⚠️ 数据合作模式：需明确数据所有权和使用边界

### 政策表达

- ⚠️ 国家战略："北豆南移"计划（需准确表述）
- ⚠️ 国家任务：承接国家战略科研任务（确认具体表述）
- ⚠️ 国家粮食安全：（需准确，避免夸大）

### 国际合作

- ⚠️ 国际国家/地区：西语系国家、拉美、南美（确认具体范围）
- ⚠️ 国际组织：联合国、粮农组织等（确认合作事实）

### 表述建议

```
❌ 避免：我们有20T数据随便用
✅ 建议：实验室积累了丰富的农业生物数据，双方将探索合规的数据合作模式

❌ 避免：我们和联合国合作推广
✅ 建议：基于大豆品种的特殊性，双方将探索与西语系国家的合作机会
```

## 四、通用避坑指南

### 开场阶段

| 避坑点 | 建议 |
|--------|------|
| 不了解对方就乱说 | 先确认对方背景，使用"据我们了解"等表述 |
| 过度承诺 | 用"探索"、"计划"而非"保证"、"一定" |
| 涉及敏感话题 | 政治、宗教、民族等话题避开 |
| 用词不当 | 避免夸张、绝对化表达 |

### 中段交流

| 避坑点 | 建议 |
|--------|------|
| 泄露内部信息 | 融资、收入、人事等信息不主动说 |
| 评价竞品 | 不贬低竞品，强调自身优势 |
| 承诺无法兑现 | 不承诺没把握的事，说"我们评估一下" |
| 技术细节不准确 | 技术问题可请技术人员确认后回答 |

### 结束阶段

| 避坑点 | 建议 |
|--------|------|
| 未明确下一步 | 一定要明确谁跟进、做什么、何时做 |
| 留下模糊承诺 | 已确认的才写下来，不确定的说"进一步确认" |
| 未确认合作边界 | 明确双方做什么、不做什么 |

## 五、不同对象的风险关注点

### 企业客户

| 风险点 | 建议 |
|--------|------|
| 竞品关系 | 了解对方是否与竞品合作，避免冲突 |
| 商业条款 | 涉及价格、分润、合同需内部确认 |
| 技术秘密 | 不泄露我方未公开的技术路线 |

### 高校/科研机构

| 风险点 | 建议 |
|--------|------|
| 数据合规 | 数据跨境使用需法务评估 |
| 学术表达 | 学术术语准确，不夸大成果 |
| 知识产权 | 明确合作成果的归属问题 |

### 政府/协会

| 风险点 | 建议 |
|--------|------|
| 政策表述 | 准确引用官方文件，不夸大 |
| 政治敏感 | 避免政治争议话题 |
| 服务承诺 | 不承诺超出能力范围的服务 |

### 国际组织

| 风险点 | 建议 |
|--------|------|
| 文化差异 | 了解对方文化，避免文化冲突 |
| 地缘政治 | 避免敏感国际政治话题 |
| 法律差异 | 不同国家法律不同，需法律确认 |

## 六、敏感信息处理流程

### 信息判断流程

```
信息是否公开？
    ↓
否 → 是否能说？
    ↓
不确定 → 标记"待确认"，会后核实
是 → 表述准确
```

### 待确认信息清单

会谈中遇到不确定的信息，记录在待确认清单：

| 信息类型 | 内容 | 需确认什么 | 责任人 |
|----------|------|-----------|--------|
| 商业数据 | 收入、分润 | 是否可公开 | 商务 |
| 技术信息 | 未发布产品 | 发布时间线 | 技术 |
| 合作信息 | 未签约合作 | 合作状态 | 项目 |
| 数据信息 | 数据规模、来源 | 数据合规性 | 法务 |

## 七、领导口径特别提醒

### 领导发言需注意

1. **分润比例、收入等财务数据** - 仅内部讨论，不对外说
2. **未签署的合作** - 用"计划"、"拟"、"推进中"表述
3. **竞争对手** - 不主动提，不贬低，强调自身
4. **技术路线** - 已公开的说，未公开的内部确认

### 可说的内容

1. **已签约合作** - 具体说清楚
2. **已落地案例** - 可详细描述
3. **已发布产品** - 可详细介绍
4. **公开数据** - 可引用官方数据

### 语气把控

- **自信但不夸张** - 能做到的说，做不到的别承诺
- **专业但不傲慢** - 尊重对方，展现专业
- **开放但有边界** - 合作态度开放，但不泄露底线

## 八、风险检查清单

会谈前检查：

- [ ] 确认对方是否与竞品合作
- [ ] 确认合作状态（已签约/拟签约/意向）
- [ ] 确认数据使用是否合规
- [ ] 确认敏感信息是否已获授权
- [ ] 确认表述是否准确无误

会谈中注意：

- [ ] 不主动透露未公开信息
- [ ] 不承诺无法兑现的事
- [ ] 不夸大我方能力
- [ ] 不贬低竞争对手
- [ ] 不涉及政治敏感话题

会谈后跟进：

- [ ] 待确认信息是否已核实
- [ ] 达成共识是否已记录
- [ ] 下一步是否已明确
- [ ] 风险点是否已标记
