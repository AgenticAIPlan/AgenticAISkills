# 二轮对话交互提示词

本文档提供在二轮对话时用户补充素材的交互提示词，方便系统快速理解和处理输入信息。

---

## 一、用户提供信息的标准格式

### 格式1：简化版（推荐日常使用）

```
【会谈对象】[机构名称]
【会谈类型】[商务拜访/签约仪式/座谈交流/战略合作/技术交流]
【地区维度】[厦门/香港/深圳/成都/长沙/东莞/武汉/南昌/广州]
【会谈时间/地点】[时间+地点]
【我方领导】[姓名]
【对方参会人】[姓名+职务]

【对方简介】
[简要描述对方是谁、做什么的]

【已有合作】
[如果有，简述合作内容]

【对方诉求】
[对方想什么、要什么]

【我方诉求】
[我方想什么、要什么]

【会谈目标】
[本次会谈的目标]

【补充材料】
[如有，提供链接或内容]
```

### 格式2：素材补充格式（更新知识库）

当用户需要补充地区信息、合作案例等素材时，使用以下格式：

```
【素材类型】[地区信息/合作案例/能力标签/话术/风险]

【地区/对象】[具体地区或对象名称]

【具体内容】
[按以下结构罗列]

【基本信息】
（负责人、定位、时间线等）

【核心合作】
（合作方、内容、状态）

【成果/亮点】
（可对外表达、数据成果）

【话术/引用】
（领导可直接引用的表达）

【后续推进】
（下一步计划）
```

## 二、用户输入信息检查清单

在生成谈参前，系统会检查以下信息：

### 基本信息检查

- [ ] 会谈对象（机构名称）
- [ ] 会谈类型
- [ ] 地区维度
- [ ] 我方领导
- [ ] 对方参会人

### 扩展信息检查（建议提供）

- [ ] 会谈时间、地点
- [ ] 对方简介（行业地位、核心业务、技术资源）
- [ ] 关键人背景（教育/职业背景、代表成果）
- [ ] 已有合作基础（合作时间线、技术合作、产品共创、落地案例）
- [ ] 双方诉求（对方诉求、我方诉求）
- [ ] 会谈目标（理想结果、保底结果）
- [ ] 特殊事项（签约、PR、商业化、数据合作、政府关系、国际事务）

---

## 三、缺失信息的处理

当信息不完整时，系统会：

1. **基于已有信息生成谈参**：利用地区维度、对象类型等已有信息生成框架性内容
2. **输出待补充信息清单**：列出缺失的重要信息，提示用户在会后或会谈前补充
3. **标注不确定内容**：对无法确认的内容用"待核实"标注

## 五、用户与系统的交互流程

### 第一次交互（生成谈参）

1. 用户按简化版格式提供会谈信息
2. 系统判断地区维度和对象类型
3. 系统查询地区合作基础、相似案例、话术模板
4. 系统生成完整谈参

### 第二次交互（补充素材）

1. 用户按素材补充格式提供新信息
2. 系统识别素材类型（地区信息/案例/政策等）
3. 系统验证信息完整性
4. 系统更新相应知识库文件
5. 系统反馈更新结果

### 第三次交互（优化谈参）

1. 用户基于新补充的素材，提供优化需求
2. 系统重新生成或更新谈参
3. 系统输出最终版本

## 七、常见问题与提示

### Q：信息不确定怎么办？

**提示**：可以标注"待确认"，系统会将其列为待核实事项

```
【对方简介】
技术实力较强，具体待确认
```

### Q：没有合作基础怎么办？

**提示**：可以留空或写"初次接触"，系统会生成关系建立型谈参

```
【已有合作】
初次接触，暂无合作基础
```

### Q：涉及多个地区怎么办？

**提示**：在"地区维度"标注"多地区"，并分别描述

```
【地区维度】
多地区（深圳、香港）

【深圳合作】
...

【香港合作】
...
```

### Q：提供原始文档链接怎么办？

**提示**：在"补充材料"中提供链接，系统会尝试读取和解析

```
【补充材料】
文档链接：https://xxx
飞书记录：https://xxx
```

## 九、系统处理逻辑

### 信息提取逻辑

```
1. 识别用户输入类型（生成谈参 / 补充素材）
2. 提取结构化信息（会谈对象、地区、类型、内容等）
3. 验证信息完整性（必填项检查）
4. 匹配知识库（地区信息、相似案例、话术模板）
5. 生成输出（谈参 / 更新确认）
```

### 补充素材处理逻辑

```
1. 识别素材类型（地区/案例/政策/话术/风险）
2. 解析内容结构（按对应格式解析）
3. 验证数据完整性
4. 更新知识库文件
5. 返回更新确认
```

## 十、质量控制

### 输入质量检查

- [ ] 地区维度是否明确
- [ ] 对象类型是否清晰
- [ ] 合作成熟度是否可判断
- [ ] 诉求分析是否合理
- [ ] 话术风格是否匹配

### 输出质量检查

- [ ] 结构是否完整（11个模块）
- [ ] 地区合作基础是否准确
- [ ] 案例引用是否相关
- [ ] 话术风格是否得体
- [ ] 风险提示是否充分

---

## 十一、用户反馈

如果系统输出不符合预期，用户可以：

1. **指出问题**：说明哪个部分不准确或不合适
2. **补充信息**：提供更多背景信息
3. **调整要求**：明确需要调整的方向（如更正式/更简洁/更详细）
