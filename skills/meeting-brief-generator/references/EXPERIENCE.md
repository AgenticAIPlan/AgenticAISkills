# 核心经验模型与方法论

本文档是从两份高质量谈参（DeepX、港中文）中抽象出的可复用经验模型。

## 一、对象类型决定谈参结构（最关键）

### 经验原则

谈参结构必须"按对象类型切换"，不能一套模板打天下。

### 对象分类与策略

| 对象类型 | 关注重点 | 表达风格 |
|----------|----------|----------|
| 企业客户/伙伴 | 商业化、技术能力、行业地位、合作成果、市场空间 | 强势、国际化、产业级 |
| 高校/科研机构 | 科研能力、国家任务、数据资源、产学研转化、长期合作 | 稳重、学术性、国家使命 |
| 政府/协会 | 政策、产业带动、示范意义、区域影响 | 庄重、战略性、服务导向 |
| 开源社区 | 技术贡献、社区活跃度、开发者生态 | 开放、技术导向、社区感 |
| 国际组织 | 全球视野、技术标准、国际合作 | 国际化、专业、中立 |

### 判断逻辑

```
IF 对象 = 企业
→ 强调：商业化、技术能力、行业地位、合作成果、市场空间

IF 对象 = 高校/科研机构
→ 强调：科研能力、国家任务、数据资源、产学研转化、长期合作

IF 对象 = 政府/协会
→ 强调：政策、产业带动、示范意义、区域影响
```

## 二、谈参本质是"三方关系建模"

### 核心框架

所有内容围绕三个核心展开：

1. **对方是谁** - 价值、资源、需求
2. **我方有什么** - 能力、产品、生态
3. **双方怎么结合** - 合作路径、战略结果

### 高级模型

```
对方价值 → 我方能力 → 合作路径 → 商业/战略结果
```

### Skill 判断逻辑

1. 对方能力识别
2. 对方需求推断
3. 我方能力匹配
4. 合作机会生成

## 三、合作成熟度模型

### 五阶段框架

过往合作不是"罗列"，而是"证明关系深度"。

| 阶段 | 定义 | 典型动作 | 可对外表达 | 下一步建议 |
|------|------|----------|------------|------------|
| 阶段1 | 技术验证 | 集成/适配 | 技术伙伴认证 | 产品共创 |
| 阶段2 | 产品共创 | PB产品 | 联合产品发布 | 落地案例 |
| 阶段3 | 落地案例 | 行业应用 | 成功案例 | 生态协同 |
| 阶段4 | 生态协同 | 活动/PR | 联合宣发 | 商业化 |
| 阶段5 | 商业化 | 分润/合同 | 商业化成果 | 战略升级 |

### DeepX 案例映射

- ✅ 阶段1：技术伙伴认证，PaddleOCR 适配 DX-M1 NPU
- ✅ 阶段2：DocMind 软硬一体产品共创
- ✅ 阶段3：华周智能摄像头、大风桥梁检测无人机
- ✅ 阶段4：CES展会、深圳AI大会百度专场
- ✅ 阶段5：软件分润协议签署

### 港中文案例映射

- ✅ 阶段1：MoU签署
- 🔄 阶段2：大豆智能服务平台（规划中）
- ⏳ 阶段3-5：待推进

## 四、时间线结构化

### 框架

好的谈参一定有"时间线"，把合作结构化成 roadmap，让领导有确定性。

| 时间维度 | 典型内容 |
|----------|----------|
| 短期（0-3个月） | PoC / 技术验证 / 方案细化 |
| 中期（3-12个月） | 产品共创 / 集成 / 平台建设 |
| 长期（1年以上） | 战略合作 / 国际拓展 / 生态共建 |

### DeepX 时间线示例

```
| 2026 Q1 | 新增文档解析能力 | PP-StructureV3 模型 | 原型验证阶段 |
| 2026 Q3 | 多模态文档理解 | PaddleOCR-VL 模型 | DeepX M2芯片预发布 |
| 2026 Q4 | 集成大语言模型 | ERNIE 4.5（0.3B） | 待LPU支持推理优化 |
```

### 港中文时间线示例

```
| 近期 | 大豆智能服务平台方案细化 | 技术对接 | 需确认数据格式 |
| 中期 | 农业生物垂类大模型 | 数据引入 | 需合规评估 |
| 长期 | 文心数据生态中心 | 平台共建 | 国际合作网络 |
```

## 五、需求层次分析

### 原则

对方诉求 ≠ 对方说的，而是你"推断的"。

### 需求层次模型

| 层次 | 定义 | 说明 |
|------|------|------|
| 表层需求 | 对方直接表达的 | "做AI+农业" |
| 中层需求 | 结构化后的需求 | 垂类模型 + 数据体系 |
| 深层需求 | 战略层面的需求 | 国家级标杆 + 国际影响力 |

### DeepX 需求分析

- 表层：强化AI芯片与百度AI生态绑定
- 中层：推进DocMind商业化、ERNIE端侧推理
- 深层：借百度生态进入中国及国际市场、背书IPO

### 港中文需求分析

- 表层：基于20T数据打造垂类大模型
- 中层：大豆全场景智能服务平台
- 深层：国家粮食安全战略任务、国际农业AI影响力

## 七、领导话术三段式

### 结构

领导话术的本质是"三段式"：

1. **认可对方**（建立关系）
2. **强调已有合作**（建立信任）
3. **引出未来合作**（推动行动）

### 示例模板

**开场寒暄：**
> 非常高兴今天能与[对方名称]团队交流。[对方]在[领域]有着深厚的技术积累和行业影响力，我们非常重视与[对方]的合作关系。

**中段表达：**
> 过去一年，双方从[阶段1]到[阶段2]已经形成了非常好的合作基础。特别是在[具体成果]方面，已经具备了进一步[下一步动作]的条件。

**结尾表达：**
> 我们希望以此次交流为新的起点，进一步推动双方在技术、产品、市场和生态层面的深度协同。

## 八、风险控制框架

### 原则

谈参必须包含"隐性风险控制"，标记不能乱说的内容。

### 风险类型

| 风险类型 | 说明 | 处理建议 |
|----------|------|----------|
| 数据合规 | 涉及数据跨境、敏感数据 | 需法务确认 |
| 商业敏感 | 融资、上市、分润 | 需内部确认后谨慎表述 |
| 未公开信息 | 未发布产品、未签约合作 | 避免主动提及 |
| 政策表达 | 涉及政府、国家战略 | 用词需准确 |
| 竞品关系 | 对方同时与竞品合作 | 避免过度承诺 |

### DeepX 风险标记示例

- ⚠️ 融资、上市信息需确认公开性
- ⚠️ 2nm工艺、量产计划需谨慎表述
- ⚠️ 分润比例（30%）属内部信息

### 港中文风险标记示例

- ⚠️ 数据跨境合规需评估
- ⚠️ "国家重点实验室"表述需准确
- ⚠️ 国际合作国家/地区需确认

## 九、核心方法论总结

**一句话定位：**

> 本方法基于"对象类型识别 + 双方关系建模 + 合作成熟度判断 + 战略目标拆解"的分析框架，将分散的会谈背景信息转化为结构化的谈参内容，支持高层会谈的高效决策与表达。

**执行流程：**

```
1. 判断会谈对象类型 → 选择对应模板
2. 提取对方核心价值 → 画像建模
3. 匹配我方能力 → 找结合点
4. 梳理过往合作 → 判断成熟度
5. 分析双方诉求 → 找共同利益
6. 设计会谈主线 → 定方向
7. 生成领导话术 → 可直接使用
8. 输出风险提示 → 避坑
9. 规划后续动作 → 可执行
```
