# ModelScope Dataset Monitor

魔搭社区数据集更新频率监控工具。通过快照对比机制，追踪两次调用之间魔搭社区新增了多少数据集，帮助决定合理的爬取频率。

## 适用场景

当运营团队需要从魔搭社区（ModelScope）向自有平台搬运数据集时，使用本 Skill 监控魔搭社区的数据集更新频率，以合理安排爬取和搬运节奏。

典型触发场景：

- 需要了解魔搭社区数据集的更新速度
- 需要决定数据集爬取的合理频率
- 在正式爬取前先做轻量级检查
- 定期追踪竞品平台的数据集增长趋势

## 输入要求

- 无必填输入，直接运行即可
- 可选参数：
  - `--pages N`：获取页数（默认 3 页，每页 50 条，共 150 条）
  - `--page-size N`：每页数量（默认 50）
  - `--dry-run`：只查看不保存快照
  - `--history`：查看上次快照信息
  - `--reset`：清除快照重新开始

## 执行步骤

1. 运行监控脚本，通过魔搭公开 API（`/api/v1/dolphin/datasets`）获取最新数据集列表。

```bash
python scripts/monitor.py
```

2. 脚本自动与上次保存的快照对比，将数据集分为三类：
   - **新创建**：创建时间在上次检查之后
   - **有更新**：修改时间在上次检查之后，但创建时间在之前
   - **新进入列表**：路径不在旧快照中

3. 根据时间间隔和新增数量，估算更新速率并给出爬取频率建议。

4. 保存当前数据集列表作为新快照（`scripts/snapshot.json`），供下次对比使用。

首次运行时无历史快照，脚本会将当前数据集列表作为基线保存。第二次运行时开始对比。

## 输出要求

输出包含以下结构化信息：

- **新增数据集数量**：两次调用之间新创建的数据集数
- **新增数据集详情**：名称、路径、创建时间、魔搭链接
- **有内容更新的数据集数量及详情**
- **更新频率估算**：
  - 平均更新速率（个/小时、个/天）
  - 爬取频率建议（每天、每 2-3 天、每周等）

## 核心机制

1. **API 调用**：使用魔搭公开 API 获取最新数据集列表（无需 Cookie 或认证）
2. **快照对比**：基于创建时间和修改时间的双维度对比
3. **频率估算**：根据新增速率自动推荐爬取频率
4. **持久化存储**：快照保存在 `scripts/snapshot.json`，跨会话持久

## 参考资料

本 Skill 无外部参考资料依赖。脚本使用 Python 标准库（`urllib`、`json`、`argparse`），无需安装额外依赖。

与数据集爬取类 Skill 配合使用时，推荐工作流：
1. 先用本 Skill 检查更新情况
2. 如果新增数据集较多，再调用爬取类 Skill 进行正式搬运
