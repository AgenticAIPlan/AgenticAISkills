# CSV格式规范

## 文件夹结构

每次采集任务创建独立文件夹，精确到秒：

```
工作目录/dataset_scout_YYYY-MM-DD_HH-MM-SS/
├── datasets.csv      # 数据集记录
└── TODO.md           # 任务进度记录
```

示例：`dataset_scout_2026-02-24_14-30-15/`

## 字段说明

| 字段名 | 说明 | 填写规则 |
|--------|------|----------|
| 是否搬运完成 | 标记数据集是否已处理 | **留空**（后续流程自动更新） |
| 数据集名称 | 数据集的名称 | 精确名称，如：xcopa |
| 来源链接 | 数据集来源页面链接 | 完整URL |
| 标签 | 数据集标签 | 多个标签用顿号分隔，最多5个，如：智能问答、逻辑推理 |
| 简介 | 数据集简介描述 | 简洁描述，通常1-2句话 |
| 开源协议 | 许可证类型 | 原始值，如：cc-by-4.0, mit, apache-2.0 |
| 大小 | 数据集大小 | 如：64.77 KB, 2.27 MB, 1.17 GB；**未显示则留空** |
| license映射 | license对应的数字ID | 自动映射，见license_mapping.md |
| 抓取渠道 | 数据来源渠道标识 | 填写实际平台名称 |

## CSV示例行

```csv
,xcopa,https://example-platform.com/datasets/xcopa,智能问答、逻辑推理、知识问答,XCOPA多语言数据集，用于跨语言因果常识推理评估。,cc-by-4.0,64.77 KB,2,AI数据集平台
```

## 大小处理规则

- **≤512GB**：正常登记到CSV
- **>512GB**：跳过，不登记
- **未显示大小**：留空，正常登记

## 大小获取方式

**重要**：大小信息在数据集详情页，不在列表页！必须点进每个数据集的详情页才能看到大小信息。
