# IP小说改编短剧/漫剧剧本生产 SOP

## Skill 边界与两阶段设计

本 skill 采用**两阶段**设计，避免单次触发过长输出：

### 阶段一（默认触发）：拆书分析 + 剧本策划

1. **拆书分析**：通读小说 → 整体分析（梗概/卖点/情感引擎/爽点/人物标记）→ 框架分析（人物关系/关键情节/弧光/主题）→ 确定集数 → 人物设定
2. **剧本策划**：剧本名称 → 集数 → 类型 → 梗概 → 人物小传 → 集纲
3. **输出策划案后须等待用户确认**，再决定是否进入阶段二

### 阶段二（需二次确认）：剧本正文创作

- 仅在用户明确要求创作剧本正文时启动
- 依据确认后的策划案，逐集创作
- 格式规范（△/括号/OS/VO/闪回等）、剧本符号说明详见 `references/novel-adaptation-sop.md`

> ⚠️ 若用户只需拆书分析、集纲或策划案，**不应自动拉向全量剧本正文创作**。

## 关键规则

1. **人物弧光是改编基石**：若原著人物弧光扁平，剧本需重构
2. **爽点分布要合理**：1-10 集密集铺排，每 10 集设置高潮
3. **钩子设置要强**：每集结尾设置悬念，引导持续追看
4. **次要人物需精简**：短剧中人物关系极简，次要人物须服务于主要人物
5. **集数与时长参考**：竖屏剧通常 60-80 集，单集 1-2 分钟对应 500-800 字

## 何时使用

**前提条件**：用户**已有小说/IP原文**，需要进行系统化的改编流程。

**适用场景**：
- "我有一部小说，想改编成短剧"
- "帮我把这本 IP 小说做改编拆书分析"
- "根据这部小说出一份短剧集纲"
- "已有小说原文，需要走短剧改编全流程"

**不适用场景**（不应触发）：
- 无原著/IP 的纯创作请求，如"帮我写一个短剧剧本"
- 通用策划咨询，如"短剧怎么策划"
- 单纯的剧本格式讨论

## 使用示例

### 示例 1：用户提供小说，要求改编

**用户**：我有一个小说，讲的是女主重生复仇的故事，想改编成 60 集短剧。

**助手**：
1. 通读小说，完成拆书分析（整体分析 → 框架分析 → 人物设定）
2. 输出剧本策划案（名称、类型、梗概、人物小传、集纲）
3. 询问用户："策划案已完成，是否需要进入剧本正文创作？"

### 示例 2：用户只需要集纲

**用户**：我有一部小说想做改编拆书，帮出个集纲就行。

**助手**：
1. 完成拆书分析
2. 输出集纲
3. 不自动进入剧本正文创作
