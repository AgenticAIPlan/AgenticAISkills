# Reviewer 终审清单

## Error：必须修复

- 正文使用了未准入事实。
- URL 未经链接审计。
- URL 返回 404、NotFound、Timeout、DNS Failed、跳首页或登录页，却仍在正文中使用。
- 页面不支持对应事实。
- 自检结果写“部分”并当作通过。
- 近期动态日期不合规却计入 5-10 条。
- 官网有机构设置或领导页，却使用预算文件或新闻拼图替代。
- 搜索结果为邮箱、OA、SSO、登录页、后台管理门户，却被当作信息公开官网。
- 基本介绍缺少成立/设立时间、性质/级别、主管部门或股东、核心定位、主要业务领域，且未说明缺口。
- 官网机构设置仅为图片时，未截图、未说明截图路径或未启动降级检索。
- 遇到 401/403/登录拦截后直接判定信息不存在，未记录拦截状态和降级检索路径。
- 把业务栏目、展区、项目写成行政内设机构。
- 把新闻中出现的部门或人物写成完整现行组织架构。
- 缺少原子事实证据表、链接审计表或事实准入表。
- 正文裸露长 URL，未使用 `[来源标题](URL)` Markdown 链接格式。
- 同一事实或段落使用了组合来源标签，例如 `「官网原文 / 总结提炼」`。
- 隶属关系图示只停在直接上级，且未说明无法继续上溯的原因。
- 最终附录顺序未按：附录 A 原子事实证据表、附录 B 候选来源清单、附录 C 链接审计表、附录 D 事实准入表、附录 E Reviewer 终审结果。

## Warning：必须提醒

- 负责人来自新闻拼图，不是官网领导页。
- 负责人只列姓名职务，未提供履历、分工、核心经历、近期公开活动，且未说明公开履历缺失。
- 内设机构只能部分确认。
- 下属及关联单位未找到完整清单。
- 近期动态包含第三方转载。
- 基本介绍来自上级部门或招聘公告，而非单位官网。
- 主要职责列表的职责名称未加粗，影响阅读但不改变事实可靠性。
- 原子事实证据表未按来源编号排序，降低人工复核效率。

## Info：可优化

- 可补充更多近期动态。
- 可补充负责人履历。
- 可补充关联单位图谱。
- 可补充会议主题相关部门排序。

## 终审结论

- 存在未修复 `error`：阶段性调研稿。
- 无 `error` 但有 `warning`：可正式报告，但必须保留风险提示。
- 无 `error` 且无关键 `warning`：正式报告。
