# 高质量输出示例导读

本文件说明 `org-research` 的高质量交付应如何判断。样例见：

- `references/example-nda-good-report.md`：基于“国家数据局”真实评测案例整理的正式报告节选样例。

## 为什么使用真实评测案例

真实评测案例比虚构示例更适合帮助执行者理解本 Skill 的质量标准：

- 它展示了官网、主管部门、关联组织等来源如何组合使用。
- 它展示了基本介绍、职责、机构设置、负责人、近期动态如何全部落到具体 URL。
- 它展示了附录 A/B/C/D/E 如何承接正文，形成可复核证据链。
- 它展示了非行政隶属关联如何写明关系类型，避免误写成直属下属。

## 阅读重点

阅读 `example-nda-good-report.md` 时，重点看以下结构，而不是照搬具体内容：

1. 基本介绍是否覆盖硬字段：成立时间、性质级别、主管部门、核心定位、主要业务领域。
2. 每条主要职责是否有具体页面来源，且职责名称是否加粗。
3. 隶属关系图示是否说明必要上溯的业务相关性。
4. 内设机构和负责人是否使用无序列表，并说明公开信息缺口。
5. 近期动态是否有 `YYYY-MM-DD` 日期和具体 URL。
6. 附录是否包含候选来源、原子事实、链接审计、事实准入和 Reviewer 终审结果。

样例为了控制 reference 体量只保留代表性表格行；正式交付时不应省略附录产物。

## 不应照搬的部分

- 不要把国家数据局的结构迁移到其他单位。
- 不要因为样例有完整资料，就假设所有单位都能生成正式报告。
- 如果目标单位公开资料不足，应降级为“阶段性调研稿”，并写清缺口。
