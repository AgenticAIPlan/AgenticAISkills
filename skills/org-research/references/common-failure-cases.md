# 常见失败案例

本文件用于 Reviewer 和报告撰写阶段对照排错。

## 1. 用首页支撑具体事实

错误：

```markdown
某单位设有综合部、业务部、技术部。来源：[某单位官网](https://example.com/)
```

问题：主站首页不能支撑具体机构设置。

修正：

- 回到候选来源收集，寻找机构设置、部门介绍、年报、招聘、预算等具体页面。
- 找不到时写“内设机构无法确认”或“部分确认”，不得补全。

## 2. 把新闻活动写成现行组织架构

错误：

```markdown
新闻中出现“数据治理中心负责人参加会议”，因此该单位现设数据治理中心。
```

问题：新闻只能证明报道时该名称出现，不能证明现行内设机构。

修正：

- 将该信息放入待核实线索。
- 只有机构设置页、正式介绍页、预算/年报等稳定来源支持时，才写入内设机构。

## 3. 把非行政关联写成直属下属

错误：

```markdown
某标准化技术委员会是该单位下属单位。
```

问题：标委会、专家委员会、工作组、联盟通常不是直属下属，必须确认关系类型。

修正：

```markdown
某标准化技术委员会为非行政隶属关联，关系类型为“秘书处承担单位”或“业务指导单位”。
```

## 4. 负责人只列姓名职务

错误：

```markdown
- 张某：院长
- 李某：副院长
```

问题：负责人章节缺少履历、分工、公开活动或缺口说明。

修正：

- 若官网有履历，简要概括履历。
- 若只找到姓名职务，逐人说明“未找到公开履历/分工信息”。
- 若来自新闻拼图，说明只能证明报道时身份，不能证明完整领导班子。

## 5. 近期动态无日期或无具体 URL

错误：

```markdown
近日，某单位推进数据平台建设。
```

问题：“近日”不能计入 5-10 条近期动态，且缺少具体 URL。

修正：

- 日期必须是 `YYYY-MM-DD` 或 `YYYY-MM-未查明日期`。
- 每条动态必须链接到具体新闻、公告、政策或通稿页面。

## 6. 正文使用未准入事实

错误：报告生成阶段临时补了一句“该单位可能负责行业标准制定”。

问题：正文只能使用事实准入表中 `准入` 的事实。

修正：

- 回到 Step 1 / Step 2 补来源和原子事实。
- 通过链接审计和事实准入后再写入正文。
- 若无法补证，放入待核实线索。

## 7. 上溯关系没有业务必要性说明

错误：

```text
最高层级机关
└── 主管部门
    └── 某单位
```

问题：若更高层级进入正文图示，必须说明其对会前沟通、合作判断、审批链条、资源协调或单位性质判断的业务必要性。

修正：

```markdown
业务相关性说明：上级主管关系会影响本次合作事项的审批链条和资源协调方式，因此进入图示。
```

## 8. 把 HTTP 200 当成事实准入

错误：

```markdown
链接可访问且返回 200，因此该事实可以写入正式报告。
```

问题：HTTP 200 只能说明链接在技术上可打开，不能说明页面内容支持对应事实，也不能替代事实准入判断。

修正：

- 在链接审计阶段记录链接状态、页面标题和页面相关性。
- 逐条核对页面是否支撑对应原子事实。
- 只有通过事实准入表标记为 `准入` 的事实，才能进入正式正文。
