# 文生图提示词指南 (Image Prompt Guide)

本文档提供 `ernie-image-turbo` 文生图模型的最佳提示词撰写实践，帮助生成高质量的世界杯主题图片。

---

## 核心原则

### 1. 使用中文提示词

`ernie-image-turbo` 对中文理解能力更强，**所有提示词必须使用中文撰写**。

### 2. 情绪方向决定视觉呈现

**⚠️ 同一事件，不同情绪方向的视觉呈现完全不同**

| 情绪方向 | 定义 | 视觉特点 | 色彩基调 |
|----------|------|---------|----------|
| `positive`（正向） | 喜悦、胜利、庆祝、骄傲 | 欢腾、光芒、热烈、明亮 | 金色、深红、翠绿 |
| `negative`（负向） | 悲伤、失落、遗憾、绝望 | 孤独、沉默、黯淡、冷色调 | 灰蓝、暗色、银白 |

**核心事件 × 情绪方向对照表**：

| 核心事件 | 正向视角（positive） | 负向视角（negative） |
|----------|---------------------|---------------------|
| 绝杀进球 | 金色光芒、胜利姿态、欢腾人群 | 孤独背影、沉默球场、黯淡灯光 |
| 帽子戏法 | 庆祝动作、聚光灯、金色数字"3" | 无力回天、空旷球门、绝望眼神 |
| 点球大战 | 拥抱庆祝、晋级横幅、惊险逃生 | 孤独背影、空旷球门、雨滴 |
| 夺冠时刻 | 举起奖杯、金色雨、国旗飘扬 | 遗憾落败、低头沉默、黯淡灯光 |

**示例对比**：

```

灰蓝色渐变背景，黯淡的月光洒落，
画面中央呈现空旷的球门和孤独的背影，低头沉默，
```

### 3. ⚠️ 文生图无法精确渲染文字

**重要限制**：文生图模型无法精确渲染文字内容。

**解决方案**：
- **方图 (1:1)**：生成纯视觉图片，不含文字
- **长图 (9:16)**：生成纯背景图片，微小说文字通过代码（Pillow/PIL）叠加到图片上

**长图提示词注意**：
- 不要在提示词中要求嵌入文字
- 生成的图片应作为背景，为文字叠加预留简洁空间
- 确保背景色调与文字颜色形成高对比（如深色背景配白色文字）

### 4. 结构化提示词

按照固定结构组织提示词，确保信息完整：

```
[情绪方向关键词] + [核心事件视觉元素] + [主体描述] + [背景环境] + [色彩基调] + [风格氛围] + [技术参数]
```

### 5. 具体而非抽象

避免模糊描述，使用具体、可感知的词汇：

| ❌ 不推荐 | ✅ 推荐 |
|----------|--------|
| 美丽的背景 | 深蓝色渐变背景，点缀金色光斑 |
| 有氛围感 | 黄昏时分的体育场，灯光穿透薄雾 |
| 简约风格 | 极简构图，大面积留白，中央对齐 |
| 体现比赛 | 球员射门瞬间的剪影，皮球划出金色弧线 |

## 文字叠加指南（长图专用）

⚠️ **重要**：文字叠加是必须执行的步骤，不可跳过！

### 叠加流程

```
Step 1: 文生图生成纯背景图片（ernie-image-turbo）
    ↓
Step 2: 下载生成的图片
    ↓
Step 3: 根据背景主色调自动选择文字颜色（白/黑）
    ↓
Step 4: 使用 Pillow 叠加微小说文字
    ↓
Step 5: 保存最终图片并验证文字可读性
```

### 文字颜色自动判断

根据背景主色调亮度自动选择文字颜色：

| 背景亮度 | 文字颜色 | RGB 值 |
|---------|---------|--------|
| < 128（深色背景） | 白色 | `(255, 255, 255)` |
| ≥ 128（浅色背景） | 黑色 | `(0, 0, 0)` |

```python
def calculate_brightness(hex_color: str) -> float:
    """计算颜色亮度（0-255）"""
    hex_color = hex_color.lstrip('#')
    r, g, b = tuple(int(hex_color[i:i+2], 16) for i in (0, 2, 4))
    return (r * 299 + g * 587 + b * 114) / 1000

# 使用示例
brightness = calculate_brightness("#1a1a2e")  # 深紫色背景
text_color = (255, 255, 255) if brightness < 128 else (0, 0, 0)
```

### Pillow 完整叠加代码

```python
from PIL import Image, ImageDraw, ImageFont
import requests
from io import BytesIO
import textwrap

def overlay_text_on_image(
    image_url: str,
    novel_text: str,
    dominant_color_hex: str,
    output_path: str = "/tmp/final_9x16.png"
) -> str:
    """
    在长图上叠加微小说文字

    Args:
        image_url: 背景图片URL
        novel_text: 微小说原文（约300字）
        dominant_color_hex: 背景主色调（如 "#1a1a2e"）
        output_path: 输出路径
    """
    # 下载图片
    response = requests.get(image_url)
    img = Image.open(BytesIO(response.content))
    draw = ImageDraw.Draw(img)

    # 根据背景亮度选择文字颜色
    brightness = calculate_brightness(dominant_color_hex)
    text_color = (255, 255, 255) if brightness < 128 else (0, 0, 0)

    # 字体设置（尝试多种路径）
    font_size = 24
    font_paths = [
        "/System/Library/Fonts/PingFang.ttc",
        "/usr/share/fonts/truetype/noto/NotoSansCJK-Regular.ttc",
        "/Windows/Fonts/msyh.ttc",
        "NotoSansSC-Regular.otf",
    ]
    font = None
    for fp in font_paths:
        try:
            font = ImageFont.truetype(fp, font_size)
            break
        except:
            continue
    if font is None:
        font = ImageFont.load_default()

    # 排版参数
    margin = 50
    line_height = int(font_size * 1.8)
    chars_per_line = 25

    # 自动换行
    lines = textwrap.wrap(novel_text, width=chars_per_line)

    # 绘制文字
    y = margin
    for line in lines:
        draw.text((margin, y), line, font=font, fill=text_color)
        y += line_height

    # 保存
    img.save(output_path)
    print(f"✅ 文字叠加完成: {output_path}")
    print(f"   文字颜色: {'白色' if text_color == (255,255,255) else '黑色'}")
    return output_path
```

### 排版参数标准

| 参数 | 标准值 | 说明 |
|------|--------|------|
| 字号 | 24px | 适合300字篇幅 |
| 行间距 | 1.8倍字号 | 阅读舒适 |
| 边距 | 50px | 四周留白 |
| 每行字符 | 25个中文字 | 确保排版整齐 |

### 推荐字体

| 字体 | 特点 | 适用场景 |
|------|------|----------|
| 思源黑体 / PingFang SC | 清晰易读，现代感 | 正文、现代风格 |
| 思源宋体 | 典雅，文学感 | 诗意、怀旧风格 |
| 微软雅黑 | Windows 默认 | 正式、新闻风格 |

### 常见问题修复

| 问题 | 解决方案 |
|------|---------|
| 文字不可见 | 检查背景亮度，切换文字颜色（白↔黑） |
| 字体缺失 | 使用代码中的字体路径列表，或降级使用默认字体 |
| 文字溢出 | 减小字号至20px或增加每行字符数至30 |
| 排版混乱 | 使用 `textwrap.wrap()` 确保自动换行 |

## 常见问题与解决方案

### 问题1：文字渲染不清晰

**解决方案**：
- 在提示词中明确要求"文字清晰可读"、"高对比度"
- 指定背景与文字的颜色对比（如"深色背景，白色文字"）
- 避免复杂的背景图案干扰文字

### 问题2：风格不符合预期

**解决方案**：
- 增加风格关键词的权重，如"极简风格为主"
- 参考本指南的关键词分类表，使用更精准的词汇
- 添加"避免：[不想要的元素]"明确排除

### 问题3：画面过于复杂

**解决方案**：
- 使用"极简"、"留白"、"干净背景"等关键词
- 明确主体位置（如"画面中央"、"顶部留白"）
- 减少描述性词汇，保留核心元素

### 问题4：情绪表达不准确

**解决方案**：
- 使用本指南的氛围关键词组合
- 结合具体场景描述（如"黄昏"、"雨天"、"聚光灯"）
- 添加感官描述（如"温暖的光线"、"冰冷的色调"）

### 问题5：无法体现核心事件

**解决方案**：
- 参考本指南的"核心事件视觉元素对照表"，选择合适的视觉元素
- 在提示词中明确描述事件相关的视觉符号（如"举起奖杯"、"球网震颤"）
- 添加事件关键词（如"绝杀时刻"、"点球失利"）
- 确保画面主体与事件性质相符（如胜利→庆祝姿态，失利→孤独背影）

## 注意事项

1. **提示词长度**：建议控制在 200-400 字符以内，过长可能导致信息稀释
2. **避免冲突**：不要同时使用相互矛盾的关键词（如"极简"和"丰富细节"）
3. **迭代优化**：首次生成效果不佳时，调整关键词权重或添加具体描述
4. **文化敏感**：避免使用可能引起争议的政治、宗教相关意象
5. **严禁内容**：
   - 可能引起争议的政治、宗教相关意象
   - 低俗、违法或不符合公序良俗的内容
   - 近现代名人肖像（三元组中确认的球员除外）
