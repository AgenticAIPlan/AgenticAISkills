# 文案写法风格

## 文案结构

```
[Emoji] [明确包含核心信息的标题]

[简短介绍，1-2句]

📊 Key Highlights:
• [具体数据点1]
• [具体数据点2]
• [技术特点3]
• [对比优势4]

[补充段落，如架构说明、应用场景]

🔗 Read more: [链接]

#标签
```

## 标题写法

| 要求 | 示例 | 避免 |
|------|------|------|
| 包含产品名 | PaddlePaddle CINN Enables... | 重磅发布 |
| 包含技术点 | Plug-and-Play Support | 新功能 |
| 核心价值前置 | 2.2x Faster Inference | 性能提升 |

## 分点符号

使用简洁的 `•` 符号，每点包含：
- **具体数字**：2.2x、27.4%、115%
- **对比优势**：faster than PyTorch
- **技术术语**：dynamic shape、three-layer decoupling
- **应用场景**：training & inference、scientific computing

## Emoji 使用

| 位置 | 用量 | 示例 |
|------|------|------|
| 标题前 | 1个 | 🚀 |
| 段落标题前 | 1个 | 📊、🏛️、🖼️ |
| 句尾点缀 | 1-2个 | 🛠️⚡ |
| 分点前 | 0个 | 避免 ❌ |

## 性能数据写法

- **推理加速**：`X times faster` 或 `X% speedup`
- **训练加速**：`X% faster` 或 `X% speedup`
- **对比优势**：`X% faster than [竞品]`
- **架构特性**：`three-layer decoupling`、`plug-and-play`

## 链接写法

- 文末统一使用：`🔗 Read more: [链接]`
- 如果有多个链接，分行列出
- 确保链接完整可访问

## 标签写法

```
#PaddlePaddle #DeepLearning #AI [技术标签] [合作方标签]
```

- 基础标签：#PaddlePaddle #DeepLearning #AI
- 技术标签：#CINN、#OCR、#HardwareAccelerator、#Compiler
- 合作方：#MetaX、#TianDays（如有）
- 不超过 4-6 个标签

## 常用词汇对照

| 中文 | 英文 |
|------|------|
| 零侵入 | zero code intrusion |
| 即插即用 | plug-and-play |
| 解耦 | decoupling |
| 端到端 | end-to-end |
| 动态形状 | dynamic shape |
| 科学计算 | scientific computing |
| 微分方程 | PDE (Partial Differential Equation) |
| 显存 | device memory |
| 编译器 | compiler |
| 算子 | operator / kernel |
