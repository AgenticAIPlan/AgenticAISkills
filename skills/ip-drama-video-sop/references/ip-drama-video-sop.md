# IP小说改编短剧全流程 SOP 详细说明

本文档提供 IP 小说改编短剧的全流程详细说明，涵盖从拆书分析到AI短视频生成的完整7步 SOP。

**核心原则**：每一步 AI 产出都须用户确认后才进入下一步。

## 核心流程表

| **环节** | **步骤** | **是否必需项** | **核心要素** | **确认节点** |
|-|-|-|-|-|
| 拆书 step1 | 通读小说 | ✅ | 通读全篇小说/前100章 | |
| | 小说整体分析 | ✅ | 梗概+卖点/情感引擎/名场面/爽点/人物标记 | |
| | 小说框架分析 | ✅ | 人物关系/关键情节/弧光/主题 | |
| | 确定集数与情节点 | ✅ | 参考标准：竖屏剧60-80集，轻量AI视频10-20集 | |
| | 人物设定及关系 | ✅ | 主要人物+次要人物（精简） | ✋ 用户确认 |

### 轻量级短剧 vs 传统平台短剧

| 维度 | 轻量级短剧（AI视频） | 传统平台短剧 |
|------|---------------------|-------------|
| 集数 | **10-20集** | 60-80集 |
| 总时长 | 10-40分钟 | 60-160分钟 |
| 人物 | 2-3个核心角色，无支线 | 多角色+支线 |
| 节奏 | 每集直接推进主线+爽点 | 一卡/二卡/三卡递进 |
| 适用 | AI视频生成、快速验证 | 平台付费短剧 |

## Step 1：拆书分析

### 1.1 通读小说 ✅

**目标**：全面理解小说的故事结构、人物关系和核心主题。

**操作要点**：
- 通读全篇小说，或至少通读前 100 章（通常前 100 章为精华部分）
- 记录重要情节点、人物关系、名场面
- 初步判断小说的核心卖点和情感引擎

---

### 1.2 小说整体分析 ✅

**目标**：提炼小说的核心要素，为改编剧本奠定基础。

#### 小说梗概 + 核心卖点

**小说梗概**：用一段话概括故事主线

**示例**：
> 女主作为豪门真千金被找回后，受尽假千金和家人的欺凌，最终含恨而死。重生后，她放弃讨好家人，转而凭借前世记忆疯狂搞事业，并联手幕后大佬对抗家族。

**核心卖点**：用一句话提炼最抓人、最具市场吸引力的核心卖点

**示例**：
> "重生归来，我让整个豪门跪地求饶"

#### 情感引擎

**定义**：明确主导情绪（复仇、甜宠、逆袭、爽文、悬疑等）

**示例**：
- 复仇：女主重生复仇，打脸渣男渣女
- 甜宠：霸道总裁爱上小助理，甜蜜互动
- 逆袭：屌丝逆袭成功，走上人生巅峰
- 爽文：主角开挂，一路碾压反派

#### 名场面清单

**定义**：列出原著中必保留的"高光时刻"和"经典台词"

**示例**：
- 女主在宴会上揭穿假千金的真面目
- 男主霸道宣言："她是我的女人，谁敢动她？"
- 女主冷笑："当初你们不要我，现在我也不需要你们"

#### 爽点提炼与分类

**定义**：系统性地梳理和分类原著中的"爽点"（如打脸、逆袭、发糖、碾压、反转），并规划其在不同剧集中的分布，确保节奏紧凑，持续吸引观众。

**爽点分类**：
- **打脸**：女主用实力打脸反派
- **逆袭**：女主从底层逆袭成功
- **发糖**：男女主甜蜜互动
- **碾压**：女主碾压反派，展现强大实力
- **反转**：剧情出现意外反转，观众意料之外

**分布原则**：
- 1-10 集：密集铺排爽点，快速立人设
- 11-60 集：每 10 集设置一个高潮，保持观众追看欲望
- 每集结尾：设置钩子/悬念，引导观众继续观看

#### 人物标记

**定义**：提炼原著中主要/次要人物，特别关注主要人物的情感转折点，以此标记关键情节点。

**示例**：
- **女主**：情感转折点 - 重生后放弃讨好家人 → 决心搞事业 → 联手男主对抗家族
- **男主**：情感转折点 - 对女主产生兴趣 → 暗中帮助女主 → 公开宣布女主是自己的女人
- **反派**：情感转折点 - 对女主嫉妒 → 设计陷害女主 → 被女主打脸

---

### 1.3 小说框架分析 ✅

**目标**：深入分析小说的结构和主题，为改编剧本提供框架支撑。

#### 人物关系与背景

**定义**：构建角色互动的情感网络与世界观基底，明确故事发生的逻辑前提。

**示例**：
- **女主**：豪门真千金，被找回后受尽欺凌，重生后决心复仇
- **男主**：神秘大佬，暗中帮助女主
- **假千金**：抢占女主位置，嫉妒女主
- **豪门家族**：重男轻女，偏心假千金

#### 关键情节点梳理

**定义**：筛选驱动叙事的核心事件（如冲突/反转），串联起改编剧本的骨架与节奏。

**示例**：
- **起点**：女主被找回豪门
- **冲突**：女主受尽欺凌，最终含恨而死
- **转折**：女主重生，决心复仇
- **高潮**：女主揭穿假千金，联手男主对抗家族
- **结局**：女主成功复仇，与男主在一起

#### 人物弧光/成长线

**定义**：人物在故事进程中，因经历事件、解决冲突产生的性格、认知、立场等方面的成长或转变。

**⚠️ 人物弧光是改编基石**——若原著塑造出色，剧本则锦上添花；若原著弧光扁平，剧本需重构。

**示例**：
- **女主弧光**：软弱讨好 → 重生后冷静理性 → 强势复仇 → 最终找到自我价值
- **男主弧光**：冷漠疏离 → 对女主产生兴趣 → 为女主改变 → 全心全意爱女主

#### 故事主题

**定义**：提炼原著核心立意与价值观（如成长、救赎、坚守等），决定改编后的情感共鸣点与市场定位。

**示例**：
- 主题：重生复仇，找回自我价值
- 立意：不向命运低头，用实力打脸所有人
- 价值观：女性独立自强，不依附他人

---

### 1.4 确定集数与情节点 ✅

**参考标准**：竖屏剧通常单集 1-2 分钟，总集数 60-80 集。

**核心关注点**：
- 原著亮点与改编难度
- 针对小说多元素混杂的特性，需结合竖屏剧市场规律，提取并深化核心亮点

**操作要点**：
- 根据情节密度决定最终集数
- 每集 1-2 分钟，对应 500-800 字剧本
- 每 10 集设置一个高潮，保持观众追看欲望

---

### 1.5 人物设定及关系 ✅

#### 主要人物

**必需输出**：
- 核心人设
- 人物弧光/成长线
- 标志性动作或台词（如有）

**示例**：
> **女主 - 沈枝熹**：豪门真千金，重生后冷静理性，决心复仇。标志性台词："当初你们不要我，现在我也不需要你们。"

#### 次要人物

**原则**：
- 通常只沿用次要人物原著姓名
- 该人物数量需精简，避免过度挖掘
- 短剧中人物关系极简，次要人物须服务于主要人物，其核心功能为推动主要人物情感线发展

**示例**：
> **鸳鸯**：女主的贴身侍女，忠心耿耿，推动女主情感线发展。

## Step 2：剧本策划

### 2.1 剧本名称 ✅

**原则**：通常为一句话最抓人的卖点

**示例**：
- 《重生归来，我让整个豪门跪地求饶》
- 《霸道总裁的小娇妻》
- 《逆袭人生，我才是真千金》

---

### 2.2 剧本集数 ✅

**格式**：xx 集

**示例**：60 集、80 集

---

### 2.3 剧本类型 ✅

**原则**：明确主导情绪定剧本类型

**示例**：
- 复仇
- 甜宠
- 逆袭
- 爽文
- 悬疑

---

### 2.4 其他策划内容（可选）

**可选项**：
- **时代背景**：通常年代剧必选
- **核心冲突**：女主 vs 假千金、女主 vs 豪门家族
- **核心卖点**："重生归来，我让整个豪门跪地求饶"
- **禁忌**：不涉及政治、宗教、暴力等敏感内容
- **付费卡点设计**：每 10 集设置一个高潮，设置付费卡点

---

### 2.5 故事梗概 ✅

**核心包含**：
- **开篇钩子**：用极具画面感的场景开场，立即抓住注意力
- **主角介绍**：清晰描述主角的身份、处境和渴望
- **核心冲突**：阐述主角面临的主要困境和对手
- **金手指设定**：说明主角解决危机的独特能力
- **故事走向**：概括主要剧情发展和转折点
- **结尾悬念**：留下让人想继续看的悬念

**示例**：
> **开篇钩子**：沈府院内，沈枝熹躺在贵妃椅上，大夫惊呼："沈小姐，您有孕了！"
>
> **主角介绍**：沈枝熹，豪门真千金，重生后决心复仇。
>
> **核心冲突**：女主 vs 假千金、女主 vs 豪门家族。
>
> **金手指设定**：女主拥有前世记忆，知道所有人的秘密。
>
> **故事走向**：女主揭穿假千金，联手男主对抗家族，最终成功复仇。
>
> **结尾悬念**：女主怀孕，男主是谁？

---

### 2.6 人物小传 ✅

#### 一般必含项

- **基本信息**：姓名、性别、年龄
- **身份背景**：社会地位、家庭出身、成长经历
- **核心事件**：打破人物平衡状态的关键冲突，确立其在剧中的核心任务

#### 选填项（视角色核心设定或编剧风格选填）

- **性格特点**：3-5 个核心标签
- **内心渴望**：表面目标 vs 深层需求
- **致命弱点**：性格缺陷或心理创伤
- **人物弧光**：在故事中的成长变化轨迹
- **人物关系**：与其他角色的关系网
- **经典台词**：3-5 句符合人物性格的标志性台词

#### 示例

> **（基本信息）** 张婆：女，70 岁，
>
> **（身份背景+形象）** 独居念旧的慈祥老人，
>
> **（核心事件）** 因陈青山推进并村激烈反对，视老房为"老伴"的寄托。年轻时嫁给了村里的一位朴实憨厚的男人，两人相濡以沫，走过了无数个贫困的岁月。随着岁月流逝，丈夫因病去世，留下她一人守着老屋和一份家园的记忆。

---

### 2.7 剧本集纲 ✅

**核心包含**：
- **每集故事走向及核心事件**（如人物出场、情绪节奏、爽点设计、钩子设置）
- 通过集纲可以看出从一集开头至结尾的核心主线、亮点
- 可以参考起承转合，把故事拆分为四阶段来阐述

#### 集纲要素

- **集数标题**：简洁有力的标题
- **核心事件**：本集发生的主要剧情
- **情绪节奏**：憋屈 → 憋屈+ → 爆发爽的层级设计
- **爽点设计**：本集的情绪爆点（打脸、反转、高光）
- **钩子设置**：结尾留下的悬念
- **人物出场**：本集出现的主要角色

#### 一卡/二卡/三卡说明

**⚠️ 注意**：
- **一卡（1-10 集）**：开篇破局，极速立人设、密集铺排爽点，利用强情绪钩子完成观众"入坑"转化，确立追看基础
- **二卡/三卡（11-60 集）**：矛盾层层递进，并在每一卡结尾设置强悬念三角形（付费卡点），利用"期待感"驱动用户持续追看及付费

#### 示例

> **第 1 集：惊世喜脉**
> - **核心事件**：女主被诊断出有孕，全府震惊
> - **情绪节奏**：平静 → 震惊 → 憋屈
> - **爽点设计**：女主冷静应对，展现重生后的理性
> - **钩子设置**：孩子的父亲是谁？
> - **人物出场**：女主、鸳鸯、大夫、侍女

## Step 3：剧本正文创作

### 3.1 剧本格式规范 ✅

#### 基本信息（每集必须标注）

- **集数**：每集开头标注第 X 集
- **场号**：场景的序号（如 1-1 代表第一集第一个场景，1-2 代表第一集第二个场景）
  - 同一集中，更换不同场景，就得用不同的场号
  - 更换场景时需要空一行
- **时间**：白天/晚上，在剧本中记为 日/夜
- **环境**：内景/外景，在剧本中记为 内/外
- **地点**：场景的具体地点（如办公室/茶水间等）
- **人物**：这个场景中出现的主要人物

#### 剧本符号

- **△**：除台词外，能通过镜头演绎出来的部分（如人物动作、连贯的场景描写等）
- **( )**：括号一般伴随着人物台词，括号中的内容是指画面中人物的语气、表情、神态、情绪、动作等提示，并且一定伴随着人物讲这句台词时，同时发生的表情或动作
  - 示例：人名(冷笑)：台词内容 xxx
  - 如果不是同时发生的，而是做完动作以后再去说话的，则需用 △ 来代表演绎的部分
- **OS**：角色心理活动。记为：人名（OS）：心理台词内容
  - 注：OS 同时又有表情动作的，需先描述表情动作再描述 OS
  - 示例：柯南（勾唇 OS）：凶手就是那个人！
- **VO**：画外音。人物未出现，声音出现（如：接电话/广播里的声音）
- **闪回/闪出**：角色进入到回忆画面内容的标注

#### 格式示例

```
第 X 集：

场次 X：[日/夜 - 室内/室外 - 地点]
人物：[角色1]、[角色2]、[角色3]

△ [简洁的动作/场景描写，不超过2行]

角色名：（台词，需体现性格）
△ [伴随动作/反应]
```

#### 实际示例

```
第一集：

1-1：日 外 沈府院内
人物：沈枝熹、鸳鸯、大夫、侍女若干

△ 沈府，满院上下忙忙碌碌却各司其职，一阵风吹起院中樱花树上的风铃，花瓣落下，只见沈枝熹一身华服，躺在树下的贵妃椅上，旁边放着茶水和精致点心。

△ 鸳鸯立在沈枝熹身后，旁边一众侍女随侍，大夫紧张的在替沈枝熹把脉。

大夫：（擦着汗水，突然瘫倒在地）沈.....沈小姐......我知你还待字闺中，但.....这竟然是喜脉，您.....好像有孕了！

沈枝熹：什么！（站起来）

△ 大夫颤颤巍巍的跪下来。
```

---

### 3.2 字数与时长标准

- 1-2 分钟/集 对应 500-800 字
- 2-3 分钟/集 对应 1000 字以内
- 按此标准，一部 80 集的短剧总剧本字数约为 **8 万字**

---

### 3.3 创作建议

- 依据"一/二/三卡"剧本结构，覆盖第 1 集至 60 集（或更多）全量内容
- 遵循"集数-场次-人物-△xxx"格式推进创作，确保叙事连贯
- 每集结尾设置钩子/悬念，引导用户继续观看下一集

---


## Step 4：分镜头脚本制作

### 4.1 剧本场景拆解 ✅

**目标**：将剧本正文按场景逐一拆解，识别每个独立拍摄单元。

**操作要点**：
- 以剧本中的"场次"标识（如 1-1、1-2）为天然分界
- 每次场景切换（地点/时间/环境变化）对应一条新的分镜记录
- 同一场次内，根据情节需要可拆分为多个镜头

**剧本到分镜的拆解原则**：
- △ 标记的动作/场景描写 → 直接转化为镜头画面
- 角色台词 → 对应镜头的台词/字幕列
- ( ) 内的表情/动作提示 → 纳入镜头画面描述
- OS/VO → 在配音列标注，画面可设计对应视觉表达

---

### 4.2 景别与镜头设计 ✅

**目标**：为每个镜头选择合适的景别和镜头运动方式，增强视觉表现力。

#### 景别功能与适用场景

| 景别 | 功能 | 适用场景 | 示例 |
|------|------|---------|------|
| **远景** | 视野宽阔，以景抒情 | 开场环境交代、转场 | "△ 沈府，满院忙碌" |
| **全景** | 看清全貌，突出人物关系 | 多人互动、群戏 | 宴会场景、对峙场景 |
| **中景** | 表现力强，常用于叙事 | 日常对话、动作推进 | 两人对话、走动交互 |
| **近景** | 看清神态，传递情绪 | 情感爆发、关键反应 | 角色震惊、愤怒、冷笑 |
| **特写** | 专注细节，洞察心理 | 美食/物品展示、情绪高潮 | 手部动作、眼泪、关键道具 |

**景别组合技巧**：
- 开场：远景→全景（交代环境→引入人物）
- 对话：中景→近景交替（叙事→情绪递进）
- 高潮：近景→特写（情绪推向极致）
- 转场：远景（以景过渡）

#### 镜头运动方式

| 运动方式 | 效果 | 适用场景 |
|---------|------|---------|
| **推** | 靠近主体，聚焦重点 | 强调角色反应、揭示关键信息 |
| **拉** | 远离主体，展现环境 | 从细节回到全局、情绪释放 |
| **摇** | 左右/上下转动，展现空间 | 展示环境全貌、跟随视线 |
| **移** | 平行移动，跟随主体 | 跟随角色走动、展现空间关系 |
| **跟** | 紧跟运动中的主体 | 追逐、逃跑、急行 |
| **升/降** | 垂直运动，改变视角 | 俯瞰全局、仰拍显气势 |
| **固定** | 静态构图，稳定叙事 | 对话场景、情绪凝视 |

### 4.3 时长与台词匹配 ✅

**核心原则**：1秒镜头 ≈ 4-6字台词

**时长计算参考**：
| 台词字数 | 建议最短时长 |
|---------|-------------|
| 10字以内 | 2秒 |
| 10-20字 | 3-4秒 |
| 20-40字 | 5-7秒 |
| 40-60字 | 8-10秒 |

**注意事项**：
- 轻量级短剧单集1-2分钟（60-120秒），分镜总时长须匹配
- △ 动作描写需要独立时长，不可与台词重叠计算
- 留出情绪停顿和音乐过渡的时间（每场约1-2秒缓冲）
- 每集结尾预留1-2秒悬念留白

## Step 5：角色形象设定

### 5.1 角色概念图生成 ✅

**目标**：确定视频的整体美术风格和主角形象。

**操作流程**：
1. 根据剧本中人物小传描述，构建角色视觉 Prompt
2. 生成3张不同风格的角色概念图（如：皮克斯风格/吉卜力风格/写实风格）
3. 用户选择一张作为基础风格

**角色 Prompt 模板**：
```
[角色描述]，[服装/造型描述]，[场景氛围]，[艺术风格]，16:9比例
```

**示例**：
```
一位身着华服的古代贵女子，气质高贵冷艳，躺在樱花树下的贵妃椅上，皮克斯3D动画风格，色彩明亮，光线柔和，16:9
```

**角色一致性策略**：
- 将用户选定的角色图作为参考图（reference image）
- 后续所有分镜生成时，将参考图作为风格/角色锚点
- 在 Prompt 中保持核心角色描述不变，仅更换动作和场景

## Step 6：分镜图片生成

### 6.1 逐镜生成画面 ✅

**目标**：依据分镜头脚本和选定的角色风格，批量生成所有镜头的画面。

**操作流程**：
1. 遍历分镜表格的每一行
2. 将"镜头画面"列的内容转化为生图 Prompt
3. 结合角色参考图和风格描述，生成对应画面
4. 用户可逐张确认/修改/重新生成

**生图 Prompt 构建模板**：
```
[基础风格Prompt]，[角色动作/表情]，[场景描述]，[景别对应构图]，16:9
```

**景别与构图对应**：
| 景别 | 构图指导 |
|------|---------|
| 远景 | 大场景，人物渺小，环境为主 |
| 全景 | 人物全身可见，环境清晰 |
| 中景 | 人物腰部以上，背景适度 |
| 近景 | 人物肩部以上，表情清晰 |
| 特写 | 局部放大，细节突出 |

**角色一致性保障方案**：
1. **参考图锚定**：使用角色设定图作为风格参考（如 --cref / image reference）
2. **固定角色描述**：每条 Prompt 中保持核心角色描述（外貌/服装/风格）不变
3. **场景连贯**：相邻镜头使用前一镜头图片作为场景连贯参考
4. **批量检查**：生成后逐张对比角色形象，不一致的重新生成

## Step 7：AI短视频生成

> ⚠️ 本步骤涉及AI模型调用和资源成本，需用户明确确认后才执行。

### 7.1 TTS配音生成 ✅

**目标**：将分镜台词转为语音，时长与镜头时长匹配。

**操作要点**：
- 按镜头逐条生成语音
- 语速调整：确保语音时长不超过对应镜头时长
- 情绪匹配：根据台词内容和( )中的情绪标记选择合适的语音风格
- OS（内心独白）使用内敛语调，正常台词使用对话语调

---

### 7.2 分镜视频合成 ✅

**目标**：将每张分镜图片与对应语音合成为短视频片段。

**技术方案**：
- 图片按镜头时长显示
- 叠加对应语音轨道
- 镜头间添加简单过渡效果（如淡入淡出）

**时长控制**：
- 无台词的镜头：按分镜表格中的时长直接设定
- 有台词的镜头：以语音实际时长为准（不超过表格时长）
- 镜头间缓冲：0.3-0.5秒过渡

---

### 7.3 配乐选择 ✅

**目标**：为视频添加背景音乐，提升观感。

**音乐选择原则**：
- 根据剧本类型（复仇/甜宠/逆袭等）确定整体音乐基调
- 根据情绪节奏调整音乐强度：憋屈段低沉→爆发段激昂
- 背景音乐音量不压过台词和配音
- 推荐免版权音乐库（如新片场等）

**AI配乐辅助**：
- 用情绪关键词（如"upbeat revenge"、"gentle romance"）搜索匹配音乐
- 推荐3-5首候选，用户试听选择

---

### 7.4 最终视频串联 ✅

**目标**：将所有片段合成为最终视频文件。

**合成流程**：
1. 按镜号顺序拼接所有分镜视频片段
2. 叠加背景音乐（音量调低至不干扰配音）
3. 添加字幕（可选，与台词同步）
4. 渲染输出为 MP4 文件

**输出标准**：
- 分辨率：1080×1920（竖屏）或 1920×1080（横屏）
- 格式：MP4
- 时长：匹配单集1-2分钟标准
