# 飞桨星河社区项目评审 - 回复模板库

本模板库提供各种场景下的评审回复模板，可根据实际情况调整使用。

---

## 模板 1：完整评审报告（推荐）

适用于需要进行全面评审的场景。

```
=== 飞桨星河社区项目评审报告 ===

项目名称: <项目名称>
项目URL: <项目URL>
评审时间: <时间>

【综合评分】
总分: <分数>/10
项目等级: <优秀/良好/一般/较差>

【各维度评分】
- 创新性: <分数>/10 (权重25%)
- 技术深度: <分数>/10 (权重25%)
- 文档质量: <分数>/10 (权重20%)
- 可复现性: <分数>/10 (权重20%)
- 社区价值: <分数>/10 (权重10%)

【AI 生成内容检测】
检测结果: <无/少量/部分/大量>
减分调整: <0%/0%/15%/30%>

【详细评审】

创新性评估:
<评分理由>

技术深度评估:
<评分理由>

文档质量评估:
<评分理由>

可复现性评估:
<评分理由>

社区价值评估:
<评分理由>

【项目亮点】
1. <亮点1>
2. <亮点2>
3. <亮点3>

【主要不足】
1. <不足1>
2. <不足2>
3. <不足3>

【改进建议】
1. <建议1>
2. <建议2>
3. <建议3>

【加精建议】
<建议加精/可考虑加精/暂不建议加精>

理由: <简述理由>
```

## 模板 2：快速评审（简短版）

适用于快速浏览多个项目时的快速评审。

```
【快速评审】
项目: <项目名称>
URL: <项目URL>
评分: <分数>/10

亮点: <1-2句话描述亮点>
不足: <1-2句话描述不足>
建议: <1句话建议>
```

## 使用建议

1. **选择合适的模板**：根据项目质量和评审目的选择合适的模板
2. **调整语气**：根据情况调整语气的正式程度和鼓励程度
3. **具体化**：尽可能使用具体的例子和描述
4. **建设性**：即使是负面评审，也要提供可操作的改进建议
5. **个性化**：在模板基础上加入个人的观察和观点

## 常用评分等级术语

| 等级 | 分数范围 | 推荐模板 | 说明 |
|------|----------|----------|------|
| 优秀 | 8.0-10.0 | 模板3、6 | 质量高，建议加精 |
| 良好 | 6.0-7.9 | 模板1 | 质量较好，有改进空间 |
| 一般 | 4.0-5.9 | 模板5、8 | 需要较多改进 |
| 较差 | 0-3.9 | 模板4、10 | 需要大幅改进，或鼓励重新开始 |
