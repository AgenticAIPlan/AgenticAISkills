# 飞桨星河社区项目评分标准模板

## 评分维度详解

### 1. 创新性（权重 25%）

#### 评分标准
| 分数段 | 标准 | 示例 |
|--------|------|------|
| 9-10分 | 突破性创新 | 提出全新算法、解决行业难题、首创应用场景 |
| 7-8分 | 显著创新 | 方法创新、技术融合、跨领域应用创新 |
| 5-6分 | 中等创新 | 应用场景新颖、功能设计创新、用户体验创新 |
| 3-4分 | 微创新 | 小改进、优化实现、借鉴后微调 |
| 0-2分 | 无创新 | 完全照搬现有项目、无任何新意 |

#### 评估要点
- [ ] 是否提出了新的想法或方法？
- [ ] 是否解决了现有技术难以解决的问题？
- [ ] 与同类项目相比是否有独特之处？
- [ ] 是否有跨领域或跨学科的创新？
- [ ] 创新是否具有实际价值？

### 2. 技术深度（权重 25%）

#### 评分标准
| 分数段 | 标准 | 示例 |
|--------|------|------|
| 9-10分 | 技术卓越 | 深度优化、自研核心模块、攻克技术难点 |
| 7-8分 | 技术扎实 | 架构设计合理、代码质量高、有优化措施 |
| 5-6分 | 技术合格 | 功能实现完整、代码可读、无重大问题 |
| 3-4分 | 技术一般 | 实现简陋、存在代码问题、缺乏设计 |
| 0-2分 | 技术粗糙 | 代码混乱、无法运行、存在严重bug |

#### 评估要点
- [ ] 是否有架构设计？
- [ ] 代码是否规范、可读？
- [ ] 是否有单元测试或集成测试？
- [ ] 是否有性能优化？
- [ ] 是否有错误处理和日志记录？
- [ ] 是否使用了高级技术或深度学习框架？

### 3. 文档质量（权重 20%）

#### 评分标准
| 分数段 | 标准 | 示例 |
|--------|------|------|
| 9-10分 | 文档完善 | README详尽、API文档完整、有教程和示例 |
| 7-8分 | 文档良好 | README完整、有使用说明、代码注释充分 |
| 5-6分 | 文档基本 | 有基本说明、部分代码有注释 |
| 3-4分 | 文档简略 | README过于简单、缺少关键信息 |
| 0-2分 | 文档缺失 | 无README或文档、代码无注释 |

#### 评估要点
- [ ] 是否有完整的 README.md？
- [ ] 是否有项目背景和目标说明？
- [ ] 是否有安装和运行步骤？
- [ ] 是否有使用示例或截图？
- [ ] 代码是否有充分的注释？
- [ ] 是否有 API 文档（如适用）？

### 4. 可复现性（权重 20%）

#### 评分标准
| 分数段 | 标准 | 示例 |
|--------|------|------|
| 9-10分 | 完全可复现 | 提供完整环境、一键运行、有详细配置说明 |
| 7-8分 | 易于复现 | 环境配置清晰、依赖明确、运行步骤详细 |
| 5-6分 | 可复现 | 有基本配置说明、可能需要少量调试 |
| 3-4分 | 复现困难 | 缺少关键信息、需要较多调试 |
| 0-2分 | 无法复现 | 无法运行、缺少必要文件或配置 |

#### 评估要点
- [ ] 是否提供了 requirements.txt 或 environment.yml？
- [ ] 是否说明了 Python 版本要求？
- [ ] 是否有安装依赖的说明？
- [ ] 是否有运行命令示例？
- [ ] 是否有数据集获取说明？
- [ ] 是否提供了预训练模型（如适用）？

### 5. 社区价值（权重 10%）

#### 评分标准
| 分数段 | 标准 | 示例 |
|--------|------|------|
| 9-10分 | 高价值 | 解决社区痛点、可广泛应用、对生态贡献大 |
| 7-8分 | 较高价值 | 解决实际问题、有应用前景、促进技术传播 |
| 5-6分 | 一般价值 | 有一定应用场景、对部分用户有价值 |
| 3-4分 | 价值有限 | 应用场景窄、用户群体小 |
| 0-2分 | 无价值 | 无实际应用价值、重复造轮子 |

#### 评估要点
- [ ] 是否解决了飞桨社区用户的常见问题？
- [ ] 是否有广泛的应用前景？
- [ ] 是否促进了飞桨技术的传播？
- [ ] 是否可以开源给社区使用？
- [ ] 是否对其他开发者有参考价值？

## AI 生成内容检测标准

### 检测特征

1. **代码注释模式**
   - 过于规整的注释格式
   - 模板化的函数/类说明
   - 缺乏个人风格的表述

2. **文档表达模式**
   - 段落结构过于工整
   - 缺乏口语化表达
   - 大量使用列表和表格

3. **内容一致性**
   - 代码风格突变
   - 注释与代码不匹配
   - 不同部分语言风格差异大

### 减分规则

| 检测结果 | 减分比例 | 说明 |
|----------|----------|------|
| 大量 AI 生成 | 30% | 超过 50% 内容明显由 AI 生成 |
| 部分 AI 生成 | 15% | 20%-50% 内容由 AI 生成 |
| 少量 AI 生成 | 0% | 少于 20% 或合理使用 AI 辅助 |

## 综合评分计算

```
综合评分 = (创新性 × 0.25 + 技术深度 × 0.25 + 文档质量 × 0.20 + 可复现性 × 0.20 + 社区价值 × 0.10) × (1 - AI减分比例)
```

### 加精建议标准

- **建议加精**：综合评分 ≥ 8.0 分
- **可考虑加精**：综合评分 7.0-7.9 分，且某维度特别突出
- **暂不建议加精**：综合评分 < 7.0 分
