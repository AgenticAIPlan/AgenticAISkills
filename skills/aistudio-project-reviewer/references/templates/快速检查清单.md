## 快速评分参考

### 优秀项目特征（8-10分）
- README 完整详尽
- 代码质量高，可运行
- 有创新点或技术深度
- 文档清晰，易于复现
- 社区价值高

### 良好项目特征（6-7分）
- README 基本完整
- 代码可运行
- 有一定技术实现
- 文档较清晰
- 有一定社区价值

### 一般项目特征（4-5分）
- README 较简略
- 代码基本可运行
- 技术实现一般
- 文档不够清晰
- 社区价值有限

### 较差项目特征（0-3分）
- README 缺失或过于简单
- 代码无法运行
- 技术实现粗糙
- 缺少文档
- 无明显价值

## 常见问题快速判断

| 问题 | 影响维度 | 建议扣分 |
|------|----------|----------|
| 代码无法运行 | 可复现性 | -3 到 -5 分 |
| README 缺失 | 文档质量 | -3 到 -5 分 |
| 无依赖说明 | 可复现性 | -2 到 -3 分 |
| 代码无注释 | 文档质量 | -1 到 -2 分 |
| 缺少示例 | 文档质量 | -1 到 -2 分 |
| 技术实现简单 | 技术深度 | -1 到 -3 分 |
| 无创新点 | 创新性 | -1 到 -3 分 |
| AI 大量生成 | 综合 | -30% 总分 |
