# aiduijie 设计指南

> 飞桨华南区AI供需智能匹配 — 完整设计文档

---

## 一、定位

**飞桨华南区专属**供需匹配平台，非通用工具。

- 伙伴池：固定 160 家华南区生态伙伴
- 区域：湖南、湖北、重庆、四川、福建、广东、广西、海南、香港、澳门
- 技术栈：飞桨 + 文心

---

## 二、核心流程

```
用户输入需求（文字/图片）
       ↓
Step 1: 读取 partner-data.json（160家华南区伙伴）
       ↓
Step 2: 意图解析
       ├─ 提取：行业/技术/规模/城市偏好
       └─ 图片 → 多模态理解
       ↓
Step 3: 伙伴匹配（三层策略）
       ├─ 精确匹配：行业+技术+城市
       ├─ 放宽匹配：行业+技术（不限城市）
       └─ 全网搜索：匹配不足时补充
       ↓
Step 4: 方案生成
       ├─ 需求概要
       ├─ 推荐伙伴（Top 3-5）
       ├─ 推荐方案（技术+工具+周期）
       └─ 追问澄清（≤3个）
       ↓
Step 5: 增量更新
       └─ 用户补充 → 重新匹配 → 输出差异
```

## 三、伙伴匹配算法

### 3.1 评分规则

| 维度 | 权重 | 匹配方式 |
|------|------|----------|
| 行业匹配 | 40% | 用户行业 ≈ 伙伴 `industry` 字段 |
| 技术匹配 | 30% | 用户技术需求 ≈ 伙伴 `tech` 字段 |
| 城市匹配 | 20% | 用户城市偏好 ≈ 伙伴 `city` 字段 |
| 产品相关度 | 10% | 用户需求关键词 ≈ 伙伴 `product` 字段 |

### 3.2 匹配策略

```
第一轮：行业 + 技术 + 城市（严格匹配）
  → 结果 ≥ 3家？输出方案
  → 结果 < 3家？进入第二轮

第二轮：行业 + 技术（放宽城市）
  → 结果 ≥ 3家？输出方案，标注「放宽城市」
  → 结果 < 3家？进入第三轮

第三轮：行业（只保核心匹配）
  → 结果 ≥ 1家？输出方案
  → 无匹配？全网搜索补充
```

### 3.3 行业模糊匹配

由于行业标签粒度不一，需支持模糊匹配：

| 用户场景 | 可匹配行业 |
|----------|-----------|
| 质检/检测 | 工业、机械 |
| 票据/文档 | 金融、企业服务 |
| 客服/对话 | 企业服务、消费服务 |
| 教学/培训 | 教育 |
| 政务/办公 | 政务 |
| 医疗/影像 | 医疗 |
| 出海/法律 | 法律、企业服务 |

## 四、追问策略

| 缺失字段 | 追问模板 |
|----------|----------|
| 行业 | 「具体是什么行业？金融/工业/医疗/教育/政务/消费服务…」 |
| 技术需求 | 「需要哪类AI能力？OCR文字识别/CV图像/NLP文本/大模型对话/智能体」 |
| 城市偏好 | 「有城市偏好吗？厦门/广州/深圳/重庆/香港…或华南不限？」 |
| 数据规模 | 「数据量大概多少？<1000/1000-1万/>1万」 |
| 部署方式 | 「部署方式有要求吗？本地/云端/混合」 |

## 五、增量更新

用户补充信息后：

1. **累加参数**：`tech_need: ["OCR"] → ["OCR", "手写识别"]`
2. **重新匹配**：基于新参数重新筛选
3. **输出差异**：

```
🔄 方案更新（基于补充信息：手写体识别）

变化：
├─ 推荐伙伴调整：
│   ├─ 【伙伴A】↑ 上升（产品支持手写体）
│   └─ 【伙伴C】↓ 下降（仅印刷体）
└─ 推荐方案调整：增加手写体模型微调
```

```

├─ 部署方式：本地部署（数据不出域）
└─ 预估周期：POC 2周，落地 4周
```

## 八、踩坑经验

- 行业字段需模糊匹配，不能精确等于
- 技术标签只有「飞桨」「文心」两大类，需结合产品名判断具体能力
- 定价信息不完整，推荐时提醒用户单独询价
- 厦门伙伴最多（65家），优先推荐本地伙伴
- 非华南区需求需明确告知「仅覆盖华南区」
