## 字段说明

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `company` | string | ✅ | 开发企业名称 |
| `product` | string | ✅ | 产品名称 |
| `city` | string | ✅ | 城市 |
| `region` | string | ✅ | 华南 / 华中 / 西南 |
| `industry1` | string | ✅ | 一级行业 |
| `industry2` | string | ❌ | 二级行业 |
| `tech` | string | ✅ | 飞桨 / 文心 |
| `product_intro` | string | ✅ | 产品介绍（真实内容，最多3000字）|
| `ai_tags` | string[] | ✅ | AI能力标签数组 |
| `pricing` | string | ❌ | 产品定价 |
| `customers` | string | ❌ | 典型客户 |
| `product_type` | string | ❌ | 软件平台 / 软硬一体 / SaaS / API |

---

## 匹配建议

1. **ai_tags 匹配**（最精准）→ 客户需要的 AI 能力在供应商的 ai_tags 中
2. **product_intro 语义**（补充）→ 供应商产品介绍中提到相关技术/场景
3. **industry 匹配**（参考）→ 供应商行业与客户场景相关

> 产品介绍是匹配的核心依据，很多产品名无法直接反映能力，需结合介绍判断
