# 风格选型与提示词组装

> **核心原则**：提示词不是自由描述，而是按固定 Skeleton 从文件中**填槽**组装。
> 执行方只负责填写 CONTENT_SECTION，其余部分从 `styles-index.csv` 读取或按场景固定写法拼接。

---

## Part 1：风格推荐逻辑

### 推荐目标

给用户推荐 3 个风格方向，3 个方向来自不同的情绪大类，给出具体的视觉理由。

### 推荐流程

**Step 1**：理解用户提供的内容主题、情绪、场景。

**Step 2**：从 `references/styles-index.csv` 中选出 3 个风格，可结合 `mood`、`best_for` 列判断，唯一的硬约束：

**3 个推荐必须来自不同的情绪大类**（极简/安静、优雅/轻奢、几何/结构、强烈/冲击、活力/多巴胺、实验/前卫、复古/怀旧、做旧/质感、迷幻/朋克、未来/科技、手工/温暖、可爱/活泼、叙事/插画、拼贴/混媒）

> `styles-index.csv` 已将原先的 `字体/排版` 大类按气质并入更细的 mood 中，例如 `Serif Revival` → `优雅/轻奢`，`Kinetic Typography` → `活力/多巴胺`，`Scavenged Lettersets` → `实验/前卫`。

其他方面自由选择，不需要刻意追求反差，匹配内容气质也好，意外组合也好，都可以。

**Step 3**：为每个推荐说明视觉上会呈现什么效果，以及为什么适合这个内容。

### 推荐展示模板

```
为你的「[场景]」推荐 3 个风格方向：

**方向 A：[风格中文名]（[英文名]）**
视觉效果：[具体描述，100字内]
选这个的理由：[20-40字]

**方向 B：[风格中文名]（[英文名]）**
视觉效果：[...]
选这个的理由：[...]

**方向 C：[风格中文名]（[英文名]）**
视觉效果：[...]
选这个的理由：[...]

请选择一个方向，或描述你想要的其他风格。
```

### 场景的尺寸和构图偏好

| 场景 | 构图偏好 | 说明 |
|------|---------|------|
| 信息图 | 横版，信息层级清晰 | 允许文字标签，视觉流向左→右 |
| 文章配图 | 横版，单一焦点，海报感 | 禁止标签/箭头，表达性而非说明性 |
| 封面图 | 横版，缩略图友好，第一眼冲击 | 可含标题文字 |
| 小红书图文 | 竖版，高颜值，有分享欲 | 可含标题文字，色彩鲜艳优先 |

尺寸统一以各场景工作流和 `SKILL.md` 的默认尺寸表为准：信息图/文章配图默认 `1264x848`，封面默认 `1376x768`，小红书图文默认 `848x1264`。

### 小红书场景的额外限制

以下风格在小红书场景**推荐降权**（不是禁止，但要有充分理由才选）：

- Xerox（过于暗沉，分享欲低）
- Anti-Design（可读性低，不适合平台调性）
- Scavenged Lettersets（碎片化，算法不友好）

### 约束清单

- [ ] 3 个推荐来自不同情绪大类
- [ ] **必须完整遍历 `styles-index.csv`，不能只从印象中的少数风格里选**
- [ ] 其中 1 个是"意外之选"——和内容气质看似不搭，但有惊喜感，理由要说清楚
- [ ] 每个推荐有具体的视觉效果描述（不是泛泛"极简风"，要说清楚颜色/质感/构图会怎么呈现）
- [ ] 每个推荐有真实的理由（不是泛泛而谈）

## Part 2：提示词组装规则

### 提示词 Skeleton

每张图的提示词由 5 个 Section 按顺序拼接，用空行分隔：

```
{STYLE_SECTION}

{PALETTE_SECTION}

{LAYOUT_SECTION}

{CONTENT_SECTION}

{CONSTRAINT_SECTION}
```

---

### STYLE_SECTION 组装规则

#### 基础层（必须）

从 `references/styles-index.csv` 对应风格行读取 `rendering_keywords` + `visual_rule` 拼接：

```
[rendering_keywords]风格。
[visual_rule]。
```

**示例（Neubrutalism）**：
```
新粗野主义（Neubrutalism）风格，粗黑描边 2-4px，偏移阴影（右下 4-8px 纯黑实色块），纯色无渐变，无圆角，强对比几何分块。
所有元素有 2-4px 纯黑描边，右下方 4-8px 纯黑实色色块偏移阴影，无渐变，无圆角。
```

**示例（Risograph）**：
```
孔版印刷风（Risograph），套印偏差效果，颗粒感网点纹理，颜色叠加产生第三色，限色印刷感。
模拟 Risograph 印刷机，两色层有 2-4px 随机偏移，颗粒网点覆盖全图，颜色叠合区出现第三色。
```

---

#### 小红书风格扩展层（小红书图文场景专用）

小红书图文与其他场景不同，图片需要**明确描述视觉元素组合**，才能让模型产出高颜值效果。

在基础层之后，根据风格追加以下扩展描述：

##### cute（甜美可爱）
```
画面装饰元素：hearts、stars-sparkles、flowers；背景使用 solid-pastel 粉彩色块或 gradient-linear 渐变；
图文元素边缘加 white-solid 描边（medium 5-8px）；应用 clear-glow 或 cream-skin 滤镜，透明发光感；
花字使用 bubble 或 highlight 样式；边框使用 polaroid 或 tape-corners 风格；
整体氛围：圆润、粉嫩、少女感。
```

##### notion（极简手绘）
```
画面以手绘线稿为主，hand-drawn wobble 效果，所有线条单线无填充，极简几何形状；
背景纯白 #FFFFFF 或 off-white #FAFAFA，大量留白；装饰使用 hand-drawn-lines、arrows-curvy；
标签使用 black-white 或 pill 样式；偶用 pastel 蓝/黄/粉作点缀色，不超过 3 色；
整体氛围：干净、知识感、Notion 笔记风。
```

##### chalkboard（黑板粉笔）
```
背景黑板底色 #1A1A1A，所有元素模拟粉笔手绘质感，线条不规则有粉笔颗粒纹理；
粉笔颜色：白色主文字 #F5F5F5，黄 #FFE566 / 粉 #FF9999 / 蓝 #66B3FF / 绿 #90EE90 强调色；
装饰：手绘下划线、圆圈标注、简单涂鸦图标、箭头；可见粉笔灰尘效果；
整体氛围：教学感、手工感、黑板课堂风。
```

##### warm（温暖治愈）
```
背景奶油色 #FFFAF0 或蜜桃色 #FED7AA，应用 warm-tone 或 cream-skin 滤镜；
装饰：clouds、stars-sparkles、hearts；边框 polaroid 或 tape-corners；
花字使用 highlight 或 handwritten 样式；标签使用 ribbon 或 bubble；
整体氛围：温暖、治愈、生活感。
```

##### bold（高冲击）
```
背景深黑 #000000 或深色高饱和色块，应用 high-saturation 滤镜；
花字使用 shadow-3d 或 stroke-text，字号极大；强调元素：exclamation、star-burst、red-arrow；
装饰：arrows-curvy、squiggles；图形边缘 colored-solid 或 double 描边（thick 10-15px）；
整体氛围：强烈、冲击、警示感。
```

##### retro（复古怀旧）
```
背景做旧纸张底 #F5E6D3，应用 film-grain 或 muted-tones 滤镜；
装饰：halftone 半色调网点、vintage badges；边框 polaroid、film-strip 或 stamp-border；
花字使用 brush 或 handwritten，标签使用 stamp 或 ribbon；
纹理叠加：paper、scratch；整体氛围：做旧、年代感、胶片质感。
```

##### pop（活力漫画）
```
背景纯白或浅灰，应用 high-saturation 滤镜，高饱和色块对比强烈；
花字使用 stroke-text 或 shadow-3d；装饰：starburst、confetti、squiggles；
图形 colored-solid 描边（medium-thick），边框无；
整体氛围：漫画感、活力爆炸、年轻向。
```

##### minimal（极简高级）
```
背景纯白 #FFFFFF 或 off-white #FAFAFA，无滤镜或 muted-tones；
装饰极少：仅 underline 或 circle-mark 作强调；单一内容色点缀（内容相关色）；
无描边或 white-solid thin（2-4px）；整体氛围：高级感、呼吸感、专业。
```

##### sketch-notes（手绘信息图）
```
背景暖奶油 #F5F0E8（paper-texture 质感），所有线条 hand-drawn wobble 效果；
色块使用马卡龙配色：蓝 #A8D8EA / 薰衣草 #D5C6E0 / 薄荷 #B5E5CF / 桃 #F8D5C4；
色填不完全填满边界（手绘水彩感）；装饰：stars-sparkles、arrows-curvy、squiggles；
花字 handwritten 样式，标签 rounded-badge；整体氛围：手绘教程感、马卡龙暖系。
```

##### study-notes（学生笔记）
```
白色横线纸底，照片俯视角度；主笔迹：蓝色圆珠笔 #1E3A5F，线条不均匀；
批注：红笔 #CC0000 画圈/下划线/框出重点；关键词：黄色荧光笔 #FFFF00（50%透明度）高亮；
内容极密，整页铺满，留白最少；手持笔正在书写的动作感；整体氛围：真实学生笔记、照片感。
```

##### screen-print（丝网印刷）
```
严格限 2-5 色，平色块无渐变；halftone 网点纹理填充色块，轻微套印偏移（print offset）；
强剪影造型，负空间主动叙事；字体粗体冷凝，融入构图而非叠加；
圆形/拱形/三角几何框架构图；整体氛围：丝印海报、版画质感、符号性。
```

> 当风格为 screen-print 时，**替换**基础原则和文字风格部分，改用：
> ```
> 【特殊规则：丝网印刷风格】
> 严格限色：2-5 种平色块，禁止渐变；
> halftone 网点图案作色调变化，不用渐变；轻微套印偏移增加印刷真实感；
> 粗体冷凝字体或 Art Deco 风格，字体融入构图；
> 负空间主动叙事，每图一个标志性焦点；纸张颗粒纹理叠加底层。
> ```

##### fresh（清新自然）
```
背景纯白 #FFFFFF 或薄荷底 #F0FFF4；应用 clear-glow 或 cool-tone 滤镜；
装饰：leaves、clouds、bubbles；无描边或 white-solid thin；
花字 none 或 highlight，标签 pill 或 white-black；
整体氛围：清爽、呼吸感、自然系。
```

### LAYOUT_SECTION 组装规则

根据场景和图片类型，固定写法：

| 场景 | 写法 |
|------|------|
| 横版（信息图/文章配图/封面） | `横版宽幅构图，比例 3:2（1264×848），视觉流向从左到右。` |
| 竖版（小红书图文） | `竖版构图，比例 3:4（848×1264），手机竖屏友好，高颜值，有保存欲和分享欲。` |
| 方形 | `方形构图（1024×1024），中心对称，社交媒体分享友好。` |
| 封面（横版） | `横版封面，比例 16:9（1376×768），缩略图尺寸下标题仍清晰可读，第一眼强吸引力。` |

### CONTENT_SECTION 组装规则

**这是执行方需要填写的唯一变量部分。**

#### 表达性配图（expressive）

```
[情绪/氛围描述，15-25字]。
[单一视觉主体描述，25-45字，单一焦点，强视觉冲击，具体到形状/姿态/质感]。
```

规则：
- 视觉主体**只能有一个**，不堆砌多个元素
- 描述要具体到可以直接绘制（「被碎片打散的人脸」而不是「注意力分散」）
- 不提及文章内容的文字

**示例**：
```
压迫感与碎裂感并存，注意力消解的窒息感。
一张人脸被无数半透明矩形碎片分割打散，中心的眼睛仍然清晰，边缘区域正在消解模糊。
```

#### 说明性配图（informational）

```
[信息主题，一句话，15字以内]。
[核心信息内容：数字/要点/关系结构，30-50字]。
[文字标注需求（可选）：包含中文关键词标注，每条不超过5字，最多5条]。
```

规则：
- 明确写出需要呈现的具体数字或文字
- 结构关系用方向词描述（「左→右三个步骤」「上下双栏对比」「中心辐射六个分支」）
- 文字标注：每条 ≤5 字，总数 ≤5 条（星河模型中文字符稳定性限制）
- 所有文字标注必须来自用户原文，禁止自行补充

**示例**：
```
三步 API 接入流程。
横向三个节点：申请 Key → 配置环境 → 调用接口，方向箭头连接，步骤序号醒目。
文字标注：「申请 Key」「配置环境」「调用接口」，字体清晰可读。
```

#### 封面图（cover）

```
[核心视觉意象/情绪，20-35字]。
[标题文字（如需要）：图片[位置]显示中文标题：「[标题内容，≤20字]」，
 字体风格：[取 styles-index.csv 该风格的 title_style 字段]]。
```

规则：
- 标题文字必须使用用户提供的原标题或明确指定的文案
- 默认不发明副标题/slogan；如需补充，必须来自原文可提炼句子
- `title_style` 从 `references/styles-index.csv` 对应风格行直接读取，无需修改

#### 小红书图文页（image-card）

```
[本页视觉主体和构图描述，20-40字]。
[标题（如有）：图片上方显示中文标题：「[Hook文字，≤12字]」，字体大而清晰]。
[副文（如有）：图片[位置]显示：「[副标题/要点，≤20字]」]。
```

规则：
- Hook、标题、副文、CTA 都必须来自用户提供内容或已确认大纲
- 可以缩写、改写、重排，但不能新增原文没有的承诺、结论、体验或数据
- 加入当前风格的装饰元素描述（结合风格扩展层）

**cover-hero 示例（cute 风格）**：
```
视觉冲击强，全版大图背景，粉彩渐变背景，hearts 和 stars-sparkles 装饰点缀。
图片上方 1/3 居中显示主标题：「5个护肤神器用了就上瘾」，极大粗体 bubble 花字，白色描边，高对比。
主标题下方副标题：「亲测有效，建议收藏」，字号缩小一级。画面中下部产品主视觉，polaroid 边框。
```

**content-list 示例（notion 风格）**：
```
纯白背景，大量留白，hand-drawn wobble 线条装饰。
左上角页码 badge「02」，圆形 pastel 蓝色块。页面标题居中：「3个核心技巧」，手绘字体清晰。
中部 3 条要点列表，每条前配手绘数字序号，≤15字/条，行间距宽松，arrows-curvy 连接。
底部小字钩子：「下一个更实用👇」。
```

### CONSTRAINT_SECTION 固定内容

> **全局渲染约束（所有场景、所有路线，必须作为 CONSTRAINT_SECTION 最后两行追加）**：
> ```
> 所有中文文字清晰可辨，笔画完整，无乱码、无变形、无错位；中文字符独立排版，字体工整易读。
> 图中严禁出现任何十六进制颜色代码（如 #FFFFFF、#D4713A 等）、色值标注、颜色说明文字；颜色仅用于画面渲染，不得作为可见文字或标签显示在画面中。
> ```
>
> ⚠️ **背景说明**：`styles-index.csv` 的 `color_rule` 字段含有 HEX 值，直接拼入 prompt 后模型会将其当作文字渲染到画面上（出现 `#XXXXXX` 色块标注）。缺少此约束行会导致图中出现颜色代码文字。

#### 表达性配图

```
强视觉冲击，单一焦点构图，海报构图感，无文字标签，无流程箭头，无数据图表。
所有中文文字清晰可辨，笔画完整，无乱码、无变形、无错位；中文字符独立排版，字体工整易读。
图中严禁出现任何十六进制颜色代码（如 #FFFFFF 等）、色值标注；颜色仅用于画面渲染，不得作为可见文字或标签显示。
```

## 视觉一致性的正确做法

❌ **错误（效果差）**：
```
第2张：...内容描述...。风格和色调与封面图保持一致。
```

✅ **正确（效果好）**：
```
第2张完整提示词 =
  第1张的 STYLE_SECTION（完整复制，包含扩展层）
+ 第1张的 PALETTE_SECTION（完整复制）
+ 第2张的 LAYOUT_SECTION
+ 第2张的 CONTENT_SECTION（本张内容）
+ 第2张的 CONSTRAINT_SECTION
+ "风格、色调、渲染质感与系列第一张图保持高度一致。"
```

STYLE_SECTION 和 PALETTE_SECTION 的完整复用（含扩展层），是在不支持 `--ref` 的情况下能做到的最强一致性保证。

## 已知限制与避坑指南

> 以下问题经批量测试（46 风格 × 3 场景共 142 张）验证，稳定复现。

### 1. HEX 色值会被渲染成画面文字

**现象**：图中出现 `#D4713A`、`#0099CC` 等十六进制颜色标签色块。

**原因**：`styles-index.csv` 的 `color_rule` 字段含有 HEX 值，ERNIE-Image-Turbo 会将其视为需要渲染的文字内容。

**已修复**：在所有场景的 CONSTRAINT_SECTION 中追加禁止 HEX 渲染的约束行（见上方）。如果发现某张图仍出现色值文字，检查 CONSTRAINT_SECTION 是否漏加该行。

---

### 2. 写实摄影类风格描述触发内容审核 400

**现象**：调用返回 `Error code: 400`，重试也失败。

**触发词规律**（经测试，以下描述会稳定触发）：
- `电影级摄影质感`
- `电影摄影风格`
- `摄影风格信息图`
- 可能还包括其他含"摄影/写实人像"的描述

**处理建议**：
- 使用此类风格时，将"摄影"改为"插画"或"绘画"描述，例如：
  - ❌ `电影级摄影质感，镜头感饱满`
  - ✅ `电影海报插画风格，光影层次丰富，莫兰迪色系，胶片质感渲染`
- 若仍报 400，该风格在当前模型版本不可用，跳过或换风格。
