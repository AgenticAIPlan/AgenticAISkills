# 封面图子流程

> 封面图支持两种模式，AI 根据内容自动建议，用户确认后执行。
> 核心目标：在信息流中「停住用户的滑动手势」——第一眼决定一切。

---

## 两种封面模式

| 模式 | 名称 | 适合场景 | 生成方式 |
|------|------|---------|---------|
| **模式 A** | 视觉海报封面 | 观点文章、品牌内容、情感类 | 从标题+文章开头提炼核心意象，生成海报级图像 |
| **模式 B** | 信息图式封面 | 干货文章、数据报告、教程、「X 个方法」类 | 将文章核心信息浓缩为一张信息图，内容即封面 |

**AI 自动判断逻辑**：
- 标题含数字/清单/对比/步骤 → 建议模式 B（信息图式）
- 标题是观点/情感/故事/品牌 → 建议模式 A（视觉海报）
- 展示给用户确认，可覆盖

---

## 执行流程

### Step 1：封面信息收集 + 模式判断

从用户提供的内容中提取：

```
1. 文章标题（必需）
2. 文章开头 1-2 段（必需，用于提炼核心意象）
3. 目标平台（公众号/博客/视频/通用）
4. 是否需要封面显示标题文字（默认：是）
```

**标题截断策略**：
- 标题用于图片内文字时，最长保留 20 字
- 优先保留品牌名、数字、核心名词，不改变原意
- 超长标题先在执行阶段截断，再写入提示词中的「原样照写」字段

**封面文字来源原则**：
- 封面上出现的标题、副标题、数字、要点都必须来自用户提供的文章、标题或明确指令
- 默认不额外发明 slogan、广告语、结论句；如果原文没有合适文字，优先做无字封面或只保留原标题
- 模式 B 中的 3-5 个要点必须从用户原文提炼，不可自行补全“更像封面”的卖点

**模式判断触发词**：

| 标题特征 | 建议模式 | 示例 |
|---------|---------|------|
| 数字+清单 | 模式 B | 「5个让你效率翻倍的AI工具」 |
| 步骤/流程 | 模式 B | 「如何从零搭建个人品牌」 |
| 数据/报告 | 模式 B | 「2026年AI创作工具调研报告」 |
| 观点/宣言 | 模式 A | 「我们正在失去深度思考的能力」 |
| 情感/故事 | 模式 A | 「那段被算法绑架的日子」 |
| 品牌/产品 | 模式 A | 「星河 2.0：重新定义创作边界」 |

向用户确认模式后继续。

### Step 2A：视觉海报封面（从标题/文章开头生成）

#### 核心意象提炼

从标题 + 文章开头提炼：

```
分析维度：
1. 核心情绪（紧迫感/好奇感/激励感/沉思感/冲击感）
2. 核心隐喻（用什么视觉主体来承载文章主题？）
3. 视觉意象（非文字说明，而是可以独立成画的画面）
```

**提炼示例**：

| 文章标题 | 核心情绪 | 视觉意象 |
|---------|---------|---------|
| 「我们正在失去深度思考的能力」 | 焦虑/警示 | 被碎片切割的大脑/被通知挤碎的空间 |
| 「AI创作的下一个十年」 | 期待/未来感 | 光速穿越星际的飞行器视角 |
| 「效率不是目的，它只是工具」 | 沉思/反思 | 静止的时钟齿轮，背后透出自然光 |

#### 提示词结构

```
[风格描述，取自 styles-index.csv 的 rendering_keywords + visual_rule] 风格的横版文章封面图。
[核心视觉意象描述，30-50字，单一焦点，强视觉冲击]。
[如需标题文字：图片上方/中央固定显示中文标题：「[标题原文，不超过20字，原样照写，不得改写]」，
 字体风格：[取 styles-index.csv 该风格的 title_style 字段]]。
[构图要求：横版宽幅，封面图质感，第一眼强吸引力，
 缩略图尺寸下标题仍清晰可读，单一视觉焦点]。
[色彩规范，参考所选风格]。
```

> **⚠️ 标题保护**：PE 模型会对提示词中的标题进行语义归一化，须加「原样照写，不得改写」来保留原始标题。超过 20 字时由执行方先截断后再写入，截断策略见 cover-image.md Step 1。

### Step 2B：信息图式封面（内容浓缩为视觉）

#### 内容提炼

从文章中提取核心「可视化结构」：

```
分析维度：
1. 文章的核心结论（1-2 句）
2. 关键数字/数据点（如有）
3. 主要结构（清单/对比/步骤/关系）
4. 视觉层级（什么信息最大？什么最小？）
```

**信息图封面的视觉层级**：
- **最大**：文章标题 / 核心结论（视觉锚点）
- **中等**：3-5 个核心要点（支撑信息）
- **最小**：来源/日期/品牌（可选）

**文字约束**：
- 标题使用用户原文标题或用户明确指定的标题
- 核心结论、数字、要点必须来自用户提供内容，可压缩但不得改写事实含义

#### 提示词结构

```
[风格描述] 风格的横版信息图式封面。
主题：[文章核心主题一句话]。
图片上方显示中文标题：「[文章标题，原样照写，不得改写]」，
 字体风格：[取 styles-index.csv 该风格的 title_style 字段]，主视觉。
[核心要点列表：用简洁的视觉化方式展示 3-5 个要点，
 如：① ... ② ... ③ ...，或通过图形分区呈现]。
[构图要求：信息层级清晰，横版封面，标题优先，
 要点辅助，整体视觉平衡，缩略图友好]。
[色彩规范，参考所选风格]。
```

### Step 3：生图调用

```bash

  --prompt "[中文提示词]" \
  --size 1376x768 \
  --output cover.png

```

## 中文适配注意事项

1. **标题文字**：提示词写法 `图片上方显示中文标题：「[文字]」`，不超过 20 字
2. **星河文字生成**：模型能生成中文文字，但超过 20 字后可能出现错别字，建议标题简化
3. **横版构图**：提示词中强调「横版宽幅构图，视觉主体清晰，缩略图友好」
4. **模式 B 信息图式**：提示词中明确写出结构化信息，数字+要点的组合比长段描述效果更好
5. **点击欲优先**：封面图是第一关，视觉冲击力 > 内容准确性，「夸张处理」是合理策略
