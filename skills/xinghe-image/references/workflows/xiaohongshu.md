# 小红书图文生成

> 多图卡片 + 文案包，一键生成完整发帖包。
> 图片风格来自 `references/image-presets.csv`（直接可用的完整提示词预设）。

---

```
小红书图文进度：
```

## Step 1：内容输入

### 1.1 保存内容源文件

- 用户提供文件路径 → 直接使用
- 用户粘贴内容 → 保存为 `xiaohongshu/{slug}/source.md`
- 文件已存在 → 备份为 `source-backup-YYYYMMDD-HHMMSS.md`

**Slug 规则**：提取内容主题 2-4 词，kebab-case。例：「AI 工具推荐」→ `ai-tools-recommend`

### 1.2 文案来源原则

- 图片上所有文字，默认来自用户提供的内容或明确需求
- 允许平台化改写、缩句、口语化，**不得新增**原文没有的事实、数据、结论、承诺、卖点
- 只有主题没有正文 → 优先出视觉稿，或引导用户补文字素材

---

## Step 2：预设选型 ⚠️ BLOCKING

读取 `references/image-presets.csv`，按分类展示给用户选择：

| 分类 | 说明 |
|------|------|
| 学科知识卡片 | 科普、知识点、概念讲解，风格多样（卡通/手绘/黑板/油画/电影等） |
| 清单/目标可视化 | 清单、步骤、目标、日历，手绘涂鸦或手账风 |
| 表情包/贴纸 | 丑萌卡通角色，贴纸风，高饱和色块 |
| 海报 | 分镜故事板、东方意境、复古剪贴簿 |
| 写实/商业插画 | 产品展示、主题海报，写实或国潮插画（视觉主体根据内容主题自动确定） |
| 其他信息图 | 可爱卡通马、节日祝福等特定主题 |

**展示格式**：

```
请选择图文风格分类：

A. 学科知识卡片 — 适合知识科普、干货分享
B. 清单/目标可视化 — 适合清单、步骤教程
C. 表情包/贴纸 — 适合轻松趣味内容
D. 海报 — 适合情感/故事/美学内容
E. 写实/商业插画 — 适合产品展示、主题海报
F. 其他信息图 — 特定主题

选定分类后，我会从该分类中展示具体预设供你确认。
```

用户选定分类后，从 CSV 中筛选该分类的所有预设，展示预设描述（提取每条提示词的前 30 字作为摘要），由用户确认使用哪条。

---

## Step 3：生成大纲 ⚠️ BLOCKING

基于选定预设 + 用户内容，生成图文大纲，**展示给用户确认后**才进入生图。

**大纲格式**：

```yaml
---
preset: [预设分类 + 描述摘要]
image_count: [N]（建议 4-8 张）
size: 848x1264
---

[封面]
hook: 「[≤12字，含数字/痛点/好奇/利益，从原文提炼]」
subtitle: 「[≤20字，从原文提炼，可省略]」
swipe_hook: [下一张悬念，≤10字]

[内容-2]
title: 「[≤10字，从原文提炼]」
points:
  - [要点，从原文提炼，≤15字/条]
swipe_hook: [≤10字]

...

[总结]
title: 「[≤10字，从原文总结提炼]」
cta: 「[≤15字，互动引导]」
```

**约束**：
- 所有文案从原文提炼，禁止自行创作事实
- 封面 Hook 必须含：数字 / 痛点 / 好奇 / 利益 之一
- 总页数建议 4-8（封面 1 + 内容 2-6 + 总结 1）

展示给用户：

```
📋 图文大纲（共 [N] 张）

[大纲内容]

确认开始生图？或告诉我需要调整的地方。
```

---

## Step 4：生图

多图并发执行规则见 **SKILL.md「多图生成」节**。

**提示词组装**：

每张图的提示词 = **CSV 预设原文** + 本页内容文字：

```
[从 image-presets.csv 取出的完整预设提示词]

本页内容：
标题：「[从大纲取，≤12字]」
要点：[从大纲取，每条≤15字]
```

> **⚠️ 视觉主体替换规则**：若预设提示词中含有特定物件描述（如「食材」「植物叶片」「美食」等），**必须根据用户内容主题替换为对应的视觉主体**，禁止保留原预设的具体物件描述。例如：技术架构文章 → 替换为「服务器/代码/网络节点等科技元素」；产品推广 → 替换为该产品本身。

**尺寸**：`848x1264`（竖版 3:4，不可更改）

**视觉一致性**：同一套图使用同一条预设，所有页保持视觉统一。第 2 张起在提示词末尾追加：`风格、色调、渲染质感与系列第一张图保持高度一致。`

**文件命名**：`NN-{type}-{slug}.png`，type = cover / content / ending

## Step 5：生成文案包（可跳过）

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📝 发帖文案包

【备选标题】（3个，任选其一）
① [15-25字，数字型，基于原文改写]
② [15-25字，痛点型，基于原文改写]
③ [15-25字，好奇型，基于原文改写]

【正文】
[200-500字，基于原文，小红书语气，分段清晰，emoji 点缀，互动引导结尾]

【标签】
[话题标签 5-8 个，不含 #，按相关性排列]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**约束**：标题/正文/CTA 必须基于原文，不新增事实；正文第一行即 Hook，不以「大家好」开头。

保存到 `xiaohongshu/{slug}/copywriting.md`。

```
✅ 小红书图文生成完成！

图片：[N] 张（848×1264）

📁 文件清单：
```

## 修改单张

| 操作 | 步骤 |
|------|------|
| **重新生成** | 修改对应提示词 → 重新生图 → 末尾追加「风格和色调与系列第一张图保持一致」 |
| **新增** | 指定位置 → 补充大纲条目 → 生图 → 重新编号 |
| **删除** | 删除文件 → 重新编号 → 更新 outline.md |
