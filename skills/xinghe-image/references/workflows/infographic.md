# 信息图子流程

> 将数据、知识、流程等信息内容转化为视觉海报级信息图。
> 图片风格来自 `references/image-presets.csv`（直接可用的完整提示词预设）。
> 支持**单图**（独立信息图）和**系列**（多图拆解）两种模式。

---

## 进度 Checklist

```
信息图进度：
- [ ] Step 1: 内容结构分析
- [ ] Step 2: 预设选型 ⚠️ BLOCKING（用户确认后才生图）
- [ ] Step 3: 生成 outline.md ⚠️ BLOCKING（用户确认后才生图）
- [ ] Step 4: 生图（见 SKILL.md「多图生成」节）
- [ ] Step 5: 输出报告
```

---

## Step 1：内容结构分析

从用户提供内容中提取：

```
1. 内容主题（一句话）
2. 核心信息点（不超过 7 个，超过则归纳合并）
3. 信息的逻辑关系（并列 / 层级 / 因果 / 时间线 / 对比）
4. 关键数字或结论（如有）
5. 单图 or 系列（信息量 > 7 个核心点，或逻辑层级 > 2 层，建议拆成系列）
```

**文字来源原则**：
- 标题、数字、关键词标注都必须来自用户提供内容
- 可以做信息压缩和结构化重排，不得补写原文没有的数据、比例、步骤或结论

---

## Step 2：预设选型 ⚠️ BLOCKING

读取 `references/image-presets.csv`，按分类展示给用户选择：

| 分类 | 说明 |
|------|------|
| 学科知识卡片 | 科普、知识点、概念讲解，风格多样（卡通/手绘/黑板/油画/电影等） |
| 清单/目标可视化 | 清单、步骤、目标，手绘涂鸦或手账风 |
| 其他信息图 | 特定主题信息图 |

**展示格式**：

```
请选择信息图风格分类：

A. 学科知识卡片 — 适合知识科普、数据可视化、概念讲解
B. 清单/目标可视化 — 适合清单型、步骤型、目标追踪
C. 其他信息图 — 特定主题

选定分类后，我会展示该分类下的具体预设供你确认。
```

用户选定分类后，从 CSV 筛选该分类所有预设，展示摘要（提取提示词前 30 字），用户确认具体使用哪条。

---

## Step 3：生成 outline.md ⚠️ BLOCKING

基于 Step 1 + Step 2 结果生成大纲，**展示给用户确认后**才进入生图。

### 单图 outline 格式

```yaml
---
preset: [预设分类 + 描述摘要]
mode: single
size: 1264x848
---

## 信息图

**主题**: [核心主题一句话]
**信息结构**: [描述信息的组织方式，如：横向三节点流程 / 左右对比 / 中心辐射六分支]
**核心信息点**:
  - [要点 1，≤10字]
  - [要点 2，≤10字]
  - ...（≤7条）
**关键数字**: [如有]
**文件名**: infographic-[slug].png
```

### 系列 outline 格式

```yaml
---
preset: [预设分类 + 描述摘要]
mode: series
image_count: [N]
size: 1264x848
---

## 图 1 — 概览/封面

**主题**: [整体主题一句话]
**视觉**: [封面构图描述]
**文件名**: 01-overview-[slug].png

## 图 2 — [子主题]

**信息结构**: [描述]
**核心信息点**: ...
**文件名**: 02-[subslug].png

...
```

展示给用户：

```
📊 信息图计划 — 模式：[单图 / 系列 N 张]

[大纲内容]

确认开始生图？或告诉我需要调整的地方。
```

---

## Step 4：生图

**单图**：直接调用，无需并发。见 SKILL.md「生图调用」节。

**系列**：封面先行 + 内容页并发。见 **SKILL.md「多图生成」节**。

**提示词组装**：

每张图的提示词 = **CSV 预设原文** + 本图内容：

```
[从 image-presets.csv 取出的完整预设提示词]

本图内容：
主题：[从大纲取]
核心信息点：[从大纲取，每条≤10字，≤7条]
关键数字：[如有]
```

**尺寸**：

| 模式 | 默认尺寸 | 备选 |
|------|---------|------|
| 单图 / 系列 | `1264x848`（横版 3:2） | `1200x896`、`1024x1024` |

**系列视觉一致性**：第 2 张起在提示词末尾追加：`风格、色调、渲染质感与第一张图保持高度一致。`

```bash
open {output-dir}/
```

---

## 文件目录结构

> **`{output-dir}` 默认值**：`infographics/{slug}/`（相对于当前工作目录）。若用户指定了其他路径则以用户为准。

```
infographics/{slug}/
├── outline.md
├── 01-[slug].png
├── 02-[slug].png
└── ...
```

**Slug 规则**：提取核心主题 2-4 词，kebab-case。例：「API 接入流程」→ `api-flow`

**备份规则**：文件已存在 → 自动备份为 `*-backup-YYYYMMDD-HHMMSS`
