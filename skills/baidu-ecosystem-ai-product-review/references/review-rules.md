# 审核规则

## 一、基础字段

- 产品名称：必填
- 应用单位：必填，应为企业官方全称
- 注册地址：必填，应与应用单位一致
- 产品上线时间：必填，至少精确到月

补充口径：

- 若上线时间晚于当前审核日期，默认作为告警处理，不直接不通过
- 若同时存在调用记录、测试账号、测试链接，可理解为测试期已使用

## 二、产品功能简介

- 必填
- 原则上不少于 100 字
- 应体现：
  - 产品是什么
  - 面向谁
  - 解决什么场景
  - 用到了哪些 AI 技术
  - 主要功能是什么

## 三、百度技术模块

- 必填
- 要写“产品功能”，不要只写技术点
- 应与产品功能简介尽量对应

## 四、模型 / 套件

- 必填
- 应完整列出实际使用的百度技术
- 模型和功能需要匹配
- 使用明显较老模型应告警

## 五、占比与调用比例

- 模板要求通常不低于 30%
- 默认先接受填写方自评
- 但若明显虚高，应输出不合理并建议缩小产品边界或重写占比
- 若填写 `100%`，默认理解为仅使用该百度技术体系，不需要额外澄清

## 六、截图证据

- 代码截图：用于证明百度技术确实在代码中被调用
- 调用量截图：用于证明 API 方式下真实存在调用
- 不允许只看正文，不看截图
- 若截图只能证明部分模型或部分模块，通常记为告警
- 若关键截图完全缺失或无法证明核心主张，判不通过

### 文心代码截图关键痕迹

- `ERNIE`
- `qianfan`
- `aistudio`
- 千帆平台调用接口
- 文心 API / base_url / api_key

### 飞桨代码截图关键痕迹

- `Paddle`
- `PaddleOCR`
- `PaddleX`
- `PaddleLite`
- `PaddleServing`
- `YOLOE`

## 七、数据版权信息

- 必填
- 至少说明：
  - 数据归属
  - 数据内容
  - 数据量级

若数据内容和量级留空，通常不通过。

## 八、收益

- 必填
- 必须尽量量化
- 可接受：
  - 准确率提升
  - 时延下降
  - 效率提升
  - 成本下降
  - 收入提升
  - 转化提升
- 不可接受：
  - 大幅提升
  - 显著增强
  - 降本增效

## 九、售价与试用方式

- 售价原则上必填
- 文心模板中的试用链接 / 测试账密原则上必填
- 若因保密、内网、敏感数据无法提供，可转为澄清项，不直接否决
- 若无链接、无账号、无说明，则通常不通过

## 十、OCR 规则

- 对扫描 PDF 必须 OCR
- OCR 低质量页要单独告警
- 若关键字段或截图看不清，不强行判断，要求补更清晰材料
