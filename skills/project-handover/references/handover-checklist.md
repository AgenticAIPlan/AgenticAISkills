# Reviewer 终审检查清单

## 使用方式

Step 7 终审时，逐项检查以下所有条目。每个条目标记为 `PASS` / `FAIL` / `N/A`。

- `PASS`：检查通过
- `FAIL`：不通过，分为 `error`（阻塞）和 `warning`（非阻塞）
- `N/A`：不适用（如本次交接没有外部联系人）

---

## A. 结构完整性（error）

| # | 检查项 | 通过条件 |
|---|--------|---------|
| A1 | 手册包含所有模板要求的章节 | 对照 `assets/handover-manual-template.md`，每个一级章节都已填充 |
| A2 | 每个章节不为空 | 不存在只有标题没有内容的章节 |
| A3 | 章节顺序符合模板 | 按 1-7 章顺序排列，不得随意调整 |
| A4 | 覆盖度报告已生成 | 包含 6 个维度的覆盖率和总覆盖率 |

## B. 来源可追溯性（error）

| # | 检查项 | 通过条件 |
|---|--------|---------|
| B1 | 每条 SOP 步骤标注来源 | 至少标注来源文件名 |
| B2 | 联系人信息有来源 | 标注信息获取方式（文档/口述/系统导出） |
| B3 | 项目状态有依据 | 进展描述基于实际数据，不是推测 |
| B4 | 风险描述有背景 | 说明风险发现的时间和情境 |

## C. 真实性校验（error）

| # | 检查项 | 通过条件 |
|---|--------|---------|
| C1 | 无编造内容 | 所有标记 `[待补充]` 的内容未被填充为编造内容 |
| C2 | 矛盾信息已标注 | 所有 `[矛盾]` 信息均列出双方说法 |
| C3 | 推测内容已标注 | 基于间接信息推测的内容标记为 `[推测]` |
| C4 | 术语有定义 | 首次出现时给出简要解释 |

## D. 覆盖率达标（error）

| # | 检查项 | 通过条件 |
|---|--------|---------|
| D1 | 总覆盖率 ≥ 60% | 低于 60% 不得输出正式手册 |
| D2 | 无 0% 维度 | 任一维度覆盖率为 0%，手册降级为阶段性交接稿 |
| D3 | 核心维度 ≥ 70% | 维度 1（业务流程）和维度 2（项目待办）覆盖率 ≥ 70% |
| D4 | 信息缺口已列出 | 所有未覆盖问题已汇总为信息缺口清单 |

## E. 记忆文件（error）

| # | 检查项 | 通过条件 |
|---|--------|---------|
| E1 | MEMORY.md 索引已更新 | 索引条目包含项目名、交接人、文件路径、日期 |
| E2 | 交接专项记忆已创建 | handover-[项目名].md 文件存在且包含快速参考 |
| E3 | 记忆内容基于真实材料 | FAQ 条目有对应来源，无编造 |
| E4 | 记忆大小合理 | 专项记忆文件 < 500 行 |

## F. 格式规范（warning）

| # | 检查项 | 通过条件 |
|---|--------|---------|
| F1 | 手册命名规范 | `[项目名]-交接手册-[交接人]-[日期].md` |
| F2 | Markdown 格式正确 | 标题层级、表格、列表格式无错误 |
| F3 | 无个人信息泄露 | 不包含身份证、银行卡、私钥等敏感信息 |
| F4 | 链接可用 | 手册中的文件路径链接存在且可访问 |

## G. 实用性检查（warning）

| # | 检查项 | 通过条件 |
|---|--------|---------|
| G1 | SOP 可直接执行 | 被交接人读完后可以直接按步骤操作 |
| G2 | FAQ 覆盖常见问题 | 至少 5 条 FAQ，覆盖最常遇到的操作问题 |
| G3 | 风险应对可操作 | 每个风险至少有 1 条具体的应对措施 |
| G4 | 紧急联系人明确 | 出现紧急问题时知道第一个联系谁 |

---

## 终审结果判定

| 条件 | 结果 |
|------|------|
| 所有 A-D 类检查 PASS | **正式交接手册** |
| 存在 A-D 类 FAIL（error） | **阶段性交接稿**，必须修复后升级 |
| 所有 A-D 类 PASS，E 类有 FAIL | **交接手册（记忆不完整）**，需补写记忆文件 |
| F/G 类有 FAIL（warning） | **通过**，但建议尽快修复 |
| 总覆盖率 < 60% | **不通过**，必须回到 Step 2 补充材料 |

---

```markdown
## Reviewer 终审结果

| 类别 | 检查数 | PASS | FAIL(error) | FAIL(warning) |
|------|--------|------|-------------|---------------|
| A 结构完整性 | 4 | 4 | 0 | 0 |
| B 来源可追溯 | 4 | 3 | 1 | 0 |
| C 真实性 | 4 | 4 | 0 | 0 |
| D 覆盖率 | 4 | 4 | 0 | 0 |
| E 记忆文件 | 4 | 4 | 0 | 0 |
| F 格式规范 | 4 | 3 | 0 | 1 |
| G 实用性 | 4 | 4 | 0 | 0 |

### error 级问题
- [B2] 联系人信息未标注来源（来源：口述/文档/系统）

### warning 级建议
- [F1] 建议将手册重命名为标准格式

### 判定：阶段性交接稿（修复 B2 后可升级为正式手册）
```
