# 女频短剧剧本生成 Agent

基于多Agent协作的女频短剧剧本创作系统，支持从选题到成稿的完整创作流程。本skill基于 [drama-script-agent](https://github.com/dingjin-art/drama-script-agent) 项目。

## 核心特性

- **5个专业Agent协作**：剧本策划师、情节编剧师、对白优化师、一致性检查员、风格顾问
- **三层记忆机制**：核心设定 + 动态状态 + 会话上下文，确保剧情连贯
- **14维度质量Checklist**：全面检查剧本质量
- **10种钩子类型**：自动检查钩子多样性，避免重复
- **冲突兑现检查**：确保每个冲突都有实质性后果
- **统一剧本格式**：标准化剧本格式，符合行业要求

## 适用平台

- 红果短剧、抖音短剧、爱奇艺短剧、腾讯短剧、快手星芒短剧

## 适用题材

- 古言宫斗、现代言情、悬疑推理、喜剧、重生复仇

## 使用方法

### 基础用法

```
/drama-agent [题材] [简述]
```

示例：
```
/drama-agent 古言重生复仇 女主重生到仇人之女身上
```

### 带参数

```
/drama-agent [题材] [简述]
  genre: [题材类型]
  episodes: [集数]
  style: [风格]
  requirements: [特殊要求]
```

示例：
```
/drama-agent 古言复仇
  genre: 古言
  episodes: 80
  style: 紧张
  requirements: 大团圆结局
```

### 常用命令

| 命令 | 说明 |
|------|------|
| `/drama-agent [题材] [简述]` | 开始新项目 |
| `/drama-agent continue` | 继续创作下一集 |

## Agent 体系

### @planner - 剧本策划师

**职责**：需求理解、结构设计、人物塑造

**输出**：剧本框架（剧名、类型、人物小传、三幕结构、集数分配）

### @writer - 情节编剧师

**职责**：场景构建、情节推进、悬念设置

**输出**：详细场景描述（场次、时间、地点、人物、事件流程）

### @polisher - 对白优化师

**职责**：台词创作、人物声音、情感表达

**输出**：完整对白剧本（符合格式规范）

### @checker - 一致性检查员

**职责**：逻辑审查、漏洞发现、修正建议

**输出**：一致性检查报告

### @stylist - 风格顾问

**职责**：题材适配、风格调整、创意建议

**输出**：风格优化建议

## 剧本格式规范

### 标准格式

```
第X集

X-X
[时间]/[内景/外景] [地点]
[人物]

[场景描述]

△ [动作描写]
[情绪状态]：台词
OS [心理活动]

【钩子：[钩子类型]——[钩子描述]】
```

### 格式说明

| 元素 | 格式 | 示例 |
|------|------|------|
| 集数 | 第X集 | 第1集 |
| 场次 | X-X | 1-1 |
| 时间 | 日/夜/晨/傍晚 | 日/外景 |
| 内外景 | /内景 或 /外景 | 日/内景 闺房 |
| 人物 | 全名，群戏加"若干" | 女主角、反派、众人若干 |
| 动作 | 前加 `△` | △ 女主角缓缓抬头 |
| 台词 | 用 `：` 引出 | 女主角：（平静）这是我的选择 |
| 心理 | 用 `OS` 表示 | 女主角：OS 我不会放弃的 |
| 钩子 | 【钩子：类型——描述】 | 【钩子：悬念型——主角将如何逆转？】 |

## 质量Checklist（14个维度）

### 1. 上下文连贯性
- 时间线是否一致
- 人物状态是否连续
- 剧情逻辑是否合理

### 2. 剧本质量
- 节奏是否紧凑
- 对白是否生动
- 画面感是否强
- 情感是否到位

### 3. 钩子多样性
- 检查钩子类型是否与最近3集不同
- 10种类型：悬念型、危机型、转折型、身份型、情感型、秘密型、选择型、反转型、伏笔型、冲突型

### 4. 冲突兑现
- 冲突是否真正兑现了局面变化
- 避免只是解释或洗白
- 确保有实质性后果

### 5. 人物一致性
- 性格是否前后一致
- 能力是否合理
- 关系是否连贯
- 说话风格是否统一

### 6. 伏笔管理
- 新增了哪些伏笔
- 哪些伏笔得到回应
- 哪些伏笔需要追踪

### 7. 爽点/爆点/虐点
- 每集是否至少有一点
- 爽点 = 装 + 打脸 + 震惊 + 收获

### 8. 语言格式
- 语言质量是否达标
- 格式是否规范
- 时代感是否正确

### 9. 观众预期
- 期待是否建立
- 期待是否满足
- 是否有意料之外的反转

### 10. 原创性
- 是否避免套路
- 是否有差异化

### 11. 可执行性
- 是否可拍摄
- 预算是否合理
- 演员要求是否可行

### 12. AI自检
- 检查AI生成的提示词

### 13. 快速检查
- 创作时快速参考的检查项

### 14. 问题解决
- 诊断问题和解决方案

## 三层记忆机制

### 1. 核心设定（不可变状态）

- 剧名、类型、主题
- 人物设定（全名、年龄、性格、背景、动机、关系）
- 世界观设定
- 情节结构（三幕式/四幕式）
- 集数分配

### 2. 动态状态（每集更新）

- 人物状态变化
- 伏笔设置与回应
- 悬念点
- 已用元素

### 3. 会话上下文（临时状态）

- 当前讨论的焦点
- 用户最新要求
- 待处理事项

## 创作理念

### 核心价值观

1. **故事即生命**：好剧本不是写出来的，是活出来的
2. **人物即灵魂**：每个人物都应该有自己的梦想、恐惧、矛盾
3. **冲突即张力**：没有冲突就没有戏剧

### 爽点公式

```
爽点 = 装 + 打脸 + 震惊 + 收获
```

### 虐点类型

- 始终牢记的人忘了自己
- 永远无法说出口的爱意
- 只有自己遵守的诺言
- 永远无人知晓的巨大牺牲
- 至死未能解开的惨痛误会

## 执行流程

```
用户需求
    ↓
@planner 生成剧本框架（用户确认）
    ↓
@writer 生成详细场景
    ↓
@polisher 添加对白
    ↓
@checker 检查一致性
    ↓
最终剧本
```

## 注意事项

1. **用户确认**：剧本框架生成后必须等待用户确认
2. **状态锁定**：框架确认后，后续Agent不得擅自修改
3. **记忆维护**：每完成一集必须更新记忆卡片
4. **一致性优先**：发现不一致时主动提出修正方案
5. **钩子多样性**：确保钩子类型不与最近3集重复
