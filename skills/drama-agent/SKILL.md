# 女频短剧剧本生成 Agent

基于多Agent协作的女频短剧剧本创作系统，支持从选题到成稿的完整创作流程。

## 适用场景

当用户需要创作女频短剧剧本时使用本Skill，包括但不限于以下情况：

- 用户明确提出"创作短剧剧本"、"写一个短剧"
- 提到"古言"、"重生"、"复仇"、"言情"、"宫斗"等女频常见题材
- 要求"80集"或指定集数的短剧
- 需要生成剧本大纲、人物小传、具体剧本内容
- 需要检查剧本一致性或质量
- 需要继续创作未完成的剧本

## 输入要求

### 最小必需输入

- **题材类型**：如"古言重生复仇"、"现代言情"、"悬疑推理"等
- **简要描述**：核心梗概或故事起点

### 可选输入

- **集数**：默认80集，可根据需求调整
- **风格**：如"紧张"、"轻松"、"虐心"等
- **特殊要求**：如"大团圆结局"、"双男主"等
- **平台**：如"红果短剧"、"抖音短剧"等

## 执行步骤

### Step 1: 需求确认与框架生成 (@planner)

1. 确认用户输入的题材、集数、风格、特殊要求
2. 生成剧本框架，包含：
   - 剧名（朗朗上口，符合平台调性）
   - 类型与主题
   - 人物小传（主角、反派、配角，包含性格、背景、动机、关系）
   - 三幕式/四幕式情节结构
   - 集数分配（每集核心事件）
3. 等待用户确认框架

### Step 2: 详细场景生成 (@writer)

1. 根据确认的框架，逐集生成详细场景
2. 每集包含：
   - 场次分配（如1-1, 1-2）
   - 时间/地点/人物
   - 事件流程
   - 钩子设置（悬念型、危机型、转折型等10种类型）
   - 爽点/爆点/虐点设计
3. 确保钩子类型与最近3集不同

### Step 3: 对白完善 (@polisher)

1. 为每个场景添加对白
2. 确保符合人物性格和说话风格
3. 添加动作描写（用△标记）和心理活动（用OS标记）
4. 检查情感表达是否到位

### Step 4: 一致性检查 (@checker)

1. 检查14个维度的质量
2. 重点检查：
   - 上下文连贯性
   - 人物一致性
   - 冲突兑现（避免只是解释或洗白）
   - 钩子多样性
3. 输出检查报告，如有问题返回修正

### Step 5: 风格优化 (@stylist) - 可选

1. 检查是否符合题材风格
2. 提供创意建议
3. 优化节奏和氛围

### Step 6: 输出最终剧本

按照标准剧本格式输出，包含场次、时间地点、人物、场景描述、对白、钩子标注

## 输出要求

### 剧本格式规范

```
第X集

X-X
[时间]/[内景/外景] [地点]
[人物]

[场景描述]

△ [动作描写]
[情绪状态]：台词
OS [心理活动]

【钩子：[钩子类型]——[钩子描述]】
```

### 质量要求

- **每集至少包含**：爽点、爆点或虐点之一
- **爽点公式**：装 + 打脸 + 震惊 + 收获
- **冲突兑现**：每个冲突必须有实质性后果，不能只是解释或洗白
- **钩子多样性**：检查钩子类型是否与最近3集不同
- **格式规范**：严格按照标准格式输出

### 一致性要求

- 人物性格前后一致
- 能力设定合理
- 关系变化连贯
- 时间线清晰

## 三层记忆机制

### 核心设定（不可变状态）

- 剧名、类型、主题
- 人物设定（全名、年龄、性格、背景、动机、关系）
- 世界观设定
- 情节结构
- 集数分配

框架确认后锁定，后续不得擅自修改。

### 动态状态（每集更新）

- 人物状态变化
- 伏笔设置与回应
- 悬念点
- 已用元素（包括钩子类型）

每完成一集必须更新记忆。

### 会话上下文（临时状态）

- 当前讨论的焦点
- 用户最新要求
- 待处理事项

## 钩子类型（10种）

悬念型、危机型、转折型、身份型、情感型、秘密型、选择型、反转型、伏笔型、冲突型

确保钩子类型与最近3集不同，避免重复。

## 验证检查

完成每集剧本后，验证以下项目：

- [ ] 时间线是否一致
- [ ] 人物状态是否连续
- [ ] 剧情逻辑是否合理
- [ ] 钩子类型是否与最近3集不同
- [ ] 冲突是否真正兑现了局面变化（非解释/洗白）
- [ ] 人物性格是否前后一致
- [ ] 每集是否至少包含爽点/爆点/虐点之一
- [ ] 格式是否符合规范
- [ ] 对白是否符合人物性格

## 注意事项

1. **用户确认**：剧本框架生成后必须等待用户确认，不得擅自继续
2. **状态锁定**：框架确认后，后续Agent不得擅自修改核心设定
3. **记忆维护**：每完成一集必须更新记忆卡片
4. **一致性优先**：发现不一致时主动提出修正方案
5. **钩子多样性**：确保钩子类型不与最近3集重复
6. **冲突兑现**：避免冲突只是被"解释"或"洗白"，必须有实质性后果
