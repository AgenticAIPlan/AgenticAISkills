# 短剧剧本生成 Agent

基于标准短剧创作流程的剧本生成系统，支持从选题到成稿的完整创作流程。

## 适用场景

当用户需要创作短剧剧本时使用本Skill，包括但不限于以下情况：

- 用户明确提出"创作短剧剧本"、"写一个短剧"
- 提到"古言"、"重生"、"复仇"、"言情"、"宫斗"、"现代"、"悬疑"等题材
- 要求指定集数的短剧
- 需要生成剧本大纲、人物小传、具体剧本内容
- 需要检查剧本一致性或质量
- 需要继续创作未完成的剧本

## 输入要求

### 最小必需输入

- **题材类型**：如"古言重生复仇"、"现代言情"、"悬疑推理"等
- **核心梗概**：故事起点或核心冲突（1-3句话）

### 可选输入

- **集数**：默认80集，可根据需求调整
- **风格**：如"紧张"、"轻松"、"虐心"、"甜宠"等
- **特殊要求**：如"大团圆结局"、"双男主"、"全员恶人"等
- **平台**：如"红果短剧"、"抖音短剧"、"爱奇艺短剧"等

## 执行步骤

### Step 1: 需求确认

1. 确认用户输入的题材、集数、风格、特殊要求
2. 询问是否需要特定的人物设定或情节偏好
3. 确认输出格式需求（详细剧本/大纲/逐集梗概）

### Step 2: 剧本框架生成

根据确认的需求，生成剧本框架，包含：

**1. 剧名**
- 朗朗上口，符合平台调性
- 体现核心冲突或爽点

**2. 类型与主题**
- 明确题材类型
- 提炼核心主题

**3. 人物小传**
每个主要人物包含：
- 全名
- 年龄
- 身份/职业
- 性格特点（3-5个关键词）
- 背景故事（简短）
- 核心动机
- 与其他人物的关系

**4. 情节结构**
采用三幕式或四幕式结构：
- 开篇（前10集）：建立人物和冲突
- 发展（中段50-60集）：冲突升级，伏笔铺设
- 高潮（后10集）：冲突爆发，真相揭露
- 结局（后10集）：冲突解决，人物成长

**5. 集数分配**
- 每集标注核心事件
- 标记重要转折点
- 标注钩子设置位置

### Step 3: 框架确认

将生成的框架展示给用户，等待确认后再继续。

### Step 4: 详细场景生成

根据确认的框架，逐集或分批生成详细场景。

**每集包含：**

1. **场次分配**
   - 格式：集数-场次（如 1-1, 1-2）
   - 每集通常3-5个场次

2. **基本信息**
   - 时间（日/夜）
   - 场景类型（内景/外景）
   - 地点
   - 出场人物

3. **场景内容**
   - 事件流程描述
   - 钩子设置（悬念型、危机型、转折型等）
   - 爽点/爆点/虐点设计

### Step 5: 对白与细节完善

为每个场景添加：

1. **对白**
   - 符合人物性格和说话风格
   - 推动情节发展
   - 暗示人物关系或动机

2. **动作描写**（用△标记）
   - 描述人物动作
   - 暗示情绪状态

3. **心理活动**（用OS标记）
   - 人物内心想法
   - 暗示隐藏动机

### Step 6: 质量检查

完成每集剧本后，检查以下项目：

- [ ] 时间线是否一致
- [ ] 人物状态是否连续
- [ ] 剧情逻辑是否合理
- [ ] 每集是否至少包含爽点/爆点/虐点之一
- [ ] 格式是否符合规范
- [ ] 对白是否符合人物性格
- [ ] 是否设置了有效钩子

### Step 7: 输出最终剧本

按照标准剧本格式输出。

## 输出要求

### 剧本格式规范

```
第X集

X-X
[时间]/[内景/外景] [地点]
[人物]

[场景描述]

△ [动作描写]
[人物]：[台词]
OS [心理活动]

【钩子：[钩子类型]——[钩子描述]】
```

### 格式说明

| 标记 | 含义 | 示例 |
|------|------|------|
| △ | 动作描写 | △ 他猛地站起来，狠狠拍了桌子 |
| OS | 心理活动 | OS（他到底想干什么？） |
| 【钩子：...】 | 钩子标注 | 【钩子：悬念型——他手中握着什么？】 |

### 质量要求

**1. 每集核心要素**
- 至少包含爽点、爆点或虐点之一
- 至少设置一个有效钩子
- 人物状态有合理变化

**2. 爽点公式**
```
装 + 打脸 + 震惊 + 收获
```
示例：主角展示隐藏能力 → 反派轻视被打脸 → 众人震惊 → 主角获得资源

**3. 爆点设计**
- 冲突升级，突破预期
- 反转剧情，改变局面
- 真相揭露，引发震撼

**4. 虐点设计**
- 人物遭遇困境
- 误解或背叛
- 失去重要的人或物

**5. 钩子类型（10种）**

| 类型 | 说明 | 示例 |
|------|------|------|
| 悬念型 | 提出疑问，引发好奇 | 他到底是谁？ |
| 危机型 | 制造紧迫感 | 还有3分钟！ |
| 转折型 | 突然改变方向 | 原来他一直骗了她 |
| 身份型 | 揭示人物真实身份 | 她竟然是公主！ |
| 情感型 | 触动情感 | 那个熟悉的笑容... |
| 秘密型 | 暗示隐藏信息 | 他抽屉里锁着什么？ |
| 选择型 | 面临艰难抉择 | 救他还是救孩子？ |
| 反转型 | 预期与实际相反 | 救人的人竟然是凶手 |
| 伏笔型 | 为后续铺垫 | 他的眼神一闪而过 |
| 冲突型 | 直接矛盾 | 你骗了我十年！ |

确保钩子类型与最近3集不同，避免重复。

## 状态维护

### 需要跟踪的信息

创作过程中需要维护以下状态：

**人物状态**
- 当前处境
- 情感状态
- 已获资源/能力
- 已知秘密/信息

**情节状态**
- 已发生的核心事件
- 已设置的伏笔
- 未解决的冲突
- 已用钩子类型（避免重复）

**关系状态**
- 人物关系变化
- 新产生的关系
- 关系的秘密

### 状态更新时机

- 每完成一集后更新
- 人物关系发生变化时更新
- 新伏笔设置时更新

## 注意事项

1. **用户确认**：剧本框架生成后必须等待用户确认，不得擅自继续
2. **人物一致性**：人物性格和动机前后一致，不得随意改变
3. **逻辑合理性**：剧情发展要有逻辑支撑，避免生硬转折
4. **钩子多样性**：确保钩子类型不与最近3集重复
5. **冲突兑现**：每个冲突必须有实质性后果，不能只是解释或洗白
6. **节奏把控**：注意情节节奏，避免过快或过慢

## 质量检查清单

完成剧本后，使用以下清单进行自检：

### 剧情层面
- [ ] 核心冲突清晰明确
- [ ] 情节发展有逻辑
- [ ] 悬念设置合理
- [ ] 反转有铺垫
- [ ] 结局与主题呼应

### 人物层面
- [ ] 主角动机明确
- [ ] 反派动机合理
- [ ] 人物性格一致
- [ ] 人物关系清晰
- [ ] 人物有成长

### 格式层面
- [ ] 场次编号正确
- [ ] 时间地点明确
- [ ] 人物标注准确
- [ ] 对白格式规范
- [ ] 标记使用正确

### 质量层面
- [ ] 每集有爽点/爆点/虐点
- [ ] 钩子设置有效
- [ ] 对白符合人物性格
- [ ] 动作描写生动
- [ ] 整体节奏合理

## 扩展适用范围

本Skill不仅适用于女频短剧，也可用于：

- **现代都市剧**：职场、家庭、都市情感
- **古装剧**：宫斗、武侠、历史
- **悬疑剧**：推理、惊悚、犯罪
- **喜剧**：轻松、搞笑、喜剧
- **其他类型**：根据用户需求灵活调整

不同类型可调整：
- 风格基调
- 钩子类型偏好
- 爽点/虐点比例
- 节奏快慢

## 故障处理

### 如果用户不满意框架
- 询问具体需要调整的地方
- 提供修改方案
- 重新生成框架

### 如果创作过程中用户提出新要求
- 评估新要求与现有框架的兼容性
- 如不冲突，融入后续创作
- 如冲突，提出修改建议

### 如果发现逻辑问题
- 主动指出问题
- 提供修复方案
- 确认后再继续
