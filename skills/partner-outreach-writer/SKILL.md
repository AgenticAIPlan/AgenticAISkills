# Partner Outreach Writer

为大模型产品生态运营团队生成高质量的合作触达文案。核心目标：**让对方感受到被专门研究过，而不是批量发送的模板**。

---

## 工作流程

### Step 1｜收集关键信息

在生成文案前，需确认以下信息（可从对话上下文提取，缺失的再向用户询问）：

**关于合作方（必填）**
- 合作方名称 / 类型（企业、开发者社区、媒体、高校、行业协会…）
- 他们的核心业务或主要受众
- 为什么选择找他们（你对他们的了解/观察）

**关于合作目标（必填）**
- 合作类型（见下方「合作类型速查表」）
- 己方希望获得什么
- 能为对方带来什么价值

**关于文案形式（有默认值）**
- 触达渠道：邮件 / 微信 / LinkedIn / 提案文档（默认：邮件）
- 语气风格：正式 / 半正式 / 亲切（默认：半正式）
- 文案长度：短（<200字）/ 中（200-400字）/ 长（>400字）（默认：中）

---

### Step 2｜选择合作类型

读取 `references/partner-types.md` 获取各类型的写作重点和注意事项。

**合作类型速查表：**

| 类型 | 说明 | 对应模板区 |
|------|------|-----------|
| `cold-outreach` | 无前期接触的冷启动邀约 | A区 |
| `event-collab` | 活动联办/赞助/联合出品 | B区 |
| `content-collab` | 联合内容：共创文章、案例、白皮书 | C区 |
| `ecosystem-join` | 邀请加入开发者/合作伙伴生态 | D区 |
| `follow-up` | 跟进已有接触但无回音的合作方 | E区 |
| `intro-reply` | 回应对方主动询问合作 | F区 |

---

### Step 3｜生成文案

根据信息和类型，生成文案时严格遵守以下原则：

#### ✅ 必须做到
- **开头提及具体细节**：引用对方最近的动作、产品特点或公开内容，证明做过研究
- **价值前置**：在第3句之前点出"对你们有什么好处"
- **一个清晰的行动号召**（CTA）：明确下一步是什么，只问一件事
- **适配渠道语气**：微信比邮件更口语，LinkedIn比邮件更简洁

#### ❌ 避免
- 空泛的"强强联合""资源互补"等无意义套话
- 一次提出多个合作方向（选一个最核心的）
- 在第一封信就要求对方做大动作（先约个短会 > 直接谈合作协议）
- 过度自我介绍（对方可以自己查背景）

---

### Step 4｜输出格式

**标准输出结构：**

```
【文案正文】
---
📌 写作说明
- 开头引用点：[说明引用了对方哪个具体细节]
- 核心价值主张：[一句话总结为对方带来什么]
- CTA 设计：[说明为什么选这个行动号召]

✏️ 可调整方向
- 如果对方更注重XX，可以把第X段改为…
- 如果渠道换成微信，建议删减…
```

如用户请求多个版本（A/B测试场景），生成2个版本，分别标注策略差异。

## 快速示例

**输入：**
> 帮我写一封邮件，邀请"极客公园"做我们开发者大会的媒体合作伙伴

**触发流程：**
1. 识别：合作类型 = `event-collab`，渠道 = 邮件
2. 提取/补充：合作方（极客公园，科技媒体）、活动信息、己方能给什么曝光
3. 生成：开头引用极客公园近期报道方向 → 点出活动受众与其读者重合 → CTA 约15分钟通话

## 边界说明

此技能专注于**触达和邀约阶段**的文案。以下内容超出范围：
- 合作协议/合同起草 → 使用 `mou-draft-assistant`
- 活动完整策划方案 → 使用 `event-brief-writer`
- 合作方背景调研 → 建议结合 web search 工具
